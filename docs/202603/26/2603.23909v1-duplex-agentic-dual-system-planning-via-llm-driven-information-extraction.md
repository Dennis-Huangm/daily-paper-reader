## 摘要
虽然大语言模型（LLMs）为机器人任务规划提供了语义灵活性，但其易受幻觉和逻辑不一致影响的特性限制了其在长程领域中的可靠性。为了弥合非结构化环境与严谨规划合成之间的差距，我们提出了 DUPLEX，这是一种代理式双系统神经符号架构，它将 LLM 严格限制在基于模式引导的信息抽取任务中，而非进行端到端规划或代码生成。在我们的框架中，前馈快系统（Fast System）利用轻量级 LLM 从自然语言中提取实体、关系等信息，并将其确定性地映射为经典符号规划器的规划领域定义语言（PDDL）问题文件。为了处理复杂或描述不足的场景，慢系统（Slow System）仅在规划失败时激活，利用求解器的诊断信息驱动高容量 LLM 进行迭代反思与修复。在 12 个经典及家庭规划领域的大量评估表明，DUPLEX 在成功率和可靠性方面均显著优于现有的端到端及混合 LLM 基准。这些结果证实，关键不在于提升 LLM 的规划能力，而在于将其限制在所擅长的领域——结构化语义对齐——并将逻辑规划合成交给符号规划器。

## Abstract
While Large Language Models (LLMs) provide semantic flexibility for robotic task planning, their susceptibility to hallucination and logical inconsistency limits their reliability in long-horizon domains. To bridge the gap between unstructured environments and rigorous plan synthesis, we propose DUPLEX, an agentic dual-system neuro-symbolic architecture that strictly confines the LLM to schema-guided information extraction rather than end-to-end planning or code generation. In our framework, a feed-forward Fast System utilizes a lightweight LLM to extract entities, relations etc. from natural language, deterministically mapping them into a Planning Domain Definition Language (PDDL) problem file for a classical symbolic planner. To resolve complex or underspecified scenarios, a Slow System is activated exclusively upon planning failure, leveraging solver diagnostics to drive a high-capacity LLM in iterative reflection and repair. Extensive evaluations across 12 classical and household planning domains demonstrate that DUPLEX significantly outperforms existing end-to-end and hybrid LLM baselines in both success rate and reliability. These results confirm that The key is not to make the LLM plan better, but to restrict the LLM to the part it is good at - structured semantic grounding - and leave logical plan synthesis to a symbolic planner.

### 1. 核心问题与研究动机
*   **核心问题**：如何在长程（Long-horizon）机器人任务规划中，既利用大语言模型（LLM）的语义灵活性，又克服其易产生幻觉、逻辑不一致以及生成形式化代码（如 PDDL）时极其脆弱的问题。
*   **研究背景**：
    *   **经典符号规划器**：逻辑严密、可验证，但存在“知识获取瓶颈”，难以将非结构化环境描述转化为严谨的符号表示。
    *   **LLM 直接规划**：擅长常识推理，但在复杂逻辑链条下极易失败，且生成 PDDL 代码时常因微小语法错误导致规划器崩溃。
*   **核心动机**：作者认为不应强求 LLM 提升规划能力，而应将其职责限制在它最擅长的“信息抽取”上，将逻辑合成交给专业的符号规划器。

### 2. 方法论
DUPLEX 提出了一种**代理式双系统神经符号架构**，其核心流程为：**非结构化输入 → 信息抽取 → 确定性映射 → 符号规划**。

*   **快系统（Fast System - 前馈路径）**：
    *   **引导式信息抽取**：使用轻量级 LLM（如 Qwen3-8B）根据预定义的 Schema（模式），从环境描述中提取 JSON 格式的实体（Objects）、关系（Relations）和状态（States）。
    *   **确定性映射**：通过预编程的脚本（而非 LLM）将 JSON 数据填充到 PDDL 模板中，确保生成的 Problem 文件 100% 符合语法规范。
    *   **符号求解**：调用 Fast Downward 等经典规划器生成动作序列。
*   **慢系统（Slow System - 代理反思路径）**：
    *   **触发机制**：仅在快系统规划失败（报错或无解）时激活。
    *   **三级修复机制**：
        1.  **规则级**：脚本自动修正简单的格式或类型缺失。
        2.  **语义级**：由高能力 LLM（如 GPT-4o）对照 Domain 文件修正幻觉产生的谓词或对象。
        3.  **逻辑级**：利用规划器的诊断信息（如“目标不可达”），驱动 LLM 推断并补全隐式状态（例如：若要运行微波炉，必须先识别出“门已关闭”这一隐式前提）。

### 3. 实验设计
*   **实验场景**：
    *   **IPC 经典领域**：选取了 8 个国际规划竞赛（IPC）领域（如 Barman, Blocksworld, Tyreworld 等），侧重逻辑严密性。
    *   **家庭机器人领域**：选取了 4 个具身智能场景（PC 组装、餐桌布置、家庭清洁、办公室整理），侧重语义理解和隐式约束。
*   **Benchmark**：IPC 2014 任务集及 DELTA 长程规划框架。
*   **对比方法**：
    1.  **LLM-as-Planner**：端到端直接生成动作序列。
    2.  **LLM+P**：LLM 直接将自然语言翻译为 PDDL 代码。
    3.  **DUPLEX Fast System**：仅使用快系统的消融版本。

### 4. 资源与算力
*   **硬件环境**：实验在一台配备 **1 张 NVIDIA RTX 4090 GPU (32 GB RAM)** 的 Linux 服务器上运行。
*   **模型使用**：
    *   快系统：Qwen3-8B（本地推理）。
    *   慢系统及基准模型：GPT-4o。
*   **时间限制**：LLM 生成限时 60 秒，符号规划限时 500 秒。

### 5. 实验数量与充分性
*   **实验规模**：
    *   IPC 领域：每个领域测试 20 个任务，共 160 组。
    *   家庭领域：4 个领域，每个领域 3 个实例，每个实例重复 50 次试验，共计 **600 次试验**。
*   **充分性评价**：实验设计较为充分且客观。通过跨越“纯逻辑”到“富语义”的 12 个不同领域，验证了模型的泛化能力；通过消融实验量化了“慢系统反思机制”带来的增益（在家庭领域提升了 32.6% 的成功率），证明了双系统设计的必要性。

### 6. 主要结论与发现
*   **性能飞跃**：DUPLEX 在 IPC 领域达到 97.5% 的成功率（LLM+P 为 71.9%），在家庭领域达到 83.5%（LLM+P 仅为 20.0%）。
*   **核心发现**：将 LLM 限制在“信息抽取”而非“代码生成”能彻底消除语法错误。
*   **反思的价值**：符号规划器的报错信息是极佳的反馈信号，能有效引导 LLM 发现并修正其认知盲点（如遗漏的初始状态）。

### 7. 优点与亮点
*   **架构解耦**：巧妙地将非确定性的语义理解（LLM）与确定性的逻辑推理（符号规划器）分离，取长补短。
*   **高可靠性**：通过脚本映射规避了 LLM 生成结构化代码时的脆弱性，保证了 100% 的语法正确率。
*   **效率优化**：日常任务由轻量级模型（快系统）处理，复杂错误才调用昂贵的大模型（慢系统），平衡了成本与性能。

### 8. 不足与局限
*   **先验依赖**：系统仍需预先人工编写 Domain PDDL 文件（定义动作的先验条件和效果），尚未实现完全自动化的领域建模。
*   **实时性挑战**：慢系统的迭代反思循环（LLM 思考 + 规划器重试）会带来显著延迟，可能不适用于对实时响应要求极高的动态环境。
*   **模态局限**：目前主要基于文本信息抽取，对于视觉环境中的空间歧义（如“左边的杯子”）处理能力尚待验证。
