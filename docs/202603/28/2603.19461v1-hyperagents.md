## 摘要
自我改进的 AI 系统旨在通过学习改进自身的学习和问题解决过程，从而减少对人工工程的依赖。现有的自我改进方法依赖于固定的、手工设计的元级机制，这从根本上限制了此类系统的改进速度。达尔文哥德尔机（Darwin Gödel Machine, DGM）通过反复生成和评估自我修改的变体，展示了在编程领域的开放式自我改进。由于评估和自我修改都是编程任务，编程能力的提升可以转化为自我改进能力的提升。然而，这种一致性在编程领域之外通常并不成立。我们引入了超智能体（hyperagents），这是一种自引用智能体，它将任务智能体（负责解决目标任务）和元智能体（负责修改自身和任务智能体）集成到一个单一的可编辑程序中。关键在于，元级修改过程本身也是可编辑的，从而实现了元认知自我修改，不仅改进了任务解决行为，还改进了生成未来改进的机制。我们通过扩展 DGM 创建了 DGM-超智能体（DGM-H）来实例化这一框架，消除了任务性能与自我修改技能之间领域特定一致性的假设，从而可能支持在任何可计算任务上实现自我加速的进展。在不同领域中，DGM-H 随着时间的推移不断提高性能，并优于没有自我改进或开放式探索的基准系统，以及先前的自我改进系统。此外，DGM-H 改进了其生成新智能体的过程（例如，持久化内存、性能跟踪），并且这些元级改进可以跨领域迁移并在多次运行中累积。DGM-超智能体让我们得以窥见开放式 AI 系统的前景，这类系统不仅在寻找更好的解决方案，还在不断改进其‘如何改进’的搜索过程。

## Abstract
Self-improving AI systems aim to reduce reliance on human engineering by learning to improve their own learning and problem-solving processes. Existing approaches to self-improvement rely on fixed, handcrafted meta-level mechanisms, fundamentally limiting how fast such systems can improve. The Darwin Gödel Machine (DGM) demonstrates open-ended self-improvement in coding by repeatedly generating and evaluating self-modified variants. Because both evaluation and self-modification are coding tasks, gains in coding ability can translate into gains in self-improvement ability. However, this alignment does not generally hold beyond coding domains. We introduce \textbf{hyperagents}, self-referential agents that integrate a task agent (which solves the target task) and a meta agent (which modifies itself and the task agent) into a single editable program. Crucially, the meta-level modification procedure is itself editable, enabling metacognitive self-modification, improving not only the task-solving behavior, but also the mechanism that generates future improvements. We instantiate this framework by extending DGM to create DGM-Hyperagents (DGM-H), eliminating the assumption of domain-specific alignment between task performance and self-modification skill to potentially support self-accelerating progress on any computable task. Across diverse domains, the DGM-H improves performance over time and outperforms baselines without self-improvement or open-ended exploration, as well as prior self-improving systems. Furthermore, the DGM-H improves the process by which it generates new agents (e.g., persistent memory, performance tracking), and these meta-level improvements transfer across domains and accumulate across runs. DGM-Hyperagents offer a glimpse of open-ended AI systems that do not merely search for better solutions, but continually improve their search for how to improve.

### 1. 核心问题与整体含义
*   **研究动机**：现有的自我改进 AI 系统（如达尔文哥德尔机 DGM）通常依赖于**手工设计且固定**的元级（meta-level）机制来生成改进指令。这导致了两个瓶颈：一是改进速度受限于初始设计；二是假设“任务解决能力”与“自我修改能力”是正相关的（这在编程领域成立，但在论文评审、数学批改等领域并不成立）。
*   **核心目标**：创建一个能够针对**任何可计算任务**进行自我改进的系统，不仅能改进任务执行逻辑，还能改进“改进自身”的机制（即元认知改进）。

### 2. 方法论
*   **Hyperagents（超智能体）概念**：将“任务代理”（解决目标任务）和“元代理”（修改代码）整合进同一个**自引用、可编辑**的 Python 程序中。
*   **元认知自我修改（Metacognitive Self-modification）**：由于元代理的代码也是程序的一部分，它可以在修改任务逻辑的同时，修改自己提出建议、分析错误和实施代码变更的逻辑。
*   **DGM-H 框架流程**：
    1.  **存档（Archive）**：维护一个不断增长的超智能体种群。
    2.  **选择（Selection）**：基于性能和“新颖性”（产生的成功后代数量）概率性地选择父代。
    3.  **修改**：父代超智能体调用大模型（FM）作为引擎，对自身代码库进行编辑，生成子代。
    4.  **评估**：在目标任务上运行子代，记录性能。
    5.  **迭代**：将编译通过且表现良好的子代加入存档，作为未来改进的“垫脚石”。

### 3. 实验设计
*   **实验领域与数据集**：
    *   **编程**：Polyglot 基准测试（165 个任务）。
    *   **论文评审**：预测 AI 论文的接受/拒绝（APRES 数据集，100 个任务）。
    *   **机器人奖励设计**：在 Genesis 物理引擎中为四足机器人设计奖励函数（行走与跳跃任务）。
    *   **奥数级数学批改**：IMO-GradingBench（100 个任务）。
*   **对比方法（Baselines）**：
    *   **DGM-H w/o self-improve**：固定元代理，仅允许修改任务代理（类似于 ADAS 方法）。
    *   **DGM-H w/o open-ended exploration**：不保留存档，仅进行单线演化。
    *   **原始 DGM**：针对编程优化的手工设计系统。
    *   **DGM-custom**：为特定领域手动定制改进逻辑的 DGM。
    *   **静态基准**：如 AI-Scientist-v2 的评审员、ProofAutoGrader 等。

*   **硬件**：未明确说明具体的 GPU 数量，实验主要通过 API 调用大模型完成。

### 5. 实验数量与充分性
*   **实验规模**：每个设置运行 5 次独立实验，取中位数并计算 95% 置信区间。
*   **充分性**：实验涵盖了从纯文本到具身智能（机器人）的多种任务，并进行了消融实验。特别设计了 **imp@k（improvement@k）** 指标来量化“改进能力”的提升。

### 6. 主要结论与发现
*   **性能超越**：DGM-H 在非编程领域显著优于原始 DGM 和不具备自我改进能力的基准。
*   **元级改进的迁移性**：在论文评审中学习到的“改进策略”（如性能追踪、持久化内存）可以成功迁移到数学批改任务中，加速新领域的学习。
*   **能力累积**：自我改进的效果可以在不同运行和领域之间累积，展示了开放式进化的潜力。
*   **涌现行为**：系统自动开发出了性能监控工具、偏见检测机制和复杂的提示词模板系统，而这些并未在初始代码中定义。

### 7. 优点
*   **通用性**：打破了自我改进仅限于编程领域的局限，适用于任何可计算任务。
*   **自引用架构**：通过将元逻辑代码化，解决了元级机制僵化的问题，实现了真正的递归自我改进。
*   **定性分析深入**：论文详细展示了代码演化的 Diff 补丁，证明了系统确实学习到了结构化的决策逻辑（如从“态度指令”转向“两阶段评估流程”）。

### 8. 不足与局限
*   **任务分布固定**：虽然代理在进化，但任务集是预先定义的，缺乏任务与代理的协同进化（Co-evolution）。
*   **外部循环限制**：父代选择和评估协议等最外层逻辑仍是固定的，尚未实现完全的自我修改。
*   **成本高昂**：依赖顶级大模型的 API 调用，大规模部署的经济成本较高。
*   **评估博弈风险**：存在“指标欺骗”（Goodhart's Law）的风险，即代理可能学会通过作弊或利用评估漏洞来获得高分。
