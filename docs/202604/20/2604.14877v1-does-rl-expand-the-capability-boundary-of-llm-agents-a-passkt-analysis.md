## 摘要
强化学习是真正扩展了大语言模型（LLM）智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究倾向于后者：基础模型和强化学习模型的 pass@k 曲线在大 k 值下趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，因为在该场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种联合改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习真正扩大了能力边界：强化学习智能体的 pass 曲线高于基础模型，且差距在大 k 值下逐渐扩大而非收敛。这种扩展专门针对组合式、序列化的信息收集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在匹配训练数据的情况下，有监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新分配了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观与悲观解读：两者在不同的任务类型下都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 核心问题与研究动机
*   **核心问题**：强化学习是真正扩展了 LLM 智能体的“能力边界”（即让它学会了以前完全不会的东西），还是仅仅提高了“效率”（即让它更可靠地完成以前偶尔能做对的事情）？
*   **研究背景**：在静态数学推理领域，近期研究（如 Yue et al., 2025）认为 RL 只是重新分配了概率质量，并未扩大能力边界。本论文质疑这一悲观结论是否适用于“智能体工具使用”场景。在智能体场景中，多轮环境交互（T）产生的组合策略是单纯重采样（k）无法替代的。

### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：引入一个二维评估指标 **PASS@(k, T)**，将智能体的表现分解为两个维度：
    *   **k（采样预算）**：独立尝试的次数，用于衡量采样效率和可靠性。
    *   **T（交互深度）**：每轮尝试中与环境交互的最大回合数，用于衡量处理复杂组合任务的能力。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：在无限次重采样（大 k）下，模型能够解决的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 模型解决了一些基础模型在任何采样预算下都无法解决的问题。
    *   **效率提升 (Efficiency Improvement)**：RL 模型在 $k=1$ 时表现更好，但在大 $k$ 时与基础模型趋同。
*   **训练算法**：使用 **GRPO**（群体相对策略优化）进行强化学习，并与在相同数据上进行的 **SFT**（有监督微调）进行对比，以隔离学习信号（探索 vs. 模仿）的影响。

### 3. 实验设计
*   **任务分类**：
    *   **Category A (纯推理)**：MATH-500 数据集，不使用工具（对照组）。
    *   **Category B (独立检索)**：HotPotQA 比较类问题，需两次互不干扰的检索。
    *   **Category C (序列检索/组合任务)**：HotPotQA 桥接类问题，第二次检索依赖第一次的结果，具有高度组合性。
*   **对比方法**：基础模型 (Qwen2.5-7B-Instruct)、SFT 模型、RL 模型、带探索奖励的 RL 模型 (RL+explore)。
*   **Benchmark**：基于 HotPotQA 的检索环境，使用 BM25 检索器作为工具。

### 4. 资源与算力
*   **硬件**：单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：RL 训练（1000 步，10 个 epoch）大约耗时 **55 小时**。
*   **细节**：使用 LoRA (rank 16) 进行微调，8-bit Adam 优化器。

### 5. 实验数量与充分性
*   **实验规模**：每个类别 100 个测试问题，共 300 个。每个 (q, T) 单元进行 **n=64** 次独立采样以计算 PASS@(k, T)。
*   **充分性**：
    *   通过子采样验证（n=32, 48），证明 n=64 足以提供稳定的估计。
    *   使用了 **Bootstrap 置信区间** 验证能力边界变化的显著性。
    *   **公平性**：SFT 和 RL 使用完全相同的 200 个训练问题，确保性能差异源于学习机制而非数据暴露。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的组合任务（Category C）中，RL 模型的 pass 曲线在大 k 值下不仅没有与基础模型收敛，反而拉开了差距（+4% 绝对值），证明了能力的实质扩展。
*   **SFT 的负面效应**：在相同的组合任务上，SFT 反而导致了能力边界的**退化**（-4%），因为它倾向于让模型模仿特定的专家路径，从而抑制了多样性的探索。
*   **任务依赖性**：在简单任务（Cat B）或纯推理任务（Cat A）上，RL 的表现符合“效率提升”或“无影响”的预期，这解释了为何先前研究结论较为悲观。
*   **机制解释**：RL 并不是学会了更奇特的搜索词（检索侧），而是优化了**如何整合检索到的信息进行推理**（推理侧）。RL 通过“重加权”机制，将概率质量移向了那些能导致正确答案的现有策略子集。

### 7. 优点
*   **指标创新**：PASS@(k, T) 首次在智能体评估中解耦了“采样可靠性”和“交互深度”，为后续研究提供了标准工具。
*   **因果隔离**：通过匹配 SFT 和 RL 的训练数据，有力地证明了“自主探索”是能力扩展的关键驱动力。
*   **深入的机制分析**：结合困惑度（Perplexity）分解、策略多样性测量和跨策略交换实验，从多个角度验证了 RL 如何重塑模型行为。

### 8. 不足与局限
*   **模型规模限制**：主要实验基于 7B 模型，尚未在更大规模（如 70B+）或更强的基础模型上验证该结论的普适性。
*   **工具单一性**：仅使用了简单的搜索工具，未涉及更复杂的 API 调用或多工具协作场景。
*   **训练数据量较小**：仅使用了 200 个训练样本，虽然足以观察到趋势，但更大规模的 RL 训练可能会揭示更复杂的能力演化过程。
*   **环境闭环**：检索环境是静态且确定性的（BM25），在动态或随机环境中 RL 的表现可能有所不同。
