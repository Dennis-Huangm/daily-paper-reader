## 摘要
强化学习 (RL) 已广泛用于训练多轮交互任务的大语言模型 (LLM) 智能体，但其样本效率受到稀疏奖励和长时序的严重限制。在线策略自蒸馏 (OPSD) 通过提供来自能够访问标准答案的特权教师的密集 Token 级监督来缓解这一问题。然而，这种固定的特权信息无法捕捉智能体任务中多样化的有效策略，且简单地将 OPSD 与 RL 结合往往会导致训练崩溃。为了解决这些局限性，我们提出了 Skill-SD，这是一个将智能体自身的轨迹转化为动态的、仅限训练的监督信息的框架。完成的轨迹被总结为简洁的自然语言技能，描述了成功的行为、错误和工作流。这些技能作为动态特权信息仅对教师进行条件化，而学生始终在普通任务提示下行动，并通过蒸馏学习内化这些指导。为了稳定训练，我们推导了一种重要性加权的反向 KL 损失，以提供梯度正确的 Token 级蒸馏，并动态地将教师与不断进步的学生同步。在智能体基准测试上的实验结果表明，Skill-SD 显著优于标准的 RL 基线，在 AppWorld 和 Sokoban 任务上分别提升了原生 GRPO (+14.0%/+10.9%) 和原生 OPD (+42.1%/+40.6%)。项目页面：https://k1xe.github.io/skill-sd/

## Abstract
Reinforcement learning (RL) has been widely used to train LLM agents for multi-turn interactive tasks, but its sample efficiency is severely limited by sparse rewards and long horizons. On-policy self-distillation (OPSD) alleviates this by providing dense token-level supervision from a privileged teacher that has access to ground-truth answers. However, such fixed privileged information cannot capture the diverse valid strategies in agent tasks, and naively combining OPSD with RL often leads to training collapse. To address these limitations, we introduce Skill-SD, a framework that turns the agent's own trajectories into dynamic training-only supervision. Completed trajectories are summarized into compact natural language skills that describe successful behaviors, mistakes, and workflows. These skills serve as dynamic privileged information conditioning only the teacher, while the student always acts under the plain task prompt and learns to internalize the guidance through distillation. To stabilize the training, we derive an importance-weighted reverse-KL loss to provide gradient-correct token-level distillation, and dynamically synchronize the teacher with the improving student. Experimental results on agentic benchmarks demonstrate that Skill-SD substantially outperforms the standard RL baseline, improving both vanilla GRPO (+14.0%/+10.9% on AppWorld/Sokoban) and vanilla OPD (+42.1%/+40.6%). #### 1. 核心问题与研究动机
在训练多轮交互式大语言模型（LLM）智能体时，强化学习（RL）面临两大挑战：
*   **奖励稀疏与长时序：** 任务往往只有最终的成功/失败反馈，导致学习效率极低。
*   **策略多样性：** 与数学题不同，智能体任务（如 API 调用或路径规划）通常有多种有效路径。现有的自蒸馏方法依赖固定的“标准答案”作为特权信息，限制了智能体的探索，且直接结合 RL 与蒸馏易导致训练崩溃。

**研究动机：** 探索如何利用智能体自身的历史经验（轨迹）生成动态的、非限制性的指导信号，在不引入推理负担的前提下提升 RL 的训练效率。

#### 2. 方法论：核心思想与关键技术
Skill-SD 框架的核心是将智能体轨迹转化为“技能”，并通过一种改进的自蒸馏机制让模型内化这些技能。

*   **技能作为动态特权信号：** 
    *   将完成的轨迹异步总结为包含“成功分析”、“错误分析”和“黄金工作流”的自然语言技能。
    *   **教师模型（Teacher）：** 输入“任务提示 + 检索到的技能”。
    *   **学生模型（Student）：** 仅输入“任务提示”。学生通过蒸馏学习教师在有技能辅助下的 Token 分布。
*   **重要性加权的逆 KL 损失（SDL Loss）：**
    *   由于教师和学生使用不同的 Prompt（Prompt Discrepancy），会导致分布偏移。
    *   论文推导了一种重要性采样权重 $\rho$ 来修正 Token 级别的梯度偏差，确保在跨 Prompt 蒸馏时梯度更新是无偏的。
*   **动态教师同步：** 教师模型不再是固定的，而是定期从最新的学生检查点同步参数，确保指导信号随模型能力的提升而演进。
*   **联合优化目标：** 结合了 GRPO（组相对策略优化）损失和 SDL（自蒸馏）损失，前者负责任务级的奖励最大化，后者负责 Token 级的行为引导。

#### 6. 主要结论与发现
*   **性能大幅提升：** Skill-SD 在 AppWorld 上比 GRPO 提升了 **14.0%** 的准确率，在 Sokoban 上提升了 **10.9%**。
*   **技能应引导教师而非学生：** 直接给学生喂技能（Prompting）会导致严重的过拟合和推理依赖，而通过蒸馏（Distillation）让学生内化知识效果更好。
*   **动态同步至关重要：** 固定的教师会导致性能过早达到瓶颈，而动态同步能维持蒸馏信号的有效性。
*   **离策（Off-policy）崩溃：** 实验发现如果由教师生成轨迹供学生学习，在复杂任务中极易导致训练崩溃，证明了学生自主探索（On-policy）的重要性。

#### 7. 优点与亮点
*   **创新的特权信息设计：** 将复杂的轨迹简化为结构化的自然语言技能，既保留了策略多样性，又提供了密集的监督信号。
*   **理论严谨性：** 针对跨 Prompt 蒸馏推导了重要性加权损失，解决了 Token 级别梯度偏差的问题。
*   **推理零成本：** 技能仅在训练阶段使用，推理时学生模型不需要检索或额外的 Prompt，保持了轻量化。
*   **训练稳定性：** 成功解决了 RL 与自蒸馏结合时常见的训练崩溃问题。

#### 8. 不足与局限
*   **技能检索机制简单：** 目前采用 UCB 算法进行检索，而非基于语义的向量检索，在技能库规模极大时可能效率受限。
*   **任务池规模较小：** 实验主要集中在 90-100 个任务规模的小型训练集上，在大规模异构任务上的泛化性仍需验证。
*   **依赖外部总结器：** 技能的生成依赖于一个辅助的 LLM（如 Seed1.8），这引入了额外的预处理成本。
*   **Token 级蒸馏的局限：** 仅对采样到的 Token 进行蒸馏，而非全词表蒸馏，可能损失了一部分概率分布的信息。
