## 摘要
大语言模型（LLM）智能体的强化学习通常在静态数据分布上进行，这无法适应智能体不断演化的行为，并导致对复杂环境交互的覆盖不足。为了应对这些挑战，我们提出了 CoEvolve，这是一个智能体-数据协同演化框架，使 LLM 智能体能够通过闭环、交互驱动的训练进行提升。具体而言，CoEvolve 从展开轨迹中提取遗忘和不确定性等反馈信号，以识别易出错的交互模式，并利用这些信号指导基于 LLM 的任务合成。合成的任务通过环境交互进行验证，并用于更新数据分布，从而实现智能体及其数据的联合自适应。在 AppWorld 和 BFCL 基准测试上，针对 Qwen2.5-7B、Qwen3-4B 和 Qwen3-30B-A3B 进行的广泛实验表明，该方法相较于强基座模型取得了持续且显著的提升，绝对增益分别达到 19.43%、15.58% 和 18.14%。

## Abstract
Reinforcement learning for LLM agents is typically conducted on a static data distribution, which fails to adapt to the agent's evolving behavior and leads to poor coverage of complex environment interactions. To address these challenges, we propose CoEvolve, an agent-data mutual evolution framework that enables LLM agents to improve through closed-loop, interaction-driven training. Specifically, CoEvolve extracts feedback signals such as forgetting and uncertainty from rollout trajectories to identify failure-prone interaction patterns, and utilizes them to guide LLM-based task synthesis. The synthesized tasks are validated through environment interaction and utilized to update the data distribution, enabling joint adaptation of the agent and its data. Extensive experiments on AppWorld and BFCL across Qwen2.5-7B, Qwen3-4B, and Qwen3-30B-A3B demonstrate consistent and significant improvements over strong base models, yielding absolute gains of 19.43%, 15.58%, and 18.14%, respectively.

## 1. 核心问题与研究动机
当前大语言模型（LLM）智能体的强化学习（RL）训练主要面临两个瓶颈：
*   **静态数据分布的局限性：** 现有的训练方法（如模仿学习或基于静态合成数据的 RL）依赖于预先收集的专家轨迹或随机生成的合成任务。这些数据无法随着智能体能力的提升而演化，难以覆盖智能体在实际交互中遇到的“长尾”失败场景。
*   **数据获取成本高：** 人工标注专家轨迹极其昂贵且耗时，而无指导的合成数据生成（Open-loop）往往流于表面，缺乏对复杂环境配置的深度探索。

**核心目标：** 提出一种“智能体-数据协同演化”的闭环框架，让训练数据能够根据智能体当前的薄弱环节动态生成，实现智能体能力与训练分布的同步进化。

## 2. 方法论
CoEvolve 框架包含三个核心阶段，形成一个无需人工干预的闭环：

### 阶段 1：训练与信号提取
*   **策略优化：** 使用 **GRPO（Group Relative Policy Optimization）** 算法对智能体进行优化。
*   **弱点信号提取：** 从训练过程中的展开轨迹（Rollout Trajectories）中识别三类关键信号：
    1.  **遗忘信号 (Forgetting Signals)：** 智能体之前能成功但当前策略下失败的任务。
    2.  **边界信号 (Boundary Signals)：** 在同一任务的多次采样中结果不稳定（时而成功时而失败），表明处于决策边界。
    3.  **稀有信号 (Rare Signals)：** 出现频率极低但重复发生的动作模式，代表探索不足。

### 阶段 2：信号引导的环境再探索
*   **上下文构建：** 利用 LLM 对带有信号标注的失败轨迹进行反思，总结失败原因。
*   **引导探索：** 以总结的上下文为条件，驱动一个强大的 LLM（如 Qwen3-Max）在环境中进行多轮、多步的再探索，寻找能够克服当前弱点的替代行为。

### 阶段 3：任务抽象与验证
*   **任务合成：** 将探索得到的交互三元组（动作、观察、ID）抽象为新的任务描述和解决方案对。
*   **环境验证：** 在真实环境中执行合成的任务，仅保留能够成功完成或获得正向奖励的任务，并将其加入训练集 $D_t$，更新下一轮的训练分布。
