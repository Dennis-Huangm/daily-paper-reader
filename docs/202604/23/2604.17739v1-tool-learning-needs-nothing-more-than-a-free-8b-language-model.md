## 摘要
强化学习（RL）已成为训练工具调用智能体（tool calling agents）的流行范式，这通常需要在线交互环境。现有方法要么依赖于带有标准答案（ground truth）标注的训练数据，要么需要先进的商业语言模型（LMs）来合成一旦创建就保持固定的环境。在这项工作中，我们提出了 TRUSTEE，这是一种无数据（data-free）方法，通过完全由免费开源语言模型（小至 8B）模拟的动态环境来训练工具调用智能体。该方法涵盖了任务生成、用户模拟、工具模拟和轨迹评估，并辅以一种在训练过程中动态控制任务难度各方面的自适应课程学习机制。我们的实证结果表明，TRUSTEE 在各个领域都带来了持续的改进，并优于所有需要额外外部资源进行训练的基准模型。这些结果证实，通过足够精巧的设计，即使是以本地 8B 语言模型为骨干的模拟环境，也能为工具学习建立强大的基准，而无需昂贵的标注数据、真实的人类交互、可执行工具，或来自人类专家及商业语言模型的高成本可验证环境。我们希望所提出的范式能够启发未来在有限资源下进行环境扩展（environment scaling）的研究。

## Abstract
Reinforcement learning (RL) has become a prevalent paradigm for training tool calling agents, which typically requires online interactive environments. Existing approaches either rely on training data with ground truth annotations or require advanced commercial language models (LMs) to synthesize environments that keep fixed once created. In this work, we propose TRUSTEE, a data-free method training tool calling agents with dynamic environments fully simulated by free open-source LMs that can be as small as 8B, including task generation, user simulation, tool simulation and trajectory evaluation, paired with an adaptive curriculum learning mechanism that controls various aspects of the task difficulty dynamically during training. Our empirical results show that TRUSTEE brings consistent improvements across various domains and outperforms all the baselines which require extra external resources for training. These confirm that, with a sufficiently sophisticated design, even simulated environments with a local 8B LM as the backbone could set a strong baseline for tool learning, without expensive annotated data, realistic human interactions, executable tools or costly verifiable environments from human experts or commercial LMs. We hope our proposed paradigm could inspire future research on environment scaling with limited resources.

### 1. 核心问题与研究动机
*   **研究背景**：工具调用（Tool Calling）是使大语言模型（LM）成为主动智能体的关键。目前训练此类智能体的主流范式是强化学习（RL），但 RL 极度依赖交互环境。
*   **核心问题**：现有的环境构建方法存在显著局限：
    1.  **高成本**：依赖昂贵的人工标注数据或商业模型（如 GPT-4）来生成合成数据。
    2.  **不可扩展性**：真实世界的 API 调用昂贵且不稳定。
    3.  **静态性**：现有的合成环境一旦生成往往是固定的，无法根据智能体的当前能力动态调整难度。
*   **研究目标**：探索是否仅利用**本地、免费、小规模（8B）**的开源模型，就能构建一个完整的动态模拟环境，从而实现高效的工具学习。

### 2. 方法论：TRUSTEE 框架
TRUSTEE（Tool leaRning Using SimulaTEd Environments）是一个完全无数据（Data-free）的训练框架，其核心思想是**“用模型模拟世界”**。

*   **全流程模拟（Fully Simulated Environment）**：
    1.  **任务生成（Task Generation）**：根据工具库随机采样工具，由 8B 模型生成用户画像、意图、初始查询和预期调用的工具。
    2.  **环境交互（Simulated Interaction）**：
        *   **用户模拟器（User Simulator）**：模拟人类用户与智能体进行多轮对话。
        *   **工具模拟器（Tool Simulator）**：模拟 API 的执行结果（无需真实执行代码）。
    3.  **奖励与优化（Rewarding & Optimization）**：
        *   **验证模拟器（Verifier Simulator）**：评估整个交互轨迹，给出最终奖励。
        *   **算法**：采用 **GRPO**（Group Relative Policy Optimization）进行策略优化。

*   **自适应课程学习（Adaptive Curriculum Learning, ACL）**：
    这是 TRUSTEE 的关键技术。系统根据智能体在前一步的平均奖励动态调整任务难度：
    *   **难度维度**：包括可用工具数量、预期调用次数、交互轮数、系统提示词的详细程度、用户画像（专家到新手）、查询模糊度以及奖励准则的严苛程度。
    *   **演化机制**：如果奖励高则增加难度，反之降低，确保智能体始终在“最近发展区”学习。

### 3. 实验设计
*   **数据集与基准（Benchmarks）**：
    *   **BFCL V4 (Berkeley Function Calling Leaderboard)**：评估单轮和多轮工具调用的综合能力。
    *   **$\tau^2$-bench**：评估在航空、零售、电信三个真实场景下的多轮交互工具调用。
*   **对比方法（Baselines）**：
    *   **ToolRL / Tool-N1**：依赖标注数据的单轮 RLVR 方法。
    *   **Simia-RL**：使用商业模型模拟工具响应的方法。

*   **基础模型**：使用 **Qwen3-8B** 作为智能体模型，同时也作为所有模拟组件（任务生成、用户、工具、奖励）的骨干模型。

*   **硬件配置**：使用了单节点 **8 × NVIDIA H20 GPU**。
*   **算力分配**：4 块 GPU 用于智能体训练，另外 4 块 GPU 用于部署 vLLM 服务器以运行模拟环境模型。
*   **训练细节**：总计训练 200 个 Step，每个 Step 包含 16 个任务（Batch Size），每个任务采样 8 条轨迹（Group Size）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了单轮（Simple, Multiple, Parallel 等）和多轮（Missing Function, Long Context 等）共计十余个细分子任务。
*   **充分性评价**：
    *   **对比全面**：对比了从 SFT 到基于程序合成的多种前沿 RL 方法。
    *   **跨域验证**：在 BFCL 和 $\tau^2$-bench 两个性质不同的基准上均进行了测试。
    *   **客观性**：实验结果显示 TRUSTEE 在多个指标上不仅超过了开源基准，甚至优于那些使用了 GPT-4 等商业资源的方法，证明了其方案的有效性。

### 6. 主要结论与发现
1.  **8B 模型足矣**：证明了工具学习不需要昂贵的标注或商业模型，一个经过精巧设计的 8B 开源模型模拟环境足以训练出强大的智能体。
2.  **性能领先**：TRUSTEE 在 BFCL 和 $\tau^2$-bench 上均取得了最高平均分，尤其在复杂的多轮交互场景中提升显著。
3.  **动态环境的优势**：相比于静态合成环境，TRUSTEE 的动态模拟和自适应课程学习能更好地促进模型泛化。

### 7. 优点与亮点
*   **极低成本**：完全摆脱了对昂贵 API、人工标注和商业模型的依赖，极大地降低了工具学习的门槛。
*   **高度动态化**：通过 ACL 机制实现了训练难度的平滑过渡，解决了 RL 训练中常见的奖励稀疏或难度跨度过大的问题。
*   **通用性强**：不依赖特定的可执行代码环境，理论上可以扩展到任何有文本描述的工具领域。

### 8. 不足与局限
*   **模拟器能力上限**：模拟环境的质量受限于 8B 模型的认知水平。如果任务极其复杂（如涉及数百个工具或极长轮次），8B 模型可能无法提供准确的模拟和评估。
*   **超参数敏感性**：ACL 机制涉及大量超参数（如难度步长、奖励阈值等），寻找最优配置可能具有挑战性。
*   **稳定性风险**：由于 LM 的随机性，模拟器偶尔会产生无效响应，可能导致训练轨迹被废弃，影响训练稳定性。
