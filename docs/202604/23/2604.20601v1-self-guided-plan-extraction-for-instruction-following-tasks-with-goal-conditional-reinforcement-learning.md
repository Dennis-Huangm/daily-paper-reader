## 摘要
我们介绍了 SuperIgor，一个用于指令遵循任务的框架。与以往依赖预定义子任务的方法不同，SuperIgor 通过自学习机制使语言模型能够生成并优化高层计划，从而减少了对手动数据集标注的需求。我们的方法涉及迭代协同训练：训练强化学习（RL）智能体以遵循生成的计划，同时语言模型根据 RL 的反馈和偏好来调整和修改这些计划。这形成了一个反馈循环，使得智能体和规划器能够共同提升。我们在具有丰富动力学特性和随机性的环境中验证了我们的框架。结果表明，SuperIgor 智能体比基线方法更严格地遵循指令，同时也展现出对先前未见指令的强大泛化能力。

## Abstract
We introduce SuperIgor, a framework for instruction-following tasks. Unlike prior methods that rely on predefined subtasks, SuperIgor enables a language model to generate and refine high-level plans through a self-learning mechanism, reducing the need for manual dataset annotation. Our approach involves iterative co-training: an RL agent is trained to follow the generated plans, while the language model adapts and modifies these plans based on RL feedback and preferences. This creates a feedback loop where both the agent and the planner improve jointly. We validate our framework in environments with rich dynamics and stochasticity. Results show that SuperIgor agents adhere to instructions more strictly than baseline methods, while also demonstrating strong generalization to previously unseen instructions.

### 1. 核心问题与整体含义（研究动机和背景）
论文探讨了具身智能体如何遵循自然语言指令完成复杂任务。
*   **研究动机**：现有的指令遵循方法要么依赖昂贵的专家演示（学习成本高），要么依赖预定义的子任务库（灵活性差）。在缺乏预定义技能且奖励稀疏的动态环境中，如何让智能体自主理解长程指令并将其转化为可执行的动作序列是一个重大挑战。
*   **核心问题**：如何在没有人工标注子任务的情况下，利用大语言模型（LLM）的常识规划能力与强化学习（RL）的执行能力，构建一个能够自我进化、自我引导的指令遵循框架。

### 2. 方法论：SuperIgor 框架
SuperIgor 采用了一种**迭代协同训练（Co-training）**的机制，主要包含四个阶段：
1.  **计划生成（Plan Generation）**：
    *   **子任务库构建**：LLM 从指令集中提取原子化的子任务。
    *   **本体诱导（Ontology Induction）**：LLM 预测子任务间的依赖关系（例如“挖铁矿”前需要“制作石镐”），构建本体图。
    *   **计划扩展**：基于本体图将高层指令展开为多个候选的线性计划。
2.  **策略学习（Policy Learning）**：
    *   使用 **PPO 算法**训练 RL 智能体。
    *   **技能课程学习（Skill Curriculum Learning）**：引入动态课程，智能体先学习简单的单步任务，只有当某项技能的成功率达到阈值 $\tau$（实验设定为 0.7）时，才将其加入更复杂的长程计划中。
3.  **计划验证（Plan Validation）**：在随机环境中多次运行智能体，评估不同候选计划的平均成功率。
4.  **LLM 微调（LLM Fine-Tuning）**：
    *   利用 **直接偏好优化（DPO）**，将 RL 智能体的实际执行表现作为偏好信号反馈给 LLM。
    *   LLM 学习优先生成那些智能体更容易学会且更有效的计划，实现规划器与执行器的对齐。

### 3. 实验设计
*   **场景与数据集**：使用 **CrafText** 基准测试（一个类似 Minecraft 的动态、部分可观测环境）。研究者构建了 **FOCUS 数据集**，包含 900 多条指令，分为：

    *   **Combo**：多步骤组合任务。
    *   **Paraphrases**（测试集）：相同目标但表达方式不同的指令。

    *   **PPO-T+**：使用 GPT-4 生成固定计划的 RL 智能体。
    *   **Oracle**：使用人工编写的完美计划进行训练的智能体。

### 6. 主要结论与发现
*   **性能领先**：SuperIgor 在 Atomic 和 Combo 任务上的成功率远超非规划类基线（如 PPO-T）。
*   **泛化性强**：在“New Objects”测试中，SuperIgor 的表现甚至比 Oracle（人工计划）更稳健，因为自生成的多种候选计划增加了训练的覆盖面。
*   **协同效应**：DPO 微调能显著提升 LLM 生成计划的质量，使其更符合环境动力学和智能体的实际能力。
*   **课程学习是关键**：在稀疏奖励环境下，没有技能课程学习，智能体几乎无法学会任何复杂任务。

### 7. 优点
*   **无需人工标注**：实现了从原始指令到子任务发现、计划生成、策略学习的全自动化闭环。
*   **自适应性**：通过 DPO 将 LLM 的常识与环境的实际约束（RL 反馈）结合，解决了 LLM 规划中常见的“幻觉”或不切实际的问题。
*   **鲁棒性**：对指令的语言变体（Paraphrases）具有很强的抵抗力。

### 8. 不足与局限
*   **初始计划依赖**：框架高度依赖 LLM 初始生成的本体质量，如果 LLM 无法正确理解基础逻辑，后续训练会受阻。

*   **可解释性瓶颈**：当训练停滞时，很难界定是 RL 探索不足还是 LLM 计划空间设计有误。
*   **环境限制**：目前仅在 2D 模拟环境（CrafText）中验证，在更复杂的 3D 真实物理环境中的表现尚待观察。
