## 摘要
智能体检索增强生成（Agentic RAG）系统通过与外部检索工具的多步交互，使大语言模型（LLMs）能够解决复杂任务。然而，这种多步交互通常涉及冗余的搜索步骤，从而导致巨大的计算成本和延迟。先前的工作通过限制搜索深度（即搜索步数）来降低成本，但这往往导致对复杂问题的探索不足。为了解决这一问题，我们首先研究了搜索深度如何影响准确性，并发现了一个定义准确率与效率权衡的最小充分搜索深度，该深度由问题复杂度和智能体能力共同决定。此外，我们提出了 AutoSearch，这是一个强化学习（RL）框架，通过自生成的中间答案来评估每个搜索步骤。通过自回答机制，AutoSearch 能够识别最小充分搜索深度，并通过奖励达到该深度同时惩罚过度搜索来促进高效搜索。此外，我们还引入了奖励机制以稳定搜索行为并提高复杂问题的回答质量。在多个基准测试上的广泛实验表明，AutoSearch 实现了卓越的准确率与效率权衡，在保持搜索质量的同时缓解了过度搜索问题。

## Abstract
Agentic retrieval-augmented generation (RAG) systems enable large language models (LLMs) to solve complex tasks through multi-step interaction with external retrieval tools. However, such multi-step interaction often involves redundant search steps, incurring substantial computational cost and latency. Prior work limits search depth (i.e., the number of search steps) to reduce cost, but this often leads to underexploration of complex questions. To address this, we first investigate how search depth affects accuracy and find a minimal sufficient search depth that defines an accuracy-efficiency trade-off, jointly determined by question complexity and the agent's capability. Furthermore, we propose AutoSearch, a reinforcement learning (RL) framework that evaluates each search step via self-generated intermediate answers. By a self-answering mechanism, AutoSearch identifies the minimal sufficient search depth and promotes efficient search by rewarding its attainment while penalizing over-searching. In addition, reward mechanisms are introduced to stabilize search behavior and improve answer quality on complex questions. Extensive experiments on multiple benchmarks show that AutoSearch achieves a superior accuracy-efficiency trade-off, alleviating over-searching while preserving search quality.

### 2. 方法论
AutoSearch 的核心是一个基于自回答驱动的强化学习框架，其关键技术细节包括：
*   **自回答机制（Self-Answering Mechanism）**：在每一个检索步骤 $t$，模型都会基于当前已获取的信息生成一个“中间答案”。
*   **能力感知最优深度（$t_c$）**：通过中间答案与标准答案的对比，确定模型能够答对问题的最早步骤 $t_c$。这被定义为该模型处理该问题的“最小充分搜索深度”。
*   **奖励函数设计**：
    *   **基础奖励（Base Reward）**：包括格式奖励（确保输出结构正确）和结果奖励（最终答案是否正确）。
    *   **搜索效率奖励（Search Efficiency Reward）**：这是核心创新。如果当前步数 $t \le t_c$，给予正奖励鼓励有效搜索；如果 $t > t_c$，则视为“过度搜索”并给予负惩罚。
    *   **搜索质量奖励（Search Quality Reward）**：计算当前中间答案与标准答案的 F1 分数增益，奖励那些能显著提升信息质量的检索步骤。
*   **训练算法**：主要使用 **PPO**（近端策略优化）算法，并证明了该框架同样适用于 **GRPO** 等结果导向的 RL 算法。在训练中对外部检索到的文本进行了 Token 级别的掩码处理，以聚焦于策略学习。

### 6. 主要结论与发现
*   **性能领先**：AutoSearch 在所有数据集上均实现了最高的搜索效率（SE），在保持或提升准确率的同时，显著降低了搜索深度。
*   **有效抑制冗余**：相比 Search-R1 和 StepSearch，AutoSearch 的过度搜索比例（OSR）大幅下降（例如在 NQ 数据集上降至接近 0%）。
*   **自适应性**：模型学会了在简单问题上快速停止，在复杂问题上深入探索，证明了“最小充分深度”假设的有效性。

### 7. 优点
*   **自适应平衡**：不再依赖人工设定的固定步数或简单的长度惩罚，而是通过模型能力自适应调整。
*   **无需外部监督**：与 HiPRAG 不同，AutoSearch 不需要调用更强的外部 LLM 来提供过程奖励，降低了训练成本。
*   **稳定性高**：引入的基础奖励和格式奖励有效解决了 RL 训练中常见的格式崩溃和不稳定性问题。

### 8. 不足与局限
*   **搜索深度上限**：实验中设置的最大搜索深度相对较低（通常为 4 步），在极大规模或极复杂的长程推理任务中的表现尚待验证。
*   **训练依赖标准答案**：RL 阶段仍需 Ground Truth 来计算 $t_c$ 和奖励，这限制了其在完全无监督或开放域未知问题上的在线学习能力。
*   **领域局限**：目前主要集中在 QA 任务，对于需要复杂工具调用（如代码执行、API 组合）的通用智能体场景覆盖不足。
