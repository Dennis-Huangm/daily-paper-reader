## 摘要
现有的合成工具使用语料库主要针对离线监督微调（SFT）设计，然而强化学习（RL）需要支持奖励可检查在线展开（online rollouts）的可执行环境。我们提出了 COVERT，这是一个两阶段流水线：首先通过具有多级验证的自进化合成生成可靠的基础工具使用轨迹，然后应用保持真值（oracle-preserving）的增强手段，系统性地增加环境复杂度。这些增强引入了干扰工具、间接或模糊的用户查询，以及噪声、多格式或错误的工具输出，同时严格保留真值工具调用和最终答案作为地面真值（ground truth）。这种设计使得标准案例可以通过参考匹配进行自动奖励计算，而对于错误检测等特殊行为，则可以通过轻量级的评判辅助验证，从而支持工具调用策略的 RL 优化。在 Qwen2.5-Instruct-14B 上，COVERT-RL 将 BFCL v3 的整体准确率从 56.5 提升至 59.9，将 ACEBench 的准确率从 53.0 提升至 59.3，且在通用能力基准测试上的性能退化极小；当在 SFT 之上叠加时，准确率进一步达到 62.1 和 61.8，证实了叠加增益。这些结果表明，保持真值的合成环境提供了一个实用的 RL 精炼阶段，与 SFT 互补，能够提高在模糊性和不可靠工具反馈下的工具使用鲁棒性。

## Abstract
Existing synthetic tool-use corpora are primarily designed for offline supervised fine-tuning, yet reinforcement learning (RL) requires executable environments that support reward-checkable online rollouts. We propose COVERT, a two-stage pipeline that first generates reliable base tool-use trajectories through self-evolving synthesis with multi-level validation, and then applies oracle-preserving augmentations that systematically increase environmental complexity. These augmentations introduce distractor tools, indirect or ambiguous user queries, and noisy, multi-format, or erroneous tool outputs, while strictly preserving oracle tool calls and final answers as ground truth. This design enables automatic reward computation via reference matching for standard cases and lightweight judge-assisted verification for special behaviors such as error detection, supporting RL optimization of tool-calling policies. On Qwen2.5-Instruct-14B, COVERT-RL improves overall accuracy on BFCL v3 from 56.5 to 59.9 and on ACEBench from 53.0 to 59.3, with minimal regressions on general-ability benchmarks; when stacked on SFT, it further reaches 62.1 and 61.8, confirming additive gains. These results suggest that oracle-preserving synthetic environments offer a practical RL refinement stage, complementary to SFT, for improving tool-use robustness under ambiguity and unreliable tool feedback.

### 1. 核心问题与整体含义（研究动机和背景）
*   **研究动机**：现有的工具使用（Tool-use）数据集大多是为离线监督微调（SFT）设计的静态语料，缺乏支持强化学习在线探索（Online Rollout）的可执行环境。
*   **核心挑战**：现实中的工具调用日志稀疏且涉及隐私，而合成数据往往过于理想化。模型在面对模糊查询、干扰工具、噪声输出或工具故障等复杂现实场景时，鲁棒性较差。
*   **核心目标**：构建一个能够大规模合成、支持自动奖励计算、且能精确控制环境扰动（如增加干扰项）的强化学习训练流水线。

### 2. 方法论：核心思想与技术细节
COVERT 采用两阶段流水线构建 RL 训练环境：
*   **阶段 I：可靠基础轨迹生成**
    *   **自进化合成**：从少量种子数据开始，利用 LLM 迭代生成新轨迹，并将其作为后续生成的上下文示例。
    *   **多级验证**：通过格式检查（JSON 结构）、规则检查（参数类型、枚举值约束）和语义检查（LLM 评判工具选择是否合理）确保基础数据的绝对可靠。
*   **阶段 II：保真增强（Oracle-preserving Augmentation）**
    *   **核心思想**：在修改环境组件（查询、工具集、输出）的同时，严格保留“真值工具调用（Oracle Call）”或“最终答案”，从而实现自动化的奖励计算。
    *   **增强类型**：
        1.  **精确可验证增强**：引入干扰工具、改写模糊查询、改变输出格式或注入噪声。此类任务通过参考匹配（Reference Matching）直接计算奖励。
        2.  **评判员辅助增强**：模拟工具错误（如连接失败）或无效查询。此类任务需要模型做出特定反应（如报错而非幻觉答案），由轻量级 LLM 评判员根据预设标签进行验证。
*   **奖励函数设计**：
    *   总奖励由**格式奖励**（标签闭合、JSON 合法性）和**正确性奖励**（参数匹配或评判员打分）加权组成。
    *   采用 **GRPO**（Group Relative Policy Optimization）算法进行策略优化，无需额外的价值模型。

### 5. 实验数量与充分性
*   **实验规模**：在两个不同参数规模（7B, 14B）的模型上进行了验证，涵盖了从单轮到多轮、从简单到复杂的多种工具调用场景。
*   **充分性**：实验不仅关注工具调用准确率，还评估了 8 个通用能力指标，以检测是否存在“灾难性遗忘”。

### 6. 主要结论与发现
*   **性能提升**：COVERT-RL 显著提升了模型在复杂场景下的表现。在 14B 模型上，BFCL v3 准确率提升 3.4%，ACEBench 提升 6.3%。
*   **SFT 与 RL 的互补性**：在 SFT 模型基础上叠加 RL 效果最好（14B 达到 62.1%），证明 SFT 提供了结构化先验，而 RL 增强了语义正确性和对异常情况的鲁棒性。

*   **特定增益**：提升最明显的是“无关查询检测”和“异常输出处理”等鲁棒性指标。

### 7. 优点
*   **可控性**：通过保真增强，研究者可以像调节旋钮一样控制环境的复杂度（如干扰项的数量）。
*   **可验证性**：巧妙地解决了合成数据在 RL 阶段的奖励计算难题，降低了对昂贵人类反馈或重型评判模型的依赖。
*   **实用性**：证明了即使不使用真实 API 环境，高质量的合成模拟环境也能有效精炼智能体策略。
