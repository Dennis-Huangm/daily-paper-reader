## 摘要
强化学习是真正扩展了 LLM 智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究给出了后者的答案：基础模型与强化学习模型的 pass@k 曲线在大 k 值下趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，因为在该场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种同时改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习真正扩大了能力边界：强化学习智能体的通过率曲线高于基础模型，且差距在大 k 值下不断扩大而非收敛。这种扩展专门针对组合式、序列化的信息收集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，有监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新分配了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观与悲观解读：针对不同的任务类型，两者都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 核心问题与研究动机
*   **研究动机**：近期关于静态数学推理的研究（如 RLVR）认为，RL 并没有扩展模型的能力边界，而只是通过重新分配概率质量提高了效率（即在大规模采样 $k$ 时，RL 模型与基础模型的表现趋于一致）。
*   **核心问题**：这种“悲观”的结论是否适用于需要多轮环境交互（如工具使用）的智能体场景？在智能体任务中，交互深度 $T$ 允许产生复杂的组合策略，这可能是单纯增加采样次数 $k$ 无法弥补的。

### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：提出一个二维评估指标 **PASS@(k, T)**，将智能体的能力分解为两个维度：
    *   **采样预算 $k$**：独立尝试的次数，用于衡量“采样效率”。
    *   **交互深度 $T$**：每轮尝试中与环境交互的最大回合数，用于衡量“交互能力”。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：在无限采样（大 $k$）和固定深度 $T$ 下模型能解决的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 模型解决了基础模型在任何采样预算下都无法解决的新问题。
    *   **效率提升 (Efficiency Improvement)**：模型解决已知问题的成功率提高，但在大 $k$ 时两者表现收敛。
*   **算法实现**：使用无偏超几何估计器计算在 $n$ 次实验中至少成功一次的概率。

### 3. 实验设计
*   **任务分类**：
    *   **Category A (纯推理)**：MATH-500，不使用工具（对照组）。
    *   **Category B (独立检索)**：HotPotQA 比较类问题，需两次互不干扰的检索。
    *   **Category C (序列检索)**：HotPotQA 桥接类问题，第二次检索依赖第一次的结果（最具挑战性）。
*   **对比方法**：
    *   **Base**：Qwen2.5-7B-Instruct。
    *   **SFT**：在 200 条专家轨迹上进行有监督微调。
    *   **RL (GRPO)**：在相同的 200 个问题上使用强化学习（基于结果奖励）。
*   **环境**：使用 BM25 检索工具的 ReAct 循环。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的 Category C 任务中，RL 模型的通过率曲线在 $k$ 增大时与基础模型的差距**持续扩大**，而非收敛。这直接反驳了静态推理中的悲观结论。
*   **SFT 的负面效应**：在相同的组合任务上，SFT 反而导致了能力边界的**退化**（丢失了基础模型原本能解决的问题），证明了自主探索（Exploration）是 RL 成功的关键。

*   **机制揭示**：RL 并没有发明全新的搜索查询，而是优化了**如何整合检索到的信息**。它重新分配了权重，使模型更倾向于选择那些能引导正确推理的现有策略。

### 7. 优点与亮点
*   **维度创新**：首次提出二维指标 $PASS@(k, T)$，清晰地分离了“找得准”（效率）和“做得了”（能力）。

*   **调和矛盾**：解释了为什么之前的研究认为 RL 没用（任务太简单或缺乏交互），为 RL 在智能体领域的价值提供了有力证据。

*   **检索局限**：使用的是静态语料库的 BM25 检索，而非真实的互联网实时检索。
