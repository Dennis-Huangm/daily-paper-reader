## 摘要
人类记忆通过选择性遗忘进行适应：经验随时间推移变得难以获取，但可以通过强化或上下文线索重新激活。相比之下，记忆增强的大语言模型（LLM）智能体依赖“始终开启”的检索和“扁平化”的记忆存储，随着历史记录的增长，会导致严重的干扰和延迟。我们提出了 Oblivion，这是一个记忆控制框架，它将遗忘视为衰减驱动的可访问性降低，而非显式的删除。Oblivion 将记忆控制解耦为读路径和写路径。读路径根据智能体的不确定性和记忆缓冲区的充足性来决定何时查阅记忆，从而避免冗余的始终开启式访问。写路径通过强化对生成响应有贡献的记忆，来决定需要加强的内容。结合这两者，可以实现层级化的记忆组织，在保持持久的高层策略的同时，根据需要动态加载细节。我们在静态和动态的长程交互基准测试上进行了评估。结果表明，Oblivion 能够动态调整记忆访问和强化，在不断变化的上下文中平衡学习与遗忘，凸显了记忆控制对于有效的 LLM 智能体推理至关重要。源代码可在 https://github.com/nec-research/oblivion 获取。

## Abstract
Human memory adapts through selective forgetting: experiences become less accessible over time but can be reactivated by reinforcement or contextual cues. In contrast, memory-augmented LLM agents rely on "always-on" retrieval and "flat" memory storage, causing high interference and latency as histories grow. We introduce Oblivion, a memory control framework that casts forgetting as decay-driven reductions in accessibility, not explicit deletion. Oblivion decouples memory control into read and write paths. The read path decides when to consult memory, based on agent uncertainty and memory buffer sufficiency, avoiding redundant always-on access. The write path decides what to strengthen, by reinforcing memories contributing to forming the response. Together, this enables hierarchical memory organization that maintains persistent high-level strategies while dynamically loading details as needed. We evaluate on both static and dynamic long-horizon interaction benchmarks. Results show that Oblivion dynamically adapts memory access and reinforcement, balancing learning and forgetting under shifting contexts, highlighting that memory control is essential for effective LLM-agentic reasoning. The source code is available at https://github.com/nec-research/oblivion.

#### 1. 核心问题与整体含义（研究动机和背景）
现有的记忆增强型大语言模型（LLM）智能体在处理长程交互时面临三大挑战：
*   **检索冗余（Always-on Retrieval）**：无论是否需要，智能体在每轮对话都会尝试检索，导致高延迟和计算浪费。
*   **记忆扁平化（Flat Memory）**：无法区分持久的高层策略与瞬时的底层细节，通常采用简单的硬删除（Hard Deletion）来管理上下文限制。
*   **干扰严重（Interference）**：随着记忆库增长，无关或过时的信息会干扰推理（即“大海捞针”问题）。

**研究动机**：借鉴人类心理学中的“选择性遗忘”机制——遗忘并非物理删除，而是可访问性的降低。论文旨在将记忆管理从简单的数据库检索转变为一个**自适应的控制问题**。

#### 2. 方法论：核心思想与关键技术
Oblivion 框架的核心是**读/写路径解耦（Read/Write Decoupling）**，由中央执行器（Executor）协调：

*   **层级记忆表示**：
    *   **L1（动态程序记忆）**：作为元索引，包含类别摘要、访问频率统计和处理特定领域查询的元指令。
    *   **L2（语义记忆）**：存储带时间戳的事实陈述，解决“世界漂移”导致的事实冲突。
    *   **L3（抢占式情节记忆）**：将原始对话转化为具有行动导向的总结（例如：从“我喜欢健身”转化为“记住用户喜欢在假日健身，据此规划日程”）。
*   **读路径（不确定性门控检索）**：
    *   **Decayer（衰减器）**：利用 **Ebbinghaus 遗忘曲线公式** $R_t(c) = \exp(-n_t(c)/S_t(c))$ 计算保留分数。只有当智能体对当前缓冲区信息感到“不确定”（通过 LLM 评判和嵌入相似度衡量）时，才触发检索。
    *   **Activator（激活器）**：通过 **DAG（有向无环图）查询扩展**，将复杂问题分解为多个子查询，从持久存储中加载 L2/L3 记忆到缓冲区。
*   **写路径（反馈驱动强化）**：
    *   **Recognizer（识别器）**：识别哪些记忆条目对生成当前响应有实际贡献。
    *   **Memory Manager（记忆管理器）**：仅强化被使用的记忆（增加其 Utility 分数），未被使用的记忆随时间自然衰减，优先级降低。
