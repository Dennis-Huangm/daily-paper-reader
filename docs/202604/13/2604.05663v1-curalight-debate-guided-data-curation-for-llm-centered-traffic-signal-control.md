## 摘要
交通信号控制 (TSC) 是智能交通系统 (ITS) 的核心组成部分，旨在减少拥堵、排放和旅行时间。近期基于强化学习 (RL) 和大语言模型 (LLM) 的方法提高了适应性，但仍面临可解释性有限、交互数据不足以及对异构交叉口泛化能力弱的问题。本文提出了 CuraLight，这是一个以 LLM 为核心的框架，其中 RL 智能体辅助基于 LLM 的交通信号控制器的微调。RL 智能体探索交通环境并生成高质量的交互轨迹，这些轨迹被转换为提示-响应对，用于模仿微调。一个多 LLM 集成审议系统通过结构化辩论进一步评估候选信号配时动作，为训练提供偏好感知的监督信号。在 SUMO 中针对济南、杭州和亦庄的异构真实路网进行的实验表明，CuraLight 始终优于最先进的基准方法，使平均旅行时间减少了 5.34%，平均排队长度减少了 5.14%，平均等待时间减少了 7.02%。结果突显了将 RL 辅助探索与基于审议的数据策展相结合，对于实现可扩展且可解释的交通信号控制的有效性。

## Abstract
Traffic signal control (TSC) is a core component of intelligent transportation systems (ITS), aiming to reduce congestion, emissions, and travel time. Recent approaches based on reinforcement learning (RL) and large language models (LLMs) have improved adaptivity, but still suffer from limited interpretability, insufficient interaction data, and weak generalization to heterogeneous intersections.   This paper proposes CuraLight, an LLM-centered framework where an RL agent assists the fine-tuning of an LLM-based traffic signal controller. The RL agent explores traffic environments and generates high-quality interaction trajectories, which are converted into prompt-response pairs for imitation fine-tuning. A multi-LLM ensemble deliberation system further evaluates candidate signal timing actions through structured debate, providing preference-aware supervision signals for training.   Experiments conducted in SUMO across heterogeneous real-world networks from Jinan, Hangzhou, and Yizhuang demonstrate that CuraLight consistently outperforms state-of-the-art baselines, reducing average travel time by 5.34 percent, average queue length by 5.14 percent, and average waiting time by 7.02 percent. The results highlight the effectiveness of combining RL-assisted exploration with deliberation-based data curation for scalable and interpretable traffic signal control.

#### 1. 核心问题与研究动机
*   **核心问题**：传统的交通信号控制（TSC）方法中，规则和强化学习（RL）方法缺乏决策的可解释性；而现有的 LLM 方法在处理异构交叉口（不同几何形状和车道配置）的高维配时决策时面临挑战，且由于缺乏任务对齐的交互数据，导致其在需求波动下的鲁棒性较差。
*   **研究背景**：LLM 具有强大的推理和泛化能力，但直接用于 TSC 时，往往受限于环境交互数据不足。如何利用 RL 的探索能力来增强 LLM 的决策质量，并提供高质量的监督信号，是本文的研究重点。

#### 2. 方法论
CuraLight 框架结合了 RL 的探索能力和 LLM 的推理能力，核心流程如下：
*   **RL 辅助探索与数据生成**：
    *   使用基于**扩散模型（Diffusion-based）**的 RL 智能体在异构路网中进行探索。
    *   RL 智能体通过“压力选择器”确定相位，并利用扩散策略生成绿灯时长建议。
    *   收集 RL 产生的交互轨迹，将其转化为“结构化提示词-推理决策”对，用于 LLM 的初步模仿微调（LoRA）。
*   **多 LLM 集成辩论系统（Multi-LLM Ensemble Deliberation）**：
    *   **RL 预过滤**：利用 RL 的 Critic 网络剔除明显劣质的动作。
    *   **辩论机制**：引入多个“防御者 LLM”（Defender LLMs）为不同的候选动作辩论，阐述其在排队消散、流量平衡等方面的优势。
    *   **共识决策**：由一个“共识 LLM”（Consensus LLM）汇总辩论意见，给出最终的候选动作排名和偏好分数。
*   **优先级感知微调（Priority-Aware Fine-tuning）**：
    *   结合 RL Critic 的评分 $q^{RL}$ 和 LLM 辩论的偏好分 $r^{LLM}$，计算统一优先级分数 $s_i(a)$。
    *   在第二阶段微调中，利用该分数对数据进行加权，通过交叉熵损失（针对高质量动作）和非似然损失（针对低质量动作）进一步优化模型。

#### 7. 优点与亮点
*   **数据策展新范式**：通过 RL 探索+LLM 辩论的方式解决了 LLM 在特定任务中高质量数据稀缺的问题。
*   **可解释性**：通过 LLM 的辩论和推理过程，为每一个信号配时动作提供了文本化的逻辑支撑。
*   **异构适应性**：能够处理现实中复杂的、非标准化的交叉口相位和车道配置。

*   **推理延迟**：虽然论文提到了吞吐量，但多 LLM 辩论系统在实时控制（Real-time control）中可能会带来较高的计算开销和延迟。
