## 摘要
大语言模型（LLM）智能体在复杂的交互式决策任务中展现出了强大的能力。然而，现有的 LLM 智能体通常依赖于日益增长的交互历史，导致了高昂的计算成本和有限的可扩展性。在本文中，我们提出了 STEP-HRL，这是一种分层强化学习（HRL）框架，它通过仅以单步转移（而非完整的交互历史）为条件，实现了步级学习。STEP-HRL 对任务进行分层构建，利用已完成的子任务来表示整个任务的全局进度。通过引入局部进度模块，它还迭代且有选择性地总结每个子任务内的交互历史，从而生成局部进度的紧凑摘要。这些组件共同为高层和底层策略生成了增强的步级转移。在 ScienceWorld 和 ALFWorld 基准测试上的实验结果一致表明，STEP-HRL 在性能和泛化能力方面显著优于基准模型，同时减少了 token 的使用量。我们的代码可在 https://github.com/TonyStark042/STEP-HRL 获取。

## Abstract
Large language model (LLM) agents have demonstrated strong capabilities in complex interactive decision-making tasks. However, existing LLM agents typically rely on increasingly long interaction histories, resulting in high computational cost and limited scalability. In this paper, we propose STEP-HRL, a hierarchical reinforcement learning (HRL) framework that enables step-level learning by conditioning only on single-step transitions rather than full interaction histories. STEP-HRL structures tasks hierarchically, using completed subtasks to represent global progress of overall task. By introducing a local progress module, it also iteratively and selectively summarizes interaction history within each subtask to produce a compact summary of local progress. Together, these components yield augmented step-level transitions for both high-level and low-level policies. Experimental results on ScienceWorld and ALFWorld benchmarks consistently demonstrate that STEP-HRL substantially outperforms baselines in terms of performance and generalization while reducing token usage. Our code is available at https://github.com/TonyStark042/STEP-HRL.

#### 1. 核心问题与整体含义（研究动机和背景）
现有的 LLM 智能体在处理复杂决策任务时，通常采用“历史条件化”（History-conditioned）的建模方式，即策略的输入依赖于不断增长的交互历史。这种设计存在三大弊端：
*   **计算成本高昂**：Transformer 的注意力机制随上下文长度呈平方级增长。
*   **决策干扰**：冗长的历史包含大量冗余或无关信息，掩盖了关键的决策信号。
*   **扩展性受限**：在长程任务中，上下文窗口限制了智能体的持续作业能力。
**研究动机**：将决策过程从“依赖全量历史”转变为“依赖当前状态摘要”，通过分层强化学习实现高效的步级（Step-level）决策。

#### 2. 方法论：核心思想与关键技术
STEP-HRL 的核心思想是**通过分层抽象和局部进度建模，将长程历史压缩为紧凑的步级状态转移**。

*   **分层结构**：
    *   **高层策略 ($\pi_h$)**：负责全局规划，根据任务目标和已完成子任务序列生成下一个子任务 ($g_k$)。
    *   **底层策略 ($\pi_l$)**：负责具体执行，根据当前子任务、当前观察和局部进度生成原子动作 ($a_t^k$)。
*   **局部进度模块 ($\pi_p$)**：这是本文的关键创新。它是一个迭代更新的策略，每一步都会结合前一时刻的进度摘要、上一步动作和当前观察，生成一个新的、字数受控的**局部进度摘要**。这使得底层策略只需关注当前步的增强信息，而无需回溯整个子任务的历史。
*   **训练流程**：
    1.  **行为克隆 (BC) 初始化**：利用专家演示数据（通过 DeepSeek 标注子任务和进度）对高层、底层和进度策略进行联合微调。

#### 7. 优点
*   **状态压缩巧妙**：通过“局部进度”解决了 LLM Agent 长期存在的“上下文爆炸”问题。
*   **训练高效**：采用参数共享设计，高层、底层和进度策略共享同一个 LLM 主干，仅 Critic 网络独立，减少了显存占用并促进了知识迁移。
*   **泛化能力强**：步级转移的设计减少了对特定轨迹顺序的依赖，使模型在未见场景下更稳健。

#### 8. 不足与局限
*   **专家数据依赖**：构建步级训练数据需要高质量的子任务分解和进度描述，这通常依赖更强大的模型（如 DeepSeek/GPT-4）进行预标注，在缺乏专家数据的领域难以应用。
*   **子任务终止风险**：底层策略在生成动作的同时预测子任务是否结束，若预测提前或滞后，会影响高层策略的衔接，导致决策链条断裂。
*   **复杂环境适应性**：对于子任务边界模糊或进度难以用文字量化的极复杂环境，局部进度模块的描述质量可能成为瓶颈。
