## 摘要
尽管由可验证奖励强化学习（RLVR）驱动的大型推理模型（LRMs）取得了显著进展，但在专业或新颖领域，这种监督信号极其昂贵或难以获取，这使得该范式受到根本性限制，并为测试时自适应带来了关键挑战。虽然现有的测试时方法提供了一种潜在的解决方案，但它们受限于从静态查询集中学习，存在过拟合文本模式的风险。为填补这一空白，我们引入了测试时变分合成（TTVS），这是一个新颖的框架，通过从无标签测试查询中动态增强训练流，使 LRMs 能够实现自我进化。TTVS 包含两个协同模块：（1）在线变分合成，它将静态测试查询转换为多样化且语义等价的变体动态流，迫使模型学习底层的问题逻辑而非表层模式；（2）测试时混合探索，它在合成变体之间平衡了准确性驱动的利用与一致性驱动的探索。大量实验表明，TTVS 在八种模型架构上均表现出优异的性能。值得注意的是，仅使用无标签的测试时数据，TTVS 不仅超越了其他测试时自适应方法，还优于在海量高质量标注数据上训练的最先进的有监督强化学习技术。

## Abstract
Despite significant advances in Large Reasoning Models (LRMs) driven by reinforcement learning with verifiable rewards (RLVR), this paradigm is fundamentally limited in specialized or novel domains where such supervision is prohibitively expensive or unavailable, posing a key challenge for test-time adaptation. While existing test-time methods offer a potential solution, they are constrained by learning from static query sets, risking overfitting to textual patterns. To address this gap, we introduce Test-Time Variational Synthesis (TTVS), a novel framework that enables LRMs to self-evolve by dynamically augmenting the training stream from unlabeled test queries. TTVS comprises two synergistic modules: (1) Online Variational Synthesis, which transforms static test queries into a dynamic stream of diverse, semantically-equivalent variations, enforcing the model to learn underlying problem logic rather than superficial patterns; (2) Test-time Hybrid Exploration, which balances accuracy-driven exploitation with consistency-driven exploration across synthetic variants. Extensive experiments show TTVS yields superior performance across eight model architectures. Notably, using only unlabeled test-time data, TTVS not only surpasses other test-time adaptation methods but also outperforms state-of-the-art supervised RL-based techniques trained on vast, high-quality labeled data.

论文关注的是**大型推理模型（LRMs）在缺乏标注数据或可验证奖励（Verifiable Rewards）的专业领域（如医疗、航天）如何持续进化**的问题。
*   **研究动机**：目前的强化学习（RL）高度依赖大规模高质量标注数据。虽然“测试时训练”（Test-time Training）提供了一种无需标签的自适应方案，但现有方法通常在**静态**的测试查询集上运行，这导致模型容易过拟合特定的文本模式，而非掌握底层逻辑。
*   **核心问题**：能否在测试阶段动态地扩展训练数据，使模型通过自我探索（Self-Exploring）学习到更具泛化性的推理能力？

### 2. 方法论：核心思想与技术细节
TTVS 框架由两个协同模块组成，旨在将静态测试任务转化为动态的、逻辑驱动的训练流：

*   **在线变分合成（Online Variational Synthesis）**：
    1.  **伪标签生成**：对原始查询 $q$ 进行多次采样（Rollouts），通过“多数投票”（Majority Voting）确定共识答案作为伪标签 $y^*$。
    2.  **变分增强**：利用模型自身能力，生成 $k$ 个与原问题语义等价但表述不同的新查询 $\{q'_1, ..., q'_k\}$。
    3.  **在线过滤**：设置难度阈值 $[\tau_{low}, \tau_{high}]$。只有当模型对原题的初始准确率处于“适中”范围（既非太简单也非太难）且生成文本长度合规时，才将其纳入训练。
*   **测试时混合探索（Test-time Hybrid Exploration）**：
    1.  **组内探索（IGE）**：针对每个独立查询（原题或变体）进行 GRPO 优化，目标是最大化特定表述下的准确性（准确性驱动）。
    2.  **跨组探索（CGE）**：将原题及其所有变体的采样结果混合，进行统一的多数投票生成联合伪标签。这迫使模型在不同表述下保持逻辑一致性（一致性驱动）。

### 3. 实验设计
*   **数据集**：涵盖了数学推理和科学问答的硬核基准，包括 **MATH-500**、**AIME-2024**、**AMC-2023** 和 **GPQA**。此外还扩展到了 DeepMath-103K（防污染测试）、MedMCQA（医疗领域）和 OpenBookQA（常识推理）。

### 6. 主要结论与发现
*   **性能飞跃**：TTVS 在所有架构上均显著优于基准模型。例如在 Qwen3-4B 上，平均分提升了约 30%。

*   **效率**：TTVS 生成的答案更简洁（Token 长度更短），且在相同采样预算下比增加采样次数的基线方法更有效。

### 7. 优点（亮点）
*   **打破静态限制**：通过“变分合成”解决了测试时自适应容易过拟合表面文本的问题。
*   **双重探索机制**：IGE 保证了“解得出”，CGE 保证了“解得稳”，逻辑闭环完整。
*   **普适性强**：不挑模型（Base 或 Instruct 均可）、不挑 RL 算法，具有很强的工程落地价值。
