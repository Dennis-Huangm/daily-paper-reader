## 摘要
智能体技能是程序性知识和可执行资源的结构化包，智能体在推理时动态加载这些技能，这已成为增强大语言模型（LLM）智能体的一种可靠机制。然而，推理时的技能增强存在根本性的局限：检索噪声会引入无关的引导，注入的技能内容会带来巨大的 Token 开销，且模型从未真正习得其仅仅是在遵循的知识。我们探究技能是否可以被内化到模型参数中，从而在无需任何运行时技能检索的情况下实现零样本自主行为。我们提出了 SKILL0，这是一个专为技能内化设计的上下文内强化学习框架。SKILL0 引入了一种训练阶段的课程学习方案，从完整的技能上下文开始并逐步撤出。技能在离线状态下按类别分组，并与交互历史一起渲染成紧凑的视觉上下文，教导模型进行工具调用和多轮任务完成。随后，动态课程会评估每个技能文件的同策略辅助作用，在随线性衰减的预算内，仅保留当前策略仍能从中获益的技能，直到智能体在完全零样本的设置下运行。广泛的智能体实验表明，SKILL0 相比标准强化学习基线取得了显著提升（ALFWorld 提升 9.7%，Search-QA 提升 6.6%），同时保持了每步少于 0.5k Token 的高效上下文。我们的代码可在 https://github.com/ZJU-REAL/SkillZero 获取。

## Abstract
Agent skills, structured packages of procedural knowledge and executable resources that agents dynamically load at inference time, have become a reliable mechanism for augmenting LLM agents. Yet inference-time skill augmentation is fundamentally limited: retrieval noise introduces irrelevant guidance, injected skill content imposes substantial token overhead, and the model never truly acquires the knowledge it merely follows. We ask whether skills can instead be internalized into model parameters, enabling zero-shot autonomous behavior without any runtime skill retrieval. We introduce SKILL0, an in-context reinforcement learning framework designed for skill internalization. SKILL0 introduces a training-time curriculum that begins with full skill context and progressively withdraws it. Skills are grouped offline by category and rendered with interaction history into a compact visual context, teaching he model tool invocation and multi-turn task completion. A Dynamic Curriculum then evaluates each skill file's on-policy helpfulness, retaining only those from which the current policy still benefits within a linearly decaying budget, until the agent operates in a fully zero-shot setting. Extensive agentic experiments demonstrate that SKILL0 achieves substantial improvements over the standard RL baseline (+9.7\% for ALFWorld and +6.6\% for Search-QA), while maintaining a highly efficient context of fewer than 0.5k tokens per step. Our code is available at https://github.com/ZJU-REAL/SkillZero.

### 1. 核心问题与整体含义（研究动机和背景）
目前增强大语言模型（LLM）智能体的主流方式是**推理时技能增强**（Inference-time Skill Augmentation），即从技能库中检索相关技能并注入 Prompt。
*   **局限性**：检索噪声会引入干扰信息；注入技能内容会导致 Token 开销随对话轮数剧增；最核心的是，模型只是在“遵循”指令而非真正“掌握”知识，能力停留在上下文层面而非参数层面。
*   **核心问题**：能否通过训练让智能体将外部技能“内化”到模型参数中，从而在推理时不依赖检索也能实现零样本（Zero-shot）的自主行为？

### 2. 方法论
SKILL0 提出了**上下文内强化学习（In-Context Reinforcement Learning, ICRL）**框架，旨在实现从“依赖上下文”到“自主执行”的转变。
*   **核心思想**：在训练阶段提供技能引导作为“脚手架”，通过动态课程学习逐步撤走这些引导，强制模型将规程性知识转移到参数中。
*   **关键技术**：
    *   **视觉上下文渲染（Context Rendering）**：将冗长的交互历史和技能描述渲染成紧凑的 RGB 图像，通过视觉编码器压缩信息，大幅降低 Token 消耗。
    *   **动态课程学习（Dynamic Curriculum）**：
        1.  **离线分组**：按类别将技能组织成 Markdown 文件，并关联特定的验证子任务。
        2.  **同策略辅助性评估（On-policy Helpfulness）**：定期对比模型在“有技能引导”和“无技能引导”下的表现差异（$\Delta k$）。
        3.  **线性预算衰减**：设定一个随训练步数线性减少的技能数量预算 $M(s)$，优先保留那些对当前策略提升最大的技能，直到预算归零。
    *   **复合奖励函数**：同时优化任务成功率和图像压缩效率。

### 3. 实验设计
*   **数据集/场景**：
    *   **ALFWorld**：具身智能文本游戏，包含 6 类家庭任务（如存放物品、清洁、加热等）。
    *   **Search-based QA**：搜索增强问答，涵盖单跳（NQ, TriviaQA 等）和多跳（HotpotQA, Bamboogle 等）共 7 个数据集。
*   **Benchmark 与对比方法**：
    *   **基线**：Zero-shot/Few-shot Prompting。
    *   **强化学习方法**：GRPO、AgentOCR、EvolveR。
    *   **技能/存储增强方法**：SkillRL（推理时增强）、ExpeL、Mem0、MemRL 等。
    *   **闭源模型**：GPT-4o、Gemini-2.5-Pro。

*   **硬件**：使用了 **4 台 H800 GPU**。
*   **模型**：基于 Qwen2.5-VL 系列（3B 和 7B 版本）。
*   **训练细节**：训练步数最多为 180 步。ALFWorld 每批次采样 16 个任务，Search-QA 每批次采样 128 个任务。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了具身导航和复杂问答两个截然不同的领域，验证了框架的通用性。
*   **消融实验**：针对技能预算策略（如固定全量 vs 线性衰减）、动态课程的三个步骤（过滤、排序、选择）以及验证间隔 $d$ 进行了详细对比。
*   **客观性**：实验不仅对比了成功率，还详细统计了推理时的 Token 成本，并展示了训练过程中“辅助性”指标的变化曲线，逻辑闭环较强。

### 6. 主要结论与发现
*   **性能提升**：SKILL0 (3B) 在 ALFWorld 上比 AgentOCR 提升了 **9.7%**，在 Search-QA 上提升了 **6.6%**。
*   **Token 高效性**：推理时每步消耗的 Token 少于 **0.5k**，比 SkillRL 等推理增强方法降低了 5 倍以上。
*   **内化现象**：实验观察到“上升-下降”的辅助性曲线——早期模型学习利用技能，后期随着技能撤走，模型在无技能情况下的表现逐渐追平甚至超过有技能的情况，证明了知识已成功内化。

### 7. 优点
*   **范式创新**：首次将技能内化定义为明确的强化学习训练目标，解决了智能体对外部 Prompt 的长期依赖问题。
*   **极高的推理效率**：通过视觉压缩和去检索化，极大地优化了长程任务的推理成本。
*   **自适应性**：动态课程不是死板的时间表，而是基于模型当前学习状态（Helpfulness）的反馈调节。

### 8. 不足与局限
*   **初始化依赖**：SKILL0 的效果高度依赖于初始技能库（SkillBank）的质量。
*   **离线成本**：技能的分组和验证子任务的划分需要一定的人工或离线预处理工作。
*   **领域迁移**：当应用到全新领域时，可能需要重新构建技能关联逻辑，自动化程度仍有提升空间。
