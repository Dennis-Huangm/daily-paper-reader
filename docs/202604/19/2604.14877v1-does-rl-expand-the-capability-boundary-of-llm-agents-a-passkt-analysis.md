## 摘要
强化学习是真正扩展了大语言模型（LLM）智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究给出了后者的答案：基础模型与强化学习模型的 pass@k 曲线在大 k 值下趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，因为在这种场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种同时改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习真正扩大了能力边界：强化学习智能体的通过率曲线高于基础模型，且差距在大 k 值下逐渐扩大而非收敛。这种扩展专门针对组合式、序列化的信息收集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，有监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新分配了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观与悲观解读：针对不同的任务类型，两者都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

### 1. 论文的核心问题与整体含义
该研究探讨了一个基础性问题：**强化学习（RL）是真正扩展了 LLM 智能体的“能力边界”（即让模型学会以前完全不会的东西），还是仅仅提高了其“效率”（即让模型更稳定地完成本就能偶尔完成的任务）？**

在静态数学推理领域，近期研究（如 Yue et al., 2025）持悲观态度，认为 RL 只是重新分配了概率质量。本论文则将视角转向**智能体工具使用（Agentic Tool Use）**场景，认为在多轮交互中，RL 可能会通过组合策略实现重采样无法达到的能力突破。

### 2. 论文提出的方法论：PASS@(k, T) 框架
论文认为智能体的能力是二维的，并提出了 **PASS@(k, T)** 指标：
*   **核心思想**：将评估维度拆解为采样预算 $k$（独立尝试次数）和交互深度 $T$（每轮尝试中与环境交互的最大轮数）。
*   **关键细节**：
    *   **$k$ 轴**：探测采样效率。如果 RL 仅提升效率，那么在大 $k$ 值下，RL 模型与基础模型的表现应趋于收敛。
    *   **$T$ 轴**：探测交互能力。衡量更深层次的环境参与是否能解锁单次检索无法解决的复杂问题。
    *   **公式**：采用无偏超几何估计器计算，即在 $n$ 次独立尝试中，若有 $c_T$ 次成功，则 $PASS@(k, T) = 1 - \frac{\binom{n-c_T}{k}}{\binom{n}{k}}$。
*   **定义区分**：
    *   **能力边界 (Capability Boundary)**：在无限重采样下模型能解决的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 解决了基础模型在任何采样预算下都无法解决的新问题。
    *   **效率提升 (Efficiency Improvement)**：模型解决已知问题的概率增加。

### 3. 实验设计
*   **模型对比**：以 **Qwen2.5-7B-Instruct** 为基础模型，对比了三种变体：
    1.  **Base**：原始指令微调模型。
    2.  **SFT**：在 200 条专家轨迹上进行有监督微调。
    3.  **RL (GRPO)**：在相同的 200 个问题上使用 GRPO 算法进行强化学习，仅提供二元奖励（正确/错误）。
*   **任务分类**：
    *   **Category A (MATH-500)**：纯数学推理（无工具），作为负对照组。
    *   **Category B (HotPotQA Comparison)**：简单对比任务（两次独立检索）。
    *   **Category C (HotPotQA Bridge)**：复杂组合任务（两次序列化检索，第二步依赖第一步的结果）。
*   **环境**：使用 BM25 检索器对 HotPotQA 语料库进行检索。

### 4. 资源与算力
*   **硬件**：使用了单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：

*   **参数量**：使用了 LoRA 微调，可训练参数约为 40M。

### 5. 实验数量与充分性
*   **实验规模**：针对 3 个类别、4 种模型、多个 $k$ 值（1 到 64）和 $T$ 值（0 到 5）进行了详尽的网格评估。
*   **充分性**：
    *   每组实验进行了 $n=64$ 次独立采样以确保统计显著性。
    *   进行了**机制分析**（困惑度分解、策略多样性测量、跨策略交换实验）。
    *   包含**消融实验**（如探索奖励 Rexplore 的影响）。
*   **客观性**：通过“匹配数据”设计（SFT 和 RL 使用完全相同的 200 个问题），排除了数据暴露带来的偏差，确保了对比的公平性。

### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的 Category C（组合任务）中，RL 模型的通过率曲线在大 $k$ 时显著高于 Base 模型，且差距随 $k$ 增大而扩大，这直接反驳了静态推理中的“收敛论”。
*   **SFT 的负面效应**：在相同数据下，SFT 反而导致了能力边界的退化（在 Cat C 中净损失 4 个问题），表明专家模仿可能抑制了模型处理复杂组合路径的灵活性。
*   **任务依赖性**：在简单任务（Cat A/B）上，RL 的表现符合预期（无影响或仅微量提升），证明 RL 的优势在于处理需要“自主探索”的组合性策略。
*   **机制揭示**：RL 并非学会了全新的搜索词，而是通过**重加权（Reweighting）**优化了如何整合检索到的信息。RL 保持了策略多样性，而 SFT 导致了策略坍缩。

### 7. 优点
*   **评估维度创新**：PASS@(k, T) 指标为智能体评估提供了比单一准确率更深刻的洞察，成功解耦了效率与能力。
*   **因果分析严谨**：通过匹配数据的 SFT vs RL 对比，清晰地证明了“自主探索”而非“数据暴露”是能力扩展的关键。
*   **调和矛盾**：解释了为什么 RL 在数学推理上显得平庸，但在智能体任务上表现卓越——关键在于环境是否允许通过组合策略产生新价值。

### 8. 不足与局限
*   **模型规模有限**：主要基于 7B 模型，尚未在大规模模型（如 70B+）上验证该结论的普适性。
*   **任务多样性**：虽然使用了 HotPotQA，但工具仅限于简单的检索，未涉及更复杂的 API 调用或多工具协作场景。
*   **训练规模较小**：仅使用了 200 个问题的训练集，虽然这有助于证明 RL 的探索效率，但也限制了对大规模强化学习行为的观察。
*   **检索器单一**：仅使用了 BM25，未测试向量检索或其他神经检索器对结果的影响。
