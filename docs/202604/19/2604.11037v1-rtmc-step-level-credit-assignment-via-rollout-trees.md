## 摘要
多步智能体强化学习受益于细粒度的信用分配，但现有方法提供的选择有限：像 GRPO 这样的无评论家（critic-free）方法为轨迹中的每个动作分配统一的优势值，而学习到的价值网络则会引入显著的开销，且在稀疏奖励下可能较为脆弱。我们观察到，针对同一问题的成组展开（group rollouts）通常会经过重叠的中间状态，隐式地形成一棵树，其分支在连续的决策点处分叉。基于这一洞察，我们引入了展开树蒙特卡洛（Rollout-Tree Monte Carlo, RTMC）优势估计，它通过聚合共享相同状态的展开之间的回报统计数据，来生成每步的 Q 值和优势值，而无需任何学习到的评论家。一种状态-动作签名系统将原始交互历史压缩为紧凑且可比较的表示，使得跨展开的状态匹配变得可行。在 SWE-bench Verified 基准测试中，RTMC 相比 GRPO 将 pass@1 提高了 3.2 个百分点。

## Abstract
Multi-step agentic reinforcement learning benefits from fine-grained credit assignment, yet existing approaches offer limited options: critic-free methods like GRPO assign a uniform advantage to every action in a trajectory, while learned value networks introduce notable overhead and can be fragile under sparse rewards. We observe that group rollouts targeting the same problem often traverse overlapping intermediate states, implicitly forming a tree whose branches diverge at successive decision points. Building on this insight, we introduce Rollout-Tree Monte Carlo (RTMC) advantage estimation, which aggregates return statistics across rollouts sharing a common state to produce per-step Q-values and advantages--without any learned critic. A state-action signature system compresses raw interaction histories into compact, comparable representations, making cross-rollout state matching tractable. On SWE-bench Verified, RTMC improves pass@1 by 3.2 percentage points over GRPO.

### 1. 核心问题与研究动机
*   **核心问题**：在复杂的软件工程等任务中，智能体需要执行多个步骤（如搜索、编辑、运行测试）。现有的强化学习方法（如 GRPO）通常将最终的成功或失败奖励均匀地分配给轨迹中的每一个动作，这导致无法区分“关键的好步”和“无用的废步”。
*   **研究背景**：
    *   **无评论家方法（如 GRPO）**：虽然高效，但缺乏步级（Step-level）的分辨率。
    *   **学习价值网络（Critic）**：在奖励稀疏（只有最终成功/失败）的情况下，训练 Critic 网络开销巨大且极不稳定。
*   **核心洞察**：针对同一个问题生成的多个 Rollout（展开轨迹）往往会经过相似的中间状态。这些轨迹可以组合成一棵“展开树”，通过比较同一状态下不同动作的后续收益，可以天然地估计每个动作的优劣。

### 2. 方法论：RTMC（展开树蒙特卡洛）
*   **核心思想**：利用成组采样（Group Rollouts）产生的重叠状态构建一棵树，通过蒙特卡洛（MC）统计方法计算每个状态-动作对的 Q 值和优势值（Advantage），从而实现无需 Critic 网络的步级信用分配。
*   **关键技术细节**：
    *   **状态-动作签名系统（Signatures）**：由于 LLM 的原始上下文（Raw Context）几乎不可能完全一致，作者设计了抽象签名。
        *   **动作签名**：将工具调用分类（如 view, modify, test），并结合操作范围（如行号桶）和内容哈希。
        *   **状态签名**：记录已操作的文件列表、操作类型集合及关键标志位，实现顺序无关的状态匹配。
    *   **优势估计公式**：
        *   $\hat{Q}(s, a)$：访问过状态 $s$ 并采取动作 $a$ 的所有轨迹的平均折扣回报。
        *   $\hat{V}(s)$：访问过状态 $s$ 的所有轨迹的平均回报。
        *   $\hat{A}(s, a) = \hat{Q}(s, a) - \hat{V}(s)$。
    *   **先验平滑（Prior-based Smoothing）**：针对只被访问一次的节点（无法计算优势），引入全局成功率作为先验进行贝叶斯收缩，确保信号的丰富性。
    *   **任务特定奖励塑造**：为验证动作（如运行测试）提供微小的正向奖励，防止折扣因子 $\gamma$ 导致智能体因追求速度而跳过必要的验证步骤。

### 6. 主要结论与发现
*   **性能提升**：RTMC 在 SWE-bench Verified 上达到了 **52.2% 的 pass@1**，比 GRPO 高出 **3.2 个百分点**，比未训练的 Baseline 高出 5.4 个百分点。
*   **学习效率**：训练曲线显示，RTMC 比轨迹级 GRPO 收敛更快且最终性能更高。
*   **信用分配质量**：象限分析表明，RTMC 能够识别出成功轨迹中的低质量动作，并能“拯救”失败轨迹中的高质量动作，提供了更精准的梯度方向。

### 7. 优点
*   **轻量化且高效**：无需训练额外的 Critic 网络，计算开销极低（线性复杂度），直接复用已有的 Rollout 数据。
*   **理论完备**：基于经典的蒙特卡洛估计，通过签名系统巧妙解决了 LLM 状态空间爆炸导致的匹配难题。
*   **即插即用**：可以轻松集成到现有的 GRPO 或 PPO 训练流水线中。

### 8. 不足与局限
*   **人工设计依赖**：状态和动作的签名系统（Signature）需要较强的领域知识进行手工设计，不同领域的通用性受限。
*   **绝对增益有限**：虽然相比 GRPO 有显著提升，但 3.2% 的绝对增益在复杂任务中仍有提升空间。
*   **状态孤岛问题**：由于采样数有限（N=8），树中仍有大量节点是孤立的，依赖先验平滑而非真实的跨轨迹对比。
*   **单一基准**：仅在 SWE-bench 上验证，尚未在数学推理、通用对话等其他多步决策场景中测试。
