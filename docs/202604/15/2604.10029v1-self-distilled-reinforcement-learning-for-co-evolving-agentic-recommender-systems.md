## 摘要
大语言模型赋能的智能体推荐系统（ARS）将推荐重新定义为推荐智能体与用户智能体之间的多轮交互，从而实现了超越传统单次预测的迭代式偏好诱导与细化。然而，现有的 ARS 主要在 Reflexion 风格的范式下进行优化，即通过将过去的交互轨迹存储为文本记忆，并将其检索为提示词上下文以供后续推理。尽管这种设计允许智能体召回先前的反馈和观察结果，但积累的经验仍处于模型参数之外，导致智能体依赖于通用推理，而无法通过学习逐步获得特定于推荐的决策能力。因此，强化学习（RL）提供了一种将此类交互经验内化到参数中的自然方式。然而，现有的针对 ARS 的强化学习方法仍面临两个关键局限：首先，它们未能捕捉到 ARS 的交互本质，即推荐智能体和用户智能体在交互过程中不断相互影响，并能通过交互反馈自然地产生内生监督；其次，它们将丰富的多轮交互过程简化为最终结果，忽视了贯穿整个轨迹的密集监督信号。为此，我们提出了 CoARS，一种用于协同演化智能体推荐系统的自蒸馏强化学习框架。CoARS 引入了两种互补的学习方案：交互奖励（interaction reward），从同一交互轨迹中为推荐智能体和用户智能体推导出耦合的任务级监督；以及自蒸馏信用分配（self-distilled credit assignment），在教师-学生约束下将历史轨迹转化为 Token 级的信用信号。在多个数据集上的实验表明，CoARS 在推荐性能和用户对齐方面均优于代表性的 ARS 基准模型。

## Abstract
Large language model-empowered agentic recommender systems (ARS) reformulate recommendation as a multi-turn interaction between a recommender agent and a user agent, enabling iterative preference elicitation and refinement beyond conventional one-shot prediction. However, existing ARS are mainly optimized in a Reflexion-style paradigm, where past interaction trajectories are stored as textual memory and retrieved as prompt context for later reasoning. Although this design allows agents to recall prior feedback and observations, the accumulated experience remains external to model parameters, leaving agents reliant on generic reasoning rather than progressively acquiring recommendation-specific decision-making ability through learning. Reinforcement learning (RL) therefore provides a natural way to internalize such interaction experience into parameters. Yet existing RL methods for ARS still suffer from two key limitations. First, they fail to capture the interactive nature of ARS, in which the recommender agent and the user agent continuously influence each other and can naturally generate endogenous supervision through interaction feedback. Second, they reduce a rich multi-turn interaction process to final outcomes, overlooking the dense supervision embedded throughout the trajectory. To this end, we propose CoARS, a self-distilled reinforcement learning framework for co-evolving agentic recommender systems. CoARS introduces two complementary learning schemes: interaction reward, which derives coupled task-level supervision for the recommender agent and the user agent from the same interaction trajectory, and self-distilled credit assignment, which converts historical trajectories into token-level credit signals under teacher-student conditioning. Experiments on multiple datasets show that CoARS outperforms representative ARS baselines in recommendation performance and user alignment.

### 1. 核心问题与整体含义（研究动机和背景）
论文关注的是**智能体推荐系统（Agentic Recommender Systems, ARS）**的演化机制。
*   **研究背景**：传统的推荐系统是单次预测，而 ARS 通过推荐智能体（RecAgent）与用户智能体（UserAgent）的多轮交互来细化用户偏好。
*   **核心问题**：
    1.  **经验外部化**：现有 ARS 多采用 Reflexion 范式，将交互历史存为文本记忆。这种方式经验留在参数之外，智能体依赖通用推理而非习得的推荐决策能力。
    2.  **奖励机制缺陷**：现有强化学习（RL）方法多依赖预定义规则或外部裁判，忽略了推荐与用户智能体之间相互影响的“内生性”监督。
    3.  **监督信号稀疏**：通常只关注最终结果（如是否点击），忽略了交互轨迹中蕴含的密集推理信息。
*   **整体含义**：提出 **CoARS** 框架，旨在通过强化学习将交互经验内化到模型参数中，实现推荐智能体与用户智能体的协同演化（Co-evolution）。

### 2. 方法论
CoARS 包含两个核心学习方案，分别解决任务级和 Token 级的优化：

*   **交互奖励（Interaction Reward）——任务级监督**：
    *   **推荐智能体奖励**：结合推荐正确性（是否命中目标物品）、用户接受强度（UserAgent 给出的分数）和交互阶段敏感度（后期错误惩罚更重）。
    *   **用户智能体奖励**：鼓励其对正确推荐给出正反馈，对错误推荐给出负反馈，并引入“同伴相似度”（Peer Similarity）作为辅助信号，模拟真实用户的决策逻辑。
*   **自蒸馏信用分配（Self-Distilled Credit Assignment）——Token 级监督**：
    *   **核心思想**：利用“事后诸葛亮”的视角，将历史轨迹重写为包含诊断信息的“参考轨迹”。
    *   **教师-学生模式**：同一个智能体切换两种模式。**教师模式**可见原始上下文+参考轨迹（包含正确答案和诊断说明）；**学生模式**仅见原始上下文。
    *   **信用计算**：通过对比教师和学生在每个 Token 上的对数概率差值，计算诊断优势（Diagnostic Advantage），引导学生智能体吸收教师的推理路径。
*   **协同演化流程**：RecAgent 和 UserAgent 在同一个交互循环中共同训练，RecAgent 学习如何精准推荐，UserAgent 学习如何真实反馈。

### 6. 主要结论与发现
*   **性能领先**：CoARS 在所有数据集上均显著优于基于记忆的范式和现有的 RL 范式。
*   **参数化学习的优势**：证明了将经验内化到模型参数（RL）比单纯依赖上下文学习（Memory）更有效。

### 7. 优点
*   **内生性设计**：奖励信号源于智能体间的自然交互，无需昂贵的人工标注或复杂的外部裁判模型。
*   **双向优化**：打破了“固定用户模拟器、只练推荐器”的传统思路，实现了双智能体的共同成长。
*   **诊断驱动**：自蒸馏机制通过“事后诊断”重写轨迹，将失败案例转化为精准的教学素材。

### 8. 不足与局限
*   **计算开销**：强化学习（尤其是涉及多轮交互和教师-学生对比）的训练成本远高于传统的监督微调或提示词工程。
*   **诊断依赖**：自蒸馏的质量高度依赖于“参考轨迹”的生成质量，如果模型本身重写能力不足，可能引入偏差。
*   **冷启动风险**：在完全没有历史交互或同伴数据的场景下，交互奖励中的某些组件（如同伴相似度）可能失效。
*   **评估局限**：主要关注 Hit@1，对于长尾推荐或多样性等其他推荐指标探讨较少。
