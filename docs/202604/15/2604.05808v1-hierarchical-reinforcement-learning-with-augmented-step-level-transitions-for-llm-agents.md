## 摘要
大语言模型（LLM）智能体在复杂的交互式决策任务中展现出了强大的能力。然而，现有的 LLM 智能体通常依赖于日益增长的交互历史，导致了高昂的计算成本和有限的可扩展性。在本文中，我们提出了 STEP-HRL，这是一个分层强化学习（HRL）框架，它通过仅以单步转移（而非完整的交互历史）为条件，实现了步级学习。STEP-HRL 对任务进行分层构建，利用已完成的子任务来表示整个任务的全局进度。通过引入局部进度模块，它还迭代且有选择性地总结每个子任务内的交互历史，从而生成局部进度的紧凑摘要。这些组件共同为高层和底层策略生成了增强的步级转移。在 ScienceWorld 和 ALFWorld 基准测试上的实验结果一致表明，STEP-HRL 在性能和泛化能力方面显著优于基准模型，同时减少了 token 的使用量。我们的代码可在 https://github.com/TonyStark042/STEP-HRL 获取。

## Abstract
Large language model (LLM) agents have demonstrated strong capabilities in complex interactive decision-making tasks. However, existing LLM agents typically rely on increasingly long interaction histories, resulting in high computational cost and limited scalability. In this paper, we propose STEP-HRL, a hierarchical reinforcement learning (HRL) framework that enables step-level learning by conditioning only on single-step transitions rather than full interaction histories. STEP-HRL structures tasks hierarchically, using completed subtasks to represent global progress of overall task. By introducing a local progress module, it also iteratively and selectively summarizes interaction history within each subtask to produce a compact summary of local progress. Together, these components yield augmented step-level transitions for both high-level and low-level policies. Experimental results on ScienceWorld and ALFWorld benchmarks consistently demonstrate that STEP-HRL substantially outperforms baselines in terms of performance and generalization while reducing token usage. Our code is available at https://github.com/TonyStark042/STEP-HRL.

#### 1. 核心问题与整体含义
现有的 LLM 智能体在处理复杂决策任务时，通常采用“历史条件化（History-conditioned）”的建模方式，即策略的输入会随着交互步数的增加而变得极长。这种设计存在三大痛点：
*   **计算成本高昂**：Transformer 的注意力机制随上下文长度呈平方级增长。
*   **决策干扰**：冗长的历史包含大量冗余或无关信息，容易掩盖关键的决策信号。
*   **信用分配困难**：在长序列中难以准确评估早期动作对最终结果的贡献。

本文的研究动机在于打破“长上下文依赖”的固有假设，提出通过**步级转移（Step-level Transitions）**来实现高效决策，使智能体仅需关注当前状态的紧凑表示。

#### 2. 方法论
STEP-HRL 框架的核心思想是将复杂的长程任务分解为分层结构，并引入进度建模来压缩历史信息：
*   **分层决策结构**：
    *   **高层策略 ($\pi_h$)**：负责将全局目标分解为一系列子任务（Subtasks），利用已完成的子任务序列作为“全局进度”的摘要。
    *   **底层策略 ($\pi_l$)**：在当前子任务指导下，结合观察值执行具体的原子动作。
*   **局部进度模块 ($\pi_p$)**：这是本文的关键创新。它在每个子任务内部迭代更新一个紧凑的文本摘要（Local Progress），有选择地提取与当前子任务相关的关键信息，从而替代原始的交互历史。
*   **增强步级转移**：通过整合“全局进度”和“局部进度”，高层和底层策略的输入均被简化为固定长度的增强状态，实现了真正的步级（而非轨迹级）决策。
*   **两阶段训练流程**：
    1.  **行为克隆 (BC)**：在专家演示数据上进行初始化，使模型快速掌握子任务分解和进度总结的格式。
    2.  **步级离线强化学习 (Offline RL)**：采用隐式 Q 学习 (IQL) 范式，通过期望分位数回归（Expectile Regression）学习价值函数，并利用优势加权回归（AWR）优化策略，避免了在线探索的低效。
*   **参数共享**：高层、底层及进度策略共享同一个 LLM 主干，仅使用不同的 Critic 网络，提高了参数效率和知识迁移能力。

*   **训练高效**：步级离线 RL 相比轨迹级 RL 更加稳定，且参数共享设计降低了微调门槛。

#### 8. 不足与局限
