## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中运用多种能力，其中“能力”是指在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的一部分任务是必不可少的。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，这要求模型在不同任务中隐式地学习这些能力。我们提出了 TRACE（将智能体循环失败转化为面向能力的训练环境），这是一个用于特定环境智能体自我提升的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，为每种能力合成一个针对性的训练环境（该环境会根据是否运用了该能力提供奖励），并在每个合成环境上通过强化学习（RL）训练 LoRA 适配器，在推理时路由到相关的适配器。实验表明，TRACE 在不同环境中具有良好的泛化性，在 $τ^2$-bench（客户服务）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用）上增加了 7 个满分，分别比最强的基准模型高出 7.4 分和 4 个满分。在相同数量的展开（rollouts）情况下，TRACE 比基准模型具有更高的扩展效率，在 $τ^2$-bench 上分别超过 GRPO 和 GEPA 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

### 2. 方法论：核心思想与技术细节
TRACE 包含四个关键步骤：
1.  **对比式能力识别（Contrastive Identification）**：
    *   分析智能体在目标环境中的成功与失败轨迹。
    *   利用 LLM 识别出能显著区分成功与失败的“能力缺陷”。
    *   计算“对比差距”（Contrastive Gap），优先处理那些在失败轨迹中频繁缺失但在成功轨迹中表现良好的能力。
2.  **针对性环境合成（Environment Synthesis）**：
    *   为每种缺失能力自动生成一个合成的、可验证的训练环境。
    *   环境保持与目标环境一致的工具 Schema 和交互协议，但任务高度简化并专注于单一能力的练习。
    *   提供密集的、程序化验证的奖励信号。
3.  **强化学习训练（RL Training）**：
    *   针对每个合成环境，使用 **GRPO**（组相对策略优化）算法训练一个独立的 **LoRA 适配器**。
    *   这种方法避免了全参数更新，且每个适配器只负责一项特定能力。
4.  **能力组合与路由（Routing）**：
    *   在推理阶段，使用冻结的基础模型作为路由器，根据任务描述选择最匹配的 LoRA 适配器。
    *   如果任务不需要特殊能力，则回退到基础模型。

### 6. 主要结论与发现
*   **性能显著提升**：TRACE 在 $\tau^2$-Bench 上比基础模型提升了 **14.1%**，在 ToolSandBox 上提升了 **0.141** 的相似度。
*   **超越强基准**：性能优于直接在目标环境训练的 GRPO（+9.2%）和提示词优化方法 GEPA（+7.4%）。
*   **高数据效率**：在相同的 Rollout 预算下，TRACE 的性能提升速度远快于传统 RL 方法。
*   **路由优于合并**：将不同能力的适配器分开路由，比将它们合并成一个模型效果更好，避免了参数干扰。

### 7. 优点
*   **精准诊断**：通过对比分析自动定位“病灶”，避免了盲目训练。
*   **模块化**：使用 LoRA 适配器和路由机制，使得系统易于扩展新能力，且不会破坏基础模型的能力。
*   **自动化闭环**：从失败分析到环境生成再到训练，整个流程高度自动化。
*   **可扩展性**：实验证明其性能随资源投入（Rollouts）呈现良好的单调增长趋势。

### 8. 不足与局限
*   **推理开销**：虽然 LoRA 路由开销较小，但在推理时仍需进行一次分类判断，增加了少许延迟。
*   **依赖基础模型质量**：环境合成和能力识别依赖于一个足够强大的 LLM（如文中使用的 30B 模型），对于极弱的基础模型可能失效。
*   **能力重叠风险**：如果识别出的能力定义存在严重重叠，可能会影响路由的准确性。
*   **环境保真度**：合成环境虽然保留了 Schema，但可能无法完全模拟真实环境中的极端边缘情况。
