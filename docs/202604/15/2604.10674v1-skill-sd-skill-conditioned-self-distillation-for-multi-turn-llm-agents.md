## 摘要
强化学习 (RL) 已广泛用于训练多轮交互任务中的大语言模型 (LLM) 智能体，但其样本效率受到稀疏奖励和长时序的严重限制。同策略自蒸馏 (OPSD) 通过提供来自拥有地面真值 (ground-truth) 答案的特权教师的密集 Token 级监督来缓解这一问题。然而，这种固定的特权信息无法捕捉智能体任务中多样化的有效策略，且将 OPSD 与 RL 简单结合往往会导致训练崩溃。为了解决这些局限性，我们提出了 Skill-SD，这是一个将智能体自身的轨迹转化为动态的、仅限训练的监督信息的框架。完成的轨迹被总结为简洁的自然语言技能，描述了成功的行为、错误和工作流。这些技能作为动态特权信息仅对教师进行条件化，而学生始终在普通任务提示下行动，并通过蒸馏学习内化这些指导。为了稳定训练，我们推导了一种重要性加权的逆 KL 损失 (reverse-KL loss)，以提供梯度正确的 Token 级蒸馏，并动态地将教师与不断进步的学生同步。在智能体基准测试上的实验结果表明，Skill-SD 显著优于标准 RL 基准，在 AppWorld/Sokoban 上分别提升了原生 GRPO (+14.0%/+10.9%) 和原生 OPD (+42.1%/+40.6%)。项目主页：https://k1xe.github.io/skill-sd/

## Abstract
Reinforcement learning (RL) has been widely used to train LLM agents for multi-turn interactive tasks, but its sample efficiency is severely limited by sparse rewards and long horizons. On-policy self-distillation (OPSD) alleviates this by providing dense token-level supervision from a privileged teacher that has access to ground-truth answers. However, such fixed privileged information cannot capture the diverse valid strategies in agent tasks, and naively combining OPSD with RL often leads to training collapse. To address these limitations, we introduce Skill-SD, a framework that turns the agent's own trajectories into dynamic training-only supervision. Completed trajectories are summarized into compact natural language skills that describe successful behaviors, mistakes, and workflows. These skills serve as dynamic privileged information conditioning only the teacher, while the student always acts under the plain task prompt and learns to internalize the guidance through distillation. To stabilize the training, we derive an importance-weighted reverse-KL loss to provide gradient-correct token-level distillation, and dynamically synchronize the teacher with the improving student. Experimental results on agentic benchmarks demonstrate that Skill-SD substantially outperforms the standard RL baseline, improving both vanilla GRPO (+14.0%/+10.9% on AppWorld/Sokoban) and vanilla OPD (+42.1%/+40.6%). *   **核心问题**：现有的自蒸馏方法（如 OPSD）通常依赖固定的“特权信息”（如标准答案），这在路径多样的智能体任务中（如 API 调用、路径规划）显得过于僵化。此外，直接将自蒸馏与 RL 结合容易导致训练不稳定甚至崩溃。

### 2. 方法论：Skill-SD 框架
*   **核心思想**：将智能体成功的经验和失败的教训总结为自然语言“技能（Skills）”，作为**仅在训练阶段**提供给教师模型的动态特权信息。学生模型在不接触这些技能的情况下，通过蒸馏教师模型的概率分布来内化这些策略。
*   **关键技术细节**：
    *   **技能总结与存储**：使用辅助 LLM 将完成的轨迹总结为包含“成功分析”、“错误分析”和“黄金工作流”的结构化技能，并存储在基于 UCB（置信区间上界）算法索引的缓冲区中，以平衡技能的利用与探索。
    *   **技能条件化教师**：教师和学生共享参数，但教师的 Prompt 会额外拼接检索到的技能。
    *   **重要性加权逆 KL 损失**：为了解决教师（有技能）和学生（无技能）Prompt 不一致导致的分布偏移，推导并使用了重要性加权（Importance-weighted）的逆 KL 损失函数，确保 Token 级别的梯度更新是无偏且稳定的。
    *   **动态教师同步**：教师模型定期从最新的学生模型检查点同步参数，确保特权信号随模型能力的提升而共同演进。

*   **技能内化**：证明了将技能作为“特权信息”供教师使用，比直接喂给学生效果更好，后者容易导致严重的过拟合和推理能力受损。
*   **稳定性关键**：动态同步教师参数和使用重要性加权损失是防止训练中途崩溃（Mid-training collapse）的核心。

### 7. 优点
*   **创新性**：将静态的特权信息扩展为动态的自然语言技能，更符合智能体任务的特性。

*   **推理高效**：学生模型在推理时不需要检索技能或增加 Prompt 长度，保持了原始接口的简洁和高效。

### 8. 不足与局限
*   **检索机制简单**：目前使用 UCB 算法进行任务索引检索，当任务池极大时，可能需要更复杂的语义检索（Embedding-based）。
*   **Token 级限制**：蒸馏仅在采样到的 Token 上进行，而非全词表蒸馏，虽然节省了算力，但可能损失了一部分分布对齐的精度。
*   **依赖辅助模型**：技能的总结依赖于另一个 LLM（如 Seed1.8），这在完全闭环的自进化系统中引入了外部依赖。
