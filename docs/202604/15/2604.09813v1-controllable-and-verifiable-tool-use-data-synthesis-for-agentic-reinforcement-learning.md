## 摘要
现有的合成工具使用语料库主要针对离线监督微调（SFT）设计，而强化学习（RL）则需要支持奖励可检查在线回放（online rollouts）的可执行环境。我们提出了 COVERT，这是一个两阶段流水线：首先通过带有多级验证的自进化合成生成可靠的基础工具使用轨迹，然后应用保持真值（oracle-preserving）的增强手段，系统性地增加环境复杂度。这些增强引入了干扰工具、间接或模糊的用户查询，以及噪声、多格式或错误的工具输出，同时严格保留原始工具调用和最终答案作为地面真值（ground truth）。这种设计使得标准案例可以通过参考匹配进行自动奖励计算，而对于错误检测等特殊行为，则可以通过轻量级的评判员辅助验证，从而支持工具调用策略的 RL 优化。在 Qwen2.5-Instruct-14B 上，COVERT-RL 将 BFCL v3 的总准确率从 56.5 提升至 59.9，在 ACEBench 上从 53.0 提升至 59.3，且在通用能力基准测试上的性能退化极小；当在 SFT 基础上叠加时，准确率进一步达到 62.1 和 61.8，证实了增量收益。这些结果表明，保持真值的合成环境提供了一个实用的 RL 精炼阶段，与 SFT 互补，能够提高在模糊性和不可靠工具反馈下的工具使用鲁棒性。

## Abstract
Existing synthetic tool-use corpora are primarily designed for offline supervised fine-tuning, yet reinforcement learning (RL) requires executable environments that support reward-checkable online rollouts. We propose COVERT, a two-stage pipeline that first generates reliable base tool-use trajectories through self-evolving synthesis with multi-level validation, and then applies oracle-preserving augmentations that systematically increase environmental complexity. These augmentations introduce distractor tools, indirect or ambiguous user queries, and noisy, multi-format, or erroneous tool outputs, while strictly preserving oracle tool calls and final answers as ground truth. This design enables automatic reward computation via reference matching for standard cases and lightweight judge-assisted verification for special behaviors such as error detection, supporting RL optimization of tool-calling policies. On Qwen2.5-Instruct-14B, COVERT-RL improves overall accuracy on BFCL v3 from 56.5 to 59.9 and on ACEBench from 53.0 to 59.3, with minimal regressions on general-ability benchmarks; when stacked on SFT, it further reaches 62.1 and 61.8, confirming additive gains. These results suggest that oracle-preserving synthetic environments offer a practical RL refinement stage, complementary to SFT, for improving tool-use robustness under ambiguity and unreliable tool feedback.

### 论文总结：Controllable and Verifiable Tool-Use Data Synthesis for Agentic Reinforcement Learning (COVERT)

#### 1. 核心问题与研究动机
*   **研究背景**：大型语言模型（LLM）的工具使用能力通常通过合成数据进行监督微调（SFT）。然而，现有的合成语料库多为静态轨迹，无法提供强化学习（RL）所需的在线交互环境和可验证的奖励反馈。
*   **核心问题**：现实中的工具调用环境复杂多变（包含干扰工具、模糊查询、噪声输出等），而现有的训练数据难以覆盖这些边缘情况。此外，如何在合成环境中为在线回放（Rollout）提供自动且准确的奖励计算是一个重大挑战。
*   **研究动机**：构建一个可控且可验证的合成环境，通过强化学习进一步提升模型在复杂、不可靠反馈场景下的工具调用鲁棒性。

#### 2. 方法论：COVERT 流水线
COVERT 采用两阶段流水线构建 RL 训练环境：
*   **第一阶段：可靠基础轨迹生成**
    *   **自进化合成**：从少量种子数据开始，利用 LLM 迭代生成新的工具使用案例。
    *   **多级验证**：通过格式检查（JSON 结构）、规则检查（参数类型/约束）和语义检查（LLM 评判工具选择的合理性）确保基础数据的极高可靠性。
*   **第二阶段：保持真值的环境增强（Oracle-Preserving Augmentation）**
    *   **核心思想**：在修改环境组件（如查询、工具集、输出）的同时，严格保留原始的“真值（Oracle）”工具调用和最终答案。
    *   **增强类型**：
        *   **精确可验证增强**：引入干扰工具、重写模糊查询、改变输出格式。奖励通过参考匹配（Reference Matching）自动计算。
        *   **评判员辅助增强**：模拟工具报错或无效查询。奖励由轻量级 LLM 评判员根据预设的真值标签进行判定。
*   **强化学习算法**：采用 **GRPO**（Group Relative Policy Optimization），通过格式奖励和能力条件化的正确性奖励进行策略优化。

#### 3. 实验设计
*   **基准测试（Benchmarks）**：
    *   **工具调用**：BFCL v3（伯克利函数调用排行榜）和 ACEBench。
    *   **通用能力**：LiveCodeBench（代码）、AIME/Math500（数学）、MMLU-Pro（知识）、IFEval（指令遵循）等。
*   **对比方法**：
    *   Qwen2.5-Instruct（原始模型）。
    *   TOUCAN-SFT（在 150 万条真实 MCP 数据上微调的模型）。
    *   COVERT-RL（本文方法）。
    *   TOUCAN-SFT + COVERT-RL（叠加方案）。
*   **评估协议**：使用统一的“极简提示词（Minimal Prompt）”，仅提供工具签名，不提供行为指导，以测试模型的原生鲁棒性。

#### 4. 资源与算力
*   **硬件环境**：使用 **8 × NVIDIA H200 GPU**。
*   **训练时长**：
    *   7B 模型：单次完整 RL 训练约 **28 小时**。
    *   14B 模型：训练时长约为 7B 模型的 **1.6 倍**（约 45 小时）。
*   **框架**：基于开源 RL 框架 VERL。

#### 5. 实验数量与充分性
*   **实验规模**：涵盖了 7B 和 14B 两个参数量级，并在 10 多个主流基准测试上进行了验证。
*   **充分性**：实验不仅验证了工具调用能力的提升，还详细分析了不同增强类别（如干扰项检测、错误处理）的增益，并进行了通用能力保留测试。
*   **客观性**：通过在相同提示词下对比 SFT 模型和 RL 模型，客观展示了 RL 在处理“静态数据未覆盖场景”时的优势。

#### 6. 主要结论与发现
*   **性能显著提升**：在 Qwen2.5-14B 上，COVERT-RL 将 BFCL v3 准确率从 56.5 提升至 59.9，ACEBench 从 53.0 提升至 59.3。
*   **SFT 与 RL 的互补性**：RL 在 SFT 基础上仍有显著增益（14B 叠加后达到 62.1/61.8），表明 RL 擅长精炼 SFT 难以捕捉的鲁棒性行为。
*   **通用能力保持**：与 SFT 可能导致的数学/推理能力下降不同，COVERT-RL 在提升工具能力的同时，几乎没有造成通用能力的退化。
*   **鲁棒性增强**：模型在处理模糊查询、干扰工具和工具报错方面的表现大幅改善。

#### 7. 优点
*   **可验证性强**：通过“保持真值”的设计，解决了合成数据在 RL 训练中难以自动评分的痛点。
*   **可控性高**：允许开发者针对特定的弱点（如噪声输出）进行定向的增强训练。
*   **训练效率**：采用 GRPO 算法，无需价值模型（Value Model），降低了显存占用和训练复杂度。

#### 8. 不足与局限
*   **长程规划限制**：目前的模拟环境主要针对短到中程交互，对于需要极深状态依赖的长程智能体规划（Long-horizon planning）支持有限。
*   **评判员依赖**：部分特殊行为（如错误检测）仍依赖 LLM 评判员，虽然有真值约束，但仍存在非确定性风险，且缺乏系统的人类一致性研究。
*   **环境模拟深度**：目前的“模拟工具箱”是无状态或弱状态的，无法完全模拟复杂的真实世界状态机。
