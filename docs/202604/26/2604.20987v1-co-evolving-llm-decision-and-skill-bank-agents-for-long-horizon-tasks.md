## 摘要
长程交互环境是评估智能体技能使用能力的试验场。这些环境要求多步推理、在多个时间步长内链接多种技能，以及在延迟奖励和部分可观测性下进行稳健决策。游戏是评估智能体在环境中技能使用的良好试验场。大语言模型（LLMs）作为游戏智能体提供了一种极具前景的选择，但由于缺乏在不同回合间发现、保留和重用结构化技能的机制，它们往往难以实现一致的长程决策。我们提出了 COSPLAY，这是一个协同演化框架。在该框架中，LLM 决策智能体从可学习的技能库中检索技能以指导动作执行，而由智能体管理的技能流水线则从智能体的无标签轨迹中发现可重用技能以构建技能库。我们的框架提升了决策智能体，使其学习更好的技能检索和动作生成，同时技能库智能体不断提取、精炼并更新技能及其契约。在六个游戏环境中的实验表明，使用 8B 基础模型的 COSPLAY 在单人游戏基准测试中，相比四个前沿 LLM 基准模型实现了超过 25.1% 的平均奖励提升，同时在多人社交推理游戏中也保持了竞争力。

## Abstract
Long horizon interactive environments are a testbed for evaluating agents skill usage abilities. These environments demand multi step reasoning, the chaining of multiple skills over many timesteps, and robust decision making under delayed rewards and partial observability. Games are a good testbed for evaluating agent skill usage in environments. Large Language Models (LLMs) offer a promising alternative as game playing agents, but they often struggle with consistent long horizon decision making because they lack a mechanism to discover, retain, and reuse structured skills across episodes. We present COSPLAY, a co evolution framework in which an LLM decision agent retrieves skills from a learnable skill bank to guide action taking, while an agent managed skill pipeline discovers reusable skills from the agents unlabeled rollouts to form a skill bank. Our framework improves both the decision agent to learn better skill retrieval and action generation, while the skill bank agent continually extracts, refines, and updates skills together with their contracts. Experiments across six game environments show that COSPLAY with an 8B base model achieves over 25.1 percent average reward improvement against four frontier LLM baselines on single player game benchmarks while remaining competitive on multi player social reasoning games.

#### 1. 核心问题与研究动机
*   **研究背景**：长程交互任务（如复杂游戏）要求智能体具备多步推理、技能链条化以及在延迟奖励和部分可观测环境下的稳健决策能力。
*   **核心问题**：尽管大语言模型（LLM）在游戏决策中展现出潜力，但它们通常难以维持长期的决策一致性，主要原因是缺乏一种能够跨回合发现、保留和复用结构化技能的机制。
*   **研究动机**：现有的方法要么依赖人工标注的技能，要么依赖固定的工具调用。本文旨在构建一个能够通过自我交互，在无标签轨迹中自动学习和进化技能库的闭环系统。

#### 2. 方法论：COS-PLAY 框架
COS-PLAY 采用**协同演化（Co-evolution）**机制，包含两个核心组件：
*   **决策智能体 (Decision Agent, $A_D$)**：
    *   **功能**：负责与环境交互。它根据当前状态摘要从技能库中检索候选技能，更新内部“意图”（Intention），并执行具体的原子动作。
    *   **流程**：状态观察 -> 技能检索 -> 意图更新 -> 动作执行。
*   **技能库智能体 (Skill Bank Agent, $A_S$)**：
    *   **功能**：负责技能的无监督发现与维护。
    *   **关键技术**：
        1.  **边界提议**：识别轨迹中的潜在技能切换点。
        2.  **分段推断**：将轨迹切分为技能片段，并与现有技能匹配或标记为新技能。
        3.  **契约学习 (Contract Learning)**：学习技能执行前后的状态变化（契约），用于验证技能的有效性。
        4.  **库维护**：执行精炼、具象化、合并、拆分和淘汰等操作，保持技能库的精简与高效。
*   **优化算法**：使用 **GRPO (Group Relative Policy Optimization)** 算法，通过多个 **LoRA 适配器** 分别优化决策（动作、检索）和技能库管理（分段、契约、策展）的不同功能。

#### 3. 实验设计
*   **实验场景**：涵盖 6 个游戏环境。
    *   **单人游戏**：2048、Candy Crush（糖果传奇）、Tetris（俄罗斯方块）、Super Mario Bros（超级马里奥）。
    *   **多人社交推理游戏**：Avalon（阿瓦隆）、Diplomacy（外交）。
*   **基准模型 (Benchmarks)**：对比了前沿模型 GPT-5.4、Gemini-3.1-Pro、Claude-4.6-Sonnet 和 GPT-OSS-120B。
*   **消融实验**：对比了无技能库的 SFT/GRPO 模型、使用初始技能库但不进化的模型等，以验证协同演化的必要性。

#### 4. 资源与算力
*   **硬件环境**：实验在拥有 **8 张 NVIDIA A100 GPU** 的集群上运行。
*   **训练细节**：
    *   基础模型使用 Qwen3-8B。
    *   冷启动阶段使用 GPT-5.4 生成 60 条种子轨迹进行 SFT。
    *   协同演化训练通常在 25 个迭代步内达到强劲性能。
    *   具体的超参数（学习率、KL 系数等）针对不同游戏进行了微调。

#### 5. 实验数量与充分性
*   **实验规模**：单人游戏每项进行 16 次评估运行，多人游戏每名玩家进行 10 次运行，并报告了 95% 的置信区间。
*   **充分性**：实验覆盖了从简单逻辑到复杂社交博弈的多种维度。除了游戏表现，还测试了模型在 Math-500 和 MMLU-Pro 上的通用推理能力，证明了该方法在提升特定任务能力的同时，较好地保留了通用性。
*   **客观性**：通过与多种闭源顶级模型对比，并提供详细的消融实验和定性分析（如失败案例分析），实验设计较为客观、公平。

#### 6. 主要结论与发现
*   **性能飞跃**：基于 8B 基础模型的 COS-PLAY 在单人游戏上比 GPT-5.4 平均提升了 **25.1%** 的奖励。
*   **社交博弈能力**：在阿瓦隆和外交等复杂社交游戏中，COS-PLAY 表现出与 120B 甚至更大型模型相当的竞争力。
*   **技能复用性**：分析表明，系统学到的是具有抽象意义和可复用性的技能协议，而非简单的轨迹记忆。
*   **协同效应**：消融实验证明，仅有技能库或仅有 RL 优化是不够的，决策策略与技能库的**同步对齐**是性能提升的关键。

#### 7. 优点
*   **无监督技能发现**：无需人工干预即可从原始轨迹中提取结构化技能。
*   **数据效率高**：通过协同演化，模型能在较少的迭代次数内实现快速适配。
*   **结构化表达**：将技能定义为包含“前提、计划、成功/中止标准、契约”的结构化协议，增强了决策的可解释性。

#### 8. 不足与局限
*   **文本依赖性**：目前主要依赖文本形式的状态摘要，可能会丢失原始视觉观察中的细微信息。
*   **摘要误差累积**：在极长程的任务中，文本摘要的错误可能会随时间累积，导致技能检索失效。
*   **冷启动依赖**：初始阶段仍需高性能模型（如 GPT-5.4）提供种子轨迹。
*   **多模态扩展**：尚未在纯视觉或复杂多模态物理环境中进行验证。
