## 摘要
我们介绍了 SuperIgor，一个用于指令遵循任务的框架。与以往依赖预定义子任务的方法不同，SuperIgor 通过自学习机制使语言模型能够生成并优化高层计划，从而减少了对手动数据集标注的需求。我们的方法涉及迭代协同训练：训练强化学习（RL）智能体以遵循生成的计划，同时语言模型根据 RL 的反馈和偏好来调整和修改这些计划。这形成了一个反馈循环，使得智能体和规划器能够共同提升。我们在具有丰富动力学特性和随机性的环境中验证了我们的框架。结果表明，SuperIgor 智能体比基线方法更严格地遵循指令，同时也展现出对先前未见指令的强大泛化能力。

## Abstract
We introduce SuperIgor, a framework for instruction-following tasks. Unlike prior methods that rely on predefined subtasks, SuperIgor enables a language model to generate and refine high-level plans through a self-learning mechanism, reducing the need for manual dataset annotation. Our approach involves iterative co-training: an RL agent is trained to follow the generated plans, while the language model adapts and modifies these plans based on RL feedback and preferences. This creates a feedback loop where both the agent and the planner improve jointly. We validate our framework in environments with rich dynamics and stochasticity. Results show that SuperIgor agents adhere to instructions more strictly than baseline methods, while also demonstrating strong generalization to previously unseen instructions.

#### 1. 核心问题与整体含义（研究动机和背景）
在具身智能领域，让 AI 智能体遵循自然语言指令完成任务是一个核心挑战。现有的方法通常分为两类：**基于演示的学习**（成本高、扩展性差）和**基于经验的强化学习**（在长程任务和稀疏奖励下难以收敛）。
目前的规划方法大多依赖于**预定义的子任务库**，这限制了智能体在没有先验知识的新环境中的应用。本文提出了 **SuperIgor** 框架，旨在解决如何在**没有预定义技能、没有专家演示且奖励稀疏**的动态环境中，通过大语言模型（LLM）与强化学习（RL）的协同进化，实现自引导的指令遵循。

#### 2. 方法论：核心思想与关键技术
SuperIgor 的核心是一个**迭代协同训练循环**，包含以下四个关键阶段：
*   **自引导计划提取（Plan Generation）：** 
    *   利用 LLM 从指令集中提取原子子任务，构建**子任务库**。
    *   通过多次查询 LLM 建立子任务之间的依赖关系，生成**本体图（Ontology Graph）**。
    *   根据本体图将复杂指令展开为多个候选的线性计划。
*   **技能课程学习（Skill Curriculum Learning）：** 
    *   为了解决稀疏奖励问题，提出了一种课程学习算法。智能体从单步任务开始，只有当当前子任务的成功率达到阈值（如 0.7）时，才会在训练集中加入包含该子任务及一个新子任务的复杂计划。
*   **计划验证（Plan Validation）：** 
    *   RL 智能体执行生成的计划，收集执行轨迹和成功率，作为反馈信号。
*   **LLM 微调与对齐（LLM Fine-Tuning）：** 
    *   使用**直接偏好优化（DPO）**，根据 RL 智能体的实际表现对 LLM 进行微调。这使得 LLM 能够学习到哪些计划在当前环境中更易于被智能体执行，从而实现规划器与执行器的协同进化。

#### 3. 实验设计
*   **环境与数据集：** 使用 **CrafText** 基准测试（一个类似 Minecraft 的动态、部分可观测环境）。作者构建了 **FOCUS 数据集**，包含 900 多条指令，涵盖原子任务（Atomic）和组合任务（Combo）。
*   **评估指标：** 成功率（Success Rate, SR）。
*   **对比方法（Baselines）：**
    *   **PPO-T：** 文本增强的 PPO，直接处理指令。
    *   **PPO-T+：** 使用 GPT-4 生成的固定计划进行增强。
    *   **FiLM：** 使用特征线性调制层融合视觉和语言信息。
    *   **Oracle：** 使用人工编写的完美计划进行训练的上限模型。
*   **测试集：** 包含**同义改写（Paraphrases）**和**新对象组合（New Objects）**，用于测试泛化性。

实验设计非常充分且具有说服力：

*   **模型敏感性分析：** 测试了不同规模和家族的 LLM（Qwen, Gemma, Phi-4, NextCoder 等）作为规划器的效果。
*   **泛化性测试：** 专门设置了 OOD（分布外）测试集，验证模型对新语言表达和新任务组合的鲁棒性。

#### 6. 主要结论与发现
*   **性能卓越：** SuperIgor 在原子任务和组合任务上的表现远超 PPO-T 等基线方法。
*   **泛化能力强：** 在“新对象组合”测试中，SuperIgor 的表现比 Oracle 模型更稳定，因为它在训练中接触到了更多样化的候选计划。
*   **协同进化的必要性：** 仅靠增加 RL 训练步数无法突破性能瓶颈，LLM 规划器的迭代优化（DPO）是提升性能的关键。
*   **课程学习的作用：** 课程学习显著加速了子任务的掌握过程，尤其是在处理长程依赖任务时。

#### 7. 优点
*   **无需人工标注：** 实现了从指令到计划、再到执行的全自动化闭环，极大地降低了数据准备成本。
*   **自适应性：** 通过 DPO 将 LLM 的通用知识与特定环境的动力学特征进行了有效对齐。
*   **鲁棒性：** 能够处理自然语言的多样性（改写）和任务的组合复杂性。

#### 8. 不足与局限
*   **初始计划依赖：** 框架性能高度依赖于 LLM 初始生成的本体图质量。如果 LLM 无法理解环境的基本逻辑，后续的迭代可能失效。
*   **本体图缺陷：** 自动生成的本体图可能存在冗余、不一致或循环依赖，目前主要靠算法后处理（如剪枝）缓解。
*   **计算成本：** 虽然比人工标注便宜，但多次迭代 RL 训练和 LLM 微调仍需要可观的算力资源。
*   **可解释性：** 当训练停滞时，很难界定是 RL 探索不足还是 LLM 规划逻辑存在系统性错误。
