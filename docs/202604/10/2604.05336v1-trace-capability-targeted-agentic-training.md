## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中运用多种能力，其中能力是指在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的一部分任务是必不可少的。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，这要求模型在不同任务间隐式地学习这些能力。我们介绍了 TRACE（将循环出现的智能体失败转化为针对能力的训练环境），这是一个用于特定环境下智能体自我提升的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，为每种能力合成一个针对性的训练环境，并根据是否运用了该能力提供奖励，然后在每个合成环境上通过强化学习（RL）训练 LoRA 适配器，并在推理时路由到相关的适配器。实验表明，TRACE 在不同环境中具有良好的泛化性，在 $τ^2$-bench（客服场景）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用场景）上增加了 7 个满分，分别比最强的基准模型高出 7.4 分和 4 个满分。在给定相同采样次数的情况下，TRACE 的扩展效率高于基准模型，在 $τ^2$-bench 上分别超过 GRPO 和 GEPA 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

### 2. 方法论：核心思想与算法流程
TRACE（Turning Recurrent Agent failures into Capability-targeted training Environments）包含四个关键步骤：
1.  **对比能力识别（Contrastive Identification）**：
    *   让基础模型在目标环境中运行，收集成功和失败的轨迹。
    *   使用 LLM 分析智能体对比两类轨迹，识别出那些在失败案例中频繁缺失、但在成功案例中存在的“关键能力”。
    *   计算“对比差距”（Contrastive Gap），优先训练那些对失败贡献最大的能力。
2.  **针对性环境合成（Environment Synthesis）**：
    *   为每种识别出的能力自动编写一个程序化的合成环境。
    *   这些环境保持与目标环境相同的工具 Schema 和交互协议，但任务高度简化且专注于单一能力的验证。
    *   提供密集的、可验证的奖励信号（基于状态变化或工具参数）。
3.  **针对性强化学习（Targeted RL）**：
    *   在每个合成环境上，使用 **GRPO**（组相对策略优化）算法训练独立的 **LoRA 适配器**。
    *   这种方式避免了在复杂目标环境中学习时的噪声干扰。
4.  **能力组合与路由（Composition & Routing）**：
    *   在推理阶段，使用基础模型作为路由器，根据当前任务描述选择最匹配的 LoRA 适配器。
    *   如果任务不匹配任何已知能力，则回退到基础模型。

*   **扩展效率高**：随着采样次数的增加，TRACE 的性能提升比直接 RL 更稳定且斜率更高，没有出现直接 RL 常见的性能震荡或平台期。
*   **训练优于提示词**：实验证明，仅仅在提示词中加入能力说明是不够的，通过针对性 RL 改变模型行为至关重要。

### 7. 优点
*   **精准诊断**：通过对比分析自动定位“病灶”，避免了盲目训练。
*   **信用分配优化**：将复杂的长程任务分解为易于验证的微型环境，提供了高质量的奖励信号。
*   **模块化扩展**：采用 LoRA 路由机制，可以像插件一样不断添加新能力，而不会导致旧能力的灾难性遗忘。
*   **端到端自动化**：从失败分析到环境生成再到训练，全流程减少了人工干预。

### 8. 不足与局限
*   **路由依赖性**：系统的最终效果高度依赖于“路由器”能否准确判断任务所需的能力。如果任务需要多种能力的深度融合（而非交替使用），简单的路由可能不足。
*   **分析器瓶颈**：能力识别依赖于另一个 LLM（分析智能体）的判断，如果分析智能体本身无法理解复杂的失败原因，则无法生成有效的训练环境。
*   **环境合成的复杂性**：虽然论文展示了自动合成环境的可能性，但对于某些极度抽象或需要复杂外部物理模拟的能力，合成环境的保真度可能受限。
*   **计算成本**：虽然训练是高效的，但为每个能力维护独立的适配器在管理大规模（如数百个）能力时可能面临存储和调度挑战。
