## 摘要
大型基础模型为自主系统提供了强大的推理能力，但将语义意图映射为可靠的实时控制仍具挑战。现有方法要么 (i) 让大语言模型 (LLM) 直接生成轨迹——这种方式脆弱、难以验证且易产生延迟；要么 (ii) 在线调整模型预测控制 (MPC) 的目标——这混合了缓慢的深思熟虑与快速控制，模糊了接口界限。我们提出了“智能体快慢规划”（Agentic Fast-Slow Planning），这是一个在自然时间尺度上解耦感知、推理、规划和控制的分层框架。该框架包含两个桥梁：“感知到决策”（Perception2Decision）利用车载视觉语言模型 (VLM) 检测器将场景压缩为以自我为中心的拓扑结构，随后通过云端 LLM 决策器将其映射为符号化驾驶指令，在保持可解释性的同时减少了带宽和延迟；“决策到轨迹”（Decision2Trajectory）将指令转换为可执行路径，其中“语义引导的 A*”算法将源自语言的软成本嵌入经典搜索中，使解偏向于可行轨迹，而“智能体细化模块”则利用反馈和记忆调整规划器的超参数。最后，MPC 实时跟踪轨迹，并针对困难情况提供可选的云端引导参考。在 CARLA 中的实验表明，与纯 MPC 和 A* 引导的 MPC 基准相比，智能体快慢规划提高了扰动下的鲁棒性，横向偏差减少了高达 45%，完成时间缩短了 12% 以上。代码见 https://github.com/cjychenjiayi/icra2026_AFSP。

## Abstract
Large foundation models enable powerful reasoning for autonomous systems, but mapping semantic intent to reliable real-time control remains challenging. Existing approaches either (i) let Large Language Models (LLMs) generate trajectories directly - brittle, hard to verify, and latency-prone - or (ii) adjust Model Predictive Control (MPC) objectives online - mixing slow deliberation with fast control and blurring interfaces. We propose Agentic Fast-Slow Planning, a hierarchical framework that decouples perception, reasoning, planning, and control across natural timescales. The framework contains two bridges. Perception2Decision compresses scenes into ego-centric topologies using an on-vehicle Vision-Language Model (VLM) detector, then maps them to symbolic driving directives in the cloud with an LLM decision maker - reducing bandwidth and delay while preserving interpretability. Decision2Trajectory converts directives into executable paths: Semantic-Guided A* embeds language-derived soft costs into classical search to bias solutions toward feasible trajectories, while an Agentic Refinement Module adapts planner hyperparameters using feedback and memory. Finally, MPC tracks the trajectories in real time, with optional cloud-guided references for difficult cases. Experiments in CARLA show that Agentic Fast-Slow Planning improves robustness under perturbations, reducing lateral deviation by up to 45% and completion time by over 12% compared to pure MPC and an A*-guided MPC baseline. Code is available at https://github.com/cjychenjiayi/icra2026_AFSP.

### 2. 方法论
该框架通过两个核心“桥梁”连接高层推理与底层控制：
*   **Perception2Decision（感知到决策）**：
    *   **车载端**：使用轻量化 VLM（Topology Detector）将原始图像压缩为以自我为中心的“拓扑图”（包含物体类别、距离、方位等），减少上传带宽。
    *   **云端**：LLM 根据拓扑图生成符号化的驾驶指令（如 LEFT, RIGHT, KEEP）和驾驶风格。
*   **Decision2Trajectory（决策到轨迹）**：
    *   **语义引导 A* 算法**：将 LLM 的符号指令转化为 A* 搜索中的“软成本”（Soft Costs）。公式上扩展了代价函数 $g(s)$，根据动作是否符合指令（正确、延迟、错误、过度）给予不同的惩罚或奖励，从而引导生成符合语义意图的路径。
    *   **智能体细化模块（Agentic Refinement）**：利用 LLM 作为一个闭环智能体，根据规划反馈和记忆库自动调整 A* 算法的超参数，替代人工调优。
*   **实时控制**：使用 **Cloud-Guided MPC** 跟踪生成的参考轨迹，在常规情况下使用本地参考，在复杂情况下切换至云端引导路径。

### 3. 实验设计
*   **实验平台**：CARLA 自动驾驶仿真器。
*   **数据集**：基于 Town2-7 和 Town10HD 构建了包含 20 万张图像的训练集，涵盖 49 类物体。
*   **对比方法（Benchmark）**：
    1.  **纯 MPC**：仅依赖局部避障。
    2.  **A*-MPC**：经典 A* 引导的 MPC。
    3.  **AFSP（本文方法）**：语义引导的分层规划框架。
*   **测试场景**：包括标准地图、带扰动的平移地图、以及增大障碍物安全半径的复杂场景。

### 4. 资源与算力
*   **模型基础**：VLM 使用 InternVL-2B，决策 LLM 使用 DeepSeek-V3 和 Qwen-VL-Chat。
*   **训练算力**：使用 **4 台 NVIDIA A100 GPU** 进行 VLM 的微调。
*   **训练时长**：微调过程持续 12 个 epoch。

### 5. 实验数量与充分性
*   **感知实验**：在 1000 帧测试集上对比了三种不同的 VLM 微调策略（仅视觉、全参数、两阶段微调）。
*   **决策实验**：对比了 48 个典型场景下 VLM 与 LLM 的决策一致性。
*   **仿真实验**：在 3 种不同难度的 CARLA 场景中进行测试，**每个场景重复 10 次**以减少随机性影响。
*   **充分性评价**：实验涵盖了从感知精度、推理延迟到最终控制表现的全链路评估，并包含消融实验（如地图扰动下的鲁棒性测试），实验设计较为客观、公平。

### 6. 主要结论与发现
*   **性能提升**：相比纯 MPC，AFSP 将**横向偏差减少了约 45%**，任务**完成时间缩短了 12% 以上**。
*   **鲁棒性**：语义引导的 A* 算法在地图发生平移扰动时，仍能保持语义意图的一致性，而传统 A* 容易失效。
*   **效率**：通过将图像转化为拓扑图，显著降低了云端通信开销，且 LLM 的推理速度优于直接使用大型 VLM。
*   **自动化**：智能体细化模块能有效通过闭环反馈找到最优规划参数，无需人工干预。

### 7. 优点
*   **架构解耦**：成功实现了“慢速语义推理”与“快速几何控制”的解耦，符合自动驾驶的实时性需求。
*   **可解释性**：中间层输出符号化指令（LEFT/RIGHT），使大模型的决策过程可被监控和理解。
*   **软约束设计**：将语义指令作为 A* 的软成本而非硬约束，既保证了意图引导，又保留了避障的安全性底线。

### 8. 不足与局限
*   **云端依赖**：系统高度依赖云端 LLM 的实时响应，若网络中断或延迟波动，系统退化为纯本地控制的效果尚未深入探讨。
*   **仿真局限**：所有实验均在 CARLA 仿真环境中完成，真实世界的感知噪声和动态不确定性可能比仿真环境更复杂。
*   **计算开销**：虽然解耦了快慢规划，但 Agentic Refinement 模块涉及多次 LLM 调用，在极端紧急避障场景下的响应能力可能受限。
