## 摘要
随着大语言模型智能体从软件工程（SWE）任务向机器学习工程（MLE）任务迈进，验证智能体行为的成本呈数量级增长：虽然 SWE 任务可以通过快速执行的单元测试进行验证，但 MLE 验证需要在每个展开（rollout）步骤中在大型数据集上运行完整的 ML 流水线——包括数据预处理、模型训练和指标评估——这使得基于轨迹的在线强化学习（on-policy RL）变得极其缓慢。现有方法退而求其次，采用监督微调（SFT）或离线代理奖励，牺牲了在线强化学习在探索和泛化方面的优势。我们观察到，沙盒数据规模是这一瓶颈的主要来源。基于这一洞察，我们提出了 SandMLE，这是一个多智能体框架，能够从少量种子任务中生成多样化且可验证的合成 MLE 环境，在保留现实世界问题的结构和技术复杂性的同时，将数据集限制在微观规模（每个任务仅配有 50-200 个训练样本）。通过广泛的实验，我们证明 SandMLE 将执行时间缩短了 13 倍以上，首次在 MLE 领域实现了大规模、基于轨迹的在线强化学习。在 MLE-bench-lite 上，SandMLE 在 Qwen3-8B、14B 和 30B-A3B 模型上均取得了优于 SFT 基线的显著收益，相对奖牌率提升了 20.3% 至 66.9%。此外，训练后的策略可以泛化到未见的智能体脚手架（agentic scaffolds），在 MLE-Dojo 上的 HumanRank 分数提升了高达 32.4%。

## Abstract
As large language model agents advance beyond software engineering (SWE) tasks toward machine learning engineering (MLE), verifying agent behavior becomes orders of magnitude more expensive: while SWE tasks can be verified via fast-executing unit tests, MLE verification requires running full ML pipelines -- data preprocessing, model training, and metric evaluation -- on large datasets at each rollout step, rendering trajectory-wise on-policy reinforcement learning (RL) prohibitively slow. Existing approaches retreat to supervised fine-tuning (SFT) or offline proxy rewards, sacrificing the exploration and generalization benefits of on-policy RL. We observe that sandbox data size is the primary source of this bottleneck. Based on this insight, we introduce SandMLE, a multi-agent framework that generates diverse, verifiable synthetic MLE environments from a small number of seed tasks, preserving the structural and technical complexity of real-world problems while constraining datasets to micro-scale (each task is paired with only 50-200 training samples). Through extensive experiments, we show that SandMLE reduces execution time by over 13 times, enabling large-scale, on-policy trajectory-wise RL for the first time in the MLE domain. On MLE-bench-lite, SandMLE yields significant gains over SFT baselines across Qwen3-8B, 14B, and 30B-A3B, with relative medal rate improvements ranging from 20.3% to 66.9%. Furthermore, the trained policy generalizes across unseen agentic scaffolds, achieving up to 32.4% better HumanRank score on MLE-Dojo.

*   **核心问题**：如何在不牺牲任务复杂性和验证严谨性的前提下，大幅降低 MLE 任务的反馈延迟，从而实现大规模的在线强化学习训练？

#### 2. 方法论
*   **核心思想**：提出 **SandMLE** 框架，利用多智能体协作自动生成“微缩版”但“结构完整”的合成 MLE 环境。通过将数据集规模限制在 **50–200 个样本**，将执行时间缩短至 15 秒以内（提升 13 倍以上），从而支持轨迹级（Trajectory-wise）的在线强化学习。
*   **关键技术细节**：

    *   **训练算法**：采用 **GRPO**（Group Relative Policy Optimization）进行轨迹级强化学习。
    *   **奖励函数设计**：采用**密集里程碑奖励**（Milestone-based Reward），包括格式奖励、执行奖励以及基于性能梯度的阶梯奖励（对应 Kaggle 的中位数、铜、银、金牌水平）。

#### 3. 实验设计
*   **数据集/场景**：从 MLE-bench 选取 60 个任务作为种子，生成了 848 个合成训练任务和 64 个验证任务。

    *   **MLE-bench-lite**：包含 22 个未见的易等级任务。

    *   **SFT-SandMLE**：先 SFT 后进行在线 RL。
*   **泛化性测试**：在不同的智能体脚手架（ReAct, AIRA, AIDE, MLE-Agent）上交叉验证。

*   **时间限制**：训练时单步执行限时 90 秒，评估时单步限时 4 小时，总任务限时 24 小时。

*   **消融实验**：专门对比了“密集奖励”与“稀疏奖励”的效果，证明了里程碑奖励对稳定训练的关键作用。
*   **客观性**：通过跨框架（Framework-agnostic）评估，证明了性能提升源于模型内在推理能力的增强，而非对特定脚手架的过拟合。实验设计较为严谨，包含了与闭源顶尖模型（如 Claude-4.5-Sonnet）的直观对比。

#### 6. 主要结论与发现
*   **性能飞跃**：SandMLE 显著提升了模型的 MLE 能力。在 8B 规模下，相对奖牌率提升了 66.9%；30B 模型在 SandMLE 训练后，Valid Submission 率达到 100%。
*   **RL 优于 SFT**：单纯的 SFT（行为克隆）提升有限，而通过 SandMLE 进行的在线 RL 能让模型学习到迭代纠错和长程规划能力。
*   **强大的泛化性**：训练后的模型在未见的智能体框架（如 AIDE）和未见的数据集（MLE-Dojo）上均表现出显著的性能增益。

#### 7. 优点
*   **效率突破**：巧妙地通过“数据微缩”解决了 MLE 领域在线 RL 的计算成本难题，具有极高的工程实用价值。
*   **全自动化**：构建了一套闭环的合成环境生成流水线，能够低成本地产生高质量、多样化的训练数据。
*   **框架无关**：提升的是模型底层的工程推理能力，而非特定工具的使用技巧。

#### 8. 不足与局限
*   **上下文窗口瓶颈**：实验发现当推理轮次超过 30 轮时，由于上下文溢出导致性能下降，这限制了极长任务的处理。
*   **微型数据的偏差**：虽然保留了数学逻辑，但 50-200 个样本的环境无法模拟真实世界中超大规模数据带来的工程挑战（如内存溢出、分布式计算优化等）。
*   **任务多样性受限**：目前主要基于 Kaggle 竞赛类任务，对于更开放、非结构化的机器学习科研任务（如新算法研发）的覆盖尚不明确。
