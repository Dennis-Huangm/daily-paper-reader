## 摘要
部署在智能体环境中的大语言模型（LLMs）必须在不同的任务实例中行使多种能力，其中“能力”是指在轨迹中执行一个或多个动作，这些动作对于成功解决环境中的子集任务是必要的。许多现有方法要么依赖于并非针对模型在目标环境中实际能力缺陷的合成训练数据，要么直接在目标环境上进行训练，导致模型需要跨任务隐式地学习这些能力。我们提出了 TRACE（将循环智能体失败转化为针对能力的训练环境），这是一个用于特定环境智能体自我改进的端到端系统。TRACE 通过对比成功和失败的轨迹来自动识别缺失的能力，并为每种能力合成一个针对性的训练环境，该环境会根据是否行使了该能力提供奖励。随后，TRACE 通过强化学习（RL）在每个合成环境上训练 LoRA 适配器，并在推理时路由至相关的适配器。实验结果表明，TRACE 在不同环境中具有良好的泛化性，在 $τ^2$-bench（客户服务）上比基础智能体提升了 14.1 分，在 ToolSandbox（工具使用）上增加了 7 个满分，分别比最强基线高出 7.4 分和 4 个满分。在给定相同数量的展开（rollouts）情况下，TRACE 的扩展效率优于基线，在 $τ^2$-bench 上分别比 GRPO 和 GEPA 高出 9.2 分和 7.4 分。

## Abstract
Large Language Models (LLMs) deployed in agentic environments must exercise multiple capabilities across different task instances, where a capability is performing one or more actions in a trajectory that are necessary for successfully solving a subset of tasks in the environment. Many existing approaches either rely on synthetic training data that is not targeted to the model's actual capability deficits in the target environment or train directly on the target environment, where the model needs to implicitly learn the capabilities across tasks. We introduce TRACE (Turning Recurrent Agent failures into Capability-targeted training Environments), an end-to-end system for environment-specific agent self-improvement. TRACE contrasts successful and failed trajectories to automatically identify lacking capabilities, synthesizes a targeted training environment for each that rewards whether the capability was exercised, and trains a LoRA adapter via RL on each synthetic environment, routing to the relevant adapter at inference. Empirically, TRACE generalizes across different environments, improving over the base agent by +14.1 points on $τ^2$-bench (customer service) and +7 perfect scores on ToolSandbox (tool use), outperforming the strongest baseline by +7.4 points and +4 perfect scores, respectively. Given the same number of rollouts, TRACE scales more efficiently than baselines, outperforming GRPO and GEPA by +9.2 and +7.4 points on $τ^2$-bench.

*   **研究动机**：
    *   **现有方法的局限**：直接在目标环境进行强化学习（RL）或监督微调（SFT）效率低下，因为奖励信号稀疏，且难以将失败归因于具体的能力缺失。
    *   **合成数据的盲目性**：现有的合成数据生成方法往往是通用的，并未针对模型在特定环境下的实际“短板”进行优化。
*   **整体含义**：TRACE 提出了一种“诊断-治疗”式的自我改进框架，通过分析失败案例来精准识别能力缺口，并为此定制训练环境。

### 2. 方法论
TRACE 的核心思想是将复杂的任务拆解为原子能力，并为每个能力构建专属的“微型训练场”。其流程包含四个关键步骤：
1.  **对比式能力识别（Contrastive Identification）**：
    *   让基础模型在目标环境中运行，收集成功和失败的轨迹。
    *   使用分析智能体对比两类轨迹，识别出能显著区分成功与失败的“候选能力”。
    *   计算**对比间隙（Contrastive Gap）**和**失败覆盖率（Failure Coverage）**，筛选出高影响力的能力缺口。
2.  **针对性环境合成（Environment Synthesis）**：
    *   为每个选定的能力生成一个程序化的合成环境（基于 `GameEnv` 协议）。
    *   该环境通过随机种子生成多样化场景，并设计专门的奖励函数（如多级奖励），确保模型必须行使该特定能力才能获得高分。
3.  **强化学习训练（RL Training）**：
    *   在合成环境中使用 **GRPO**（群组相对策略优化）算法训练 **LoRA 适配器**。
    *   每个能力对应一个独立的 LoRA 适配器，保持基础模型权重冻结。
4.  **能力组合与路由（Composition & Routing）**：
    *   在推理阶段，使用基础模型作为路由器，根据任务描述选择最匹配的 LoRA 适配器进行激活。

### 3. 实验设计
*   **数据集/场景**：
    *   **$\tau^2$-Bench**：模拟真实世界的客户服务场景（航空和零售领域），涉及复杂的工具交互和用户策略。
    *   **ToolSandBox**：评估智能体在有状态环境下的工具使用能力。
*   **对比方法**：
    *   **基础模型**：Qwen3-30B。
    *   **直接 RL**：在目标环境上直接运行 GRPO。
    *   **ADP (Agent Data Protocol)**：在多样化的公共智能体轨迹上进行 SFT。
    *   **AWM (Agent World Model)**：在通用的合成环境中进行 RL。
    *   **GEPA**：一种通过进化算法优化提示词（Prompt）的方法。

### 4. 资源与算力
*   **模型规模**：使用 Qwen3-30B-A3B-Instruct 作为基础智能体和模拟器。
*   **硬件资源**：使用了 **4 到 8 台 A100-80GB GPU**。
*   **训练细节**：
    *   每个能力训练最多 40 个迭代。
    *   使用了 AdamW 优化器，学习率为 $10^{-5}$。
    *   训练过程中开启了梯度检查点（Gradient Checkpointing）和分布式数据并行。

### 5. 实验数量与充分性
*   **实验规模**：
    *   在 $\tau^2$-Bench 上测试了 164 个实例，在 ToolSandBox 上测试了 129 个场景。
    *   进行了 10 次独立运行的对比分析，以验证能力识别的稳定性（结果显示高度一致）。
*   **充分性与公平性**：
    *   **消融实验**：对比了“路由”与“模型合并（Merging）”、“多能力混合训练”的效果，证明了路由策略的优越性。
    *   **扩展性分析**：展示了性能随能力数量和展开（Rollout）数量增加的增长曲线。
    *   **公平性**：所有基线方法在相同的总训练步数或 Rollout 预算下进行比较。

### 6. 主要结论与发现
*   **性能显著提升**：TRACE 在 $\tau^2$-Bench 上比基础模型提升了 14.1%，比最强基线高出 7.4%。
*   **扩展效率更高**：在相同的训练数据量下，TRACE 的性能提升斜率明显高于直接在目标环境训练的 GRPO。
*   **训练优于提示词优化**：实验证明，即使通过 GEPA 优化了提示词，其效果仍不如 TRACE 训练出的 LoRA 适配器，说明显式训练对能力获取至关重要。
*   **路由优于合并**：将多个能力适配器合并为一个模型会导致性能下降，而简单的推理时路由能更好地保持各能力的专业性。

### 7. 优点
*   **自动化闭环**：实现了从失败分析到环境生成再到训练的自动化，减少了人工干预。
*   **高数据效率**：通过隔离能力，解决了 RL 中信用分配（Credit Assignment）难的问题，使训练信号更密集。
*   **模块化**：使用 LoRA 适配器和路由机制，使得系统易于扩展新能力，且不会干扰基础模型原有功能。

### 8. 不足与局限
*   **推理延迟**：虽然 LoRA 增加的计算开销很小，但推理时的路由步骤（分类任务）会增加额外的首字延迟。
*   **能力重叠风险**：虽然系统试图隔离能力，但在复杂任务中，不同能力之间可能存在定义上的重叠或依赖，目前的路由机制仅选择单一适配器，可能无法处理需要多种能力深度融合的极端情况。
*   **对分析模型的依赖**：系统的有效性高度依赖于“分析智能体”识别能力的准确性以及“生成智能体”构建环境的质量。
