## 摘要
强化学习（RL）已成为训练工具调用智能体的主流范式，这通常需要在线交互环境。现有方法要么依赖于带有标准答案标注的训练数据，要么需要先进的商业语言模型（LM）来合成一旦创建就保持固定的环境。在这项工作中，我们提出了 TRUSTEE，这是一种无数据方法，通过由低至 8B 的免费开源语言模型完全模拟的动态环境来训练工具调用智能体，包括任务生成、用户模拟、工具模拟和轨迹评估，并配有一种在训练过程中动态控制任务难度各方面的自适应课程学习机制。我们的实验结果表明，TRUSTEE 在各个领域都带来了持续的改进，并优于所有需要额外外部资源进行训练的基准模型。这些结果证实，通过足够精巧的设计，即使是以本地 8B 语言模型为骨干的模拟环境，也能为工具学习建立强大的基准，而无需昂贵的标注数据、真实的人机交互、可执行工具，或来自人类专家或商业语言模型的昂贵可验证环境。我们希望所提出的范式能够启发未来在有限资源下进行环境扩展的研究。

## Abstract
Reinforcement learning (RL) has become a prevalent paradigm for training tool calling agents, which typically requires online interactive environments. Existing approaches either rely on training data with ground truth annotations or require advanced commercial language models (LMs) to synthesize environments that keep fixed once created. In this work, we propose TRUSTEE, a data-free method training tool calling agents with dynamic environments fully simulated by free open-source LMs that can be as small as 8B, including task generation, user simulation, tool simulation and trajectory evaluation, paired with an adaptive curriculum learning mechanism that controls various aspects of the task difficulty dynamically during training. Our empirical results show that TRUSTEE brings consistent improvements across various domains and outperforms all the baselines which require extra external resources for training. These confirm that, with a sufficiently sophisticated design, even simulated environments with a local 8B LM as the backbone could set a strong baseline for tool learning, without expensive annotated data, realistic human interactions, executable tools or costly verifiable environments from human experts or commercial LMs. We hope our proposed paradigm could inspire future research on environment scaling with limited resources.

### 1. 核心问题与研究动机
*   **核心问题**：如何降低工具学习（Tool Learning）中强化学习（RL）的成本和门槛？
*   **研究背景**：
    *   传统的工具学习依赖于昂贵的人工标注数据或真实的 API 环境（不稳定且难以扩展）。

*   **整体含义**：论文试图打破“工具学习需要高质量数据或强力商业模型”的迷思，提出一种“自给自足”的训练范式。

### 2. 方法论：TRUSTEE 框架
TRUSTEE 的核心思想是**“全流程模拟”**与**“自适应课程学习”**。

*   **全模拟环境（Fully Simulated Environment）**：
    *   **任务生成器（Task Generator）**：根据工具库和当前难度，生成用户画像、意图、初始查询和预期工具链。
    *   **用户模拟器（User Simulator）**：模拟真实用户在多轮对话中的反馈、追问或确认。
    *   **工具模拟器（Tool Simulator）**：模拟 API 的执行结果（无需真实执行代码），返回模拟的文本结果。
    *   **奖励验证器（Verifier Simulator）**：对整个交互轨迹进行评估，给出最终奖励。
*   **自适应课程学习（ACL）**：
    *   **动态难度控制**：通过 7 个维度调节难度：可用工具数量、预期调用次数、交互轮数、系统提示词的模糊度、用户专业程度（专家到新手）、查询歧义性、奖励准则的严苛程度。
    *   **难度演化公式**：根据上一批次（Batch）的平均奖励 $\bar{r}_n$ 动态调整难度等级 $D_n$。若奖励高则增加难度，反之降低，确保智能体始终在“最近发展区”学习。
*   **优化算法**：采用 **GRPO**（Group Relative Policy Optimization），利用组内相对优势进行策略优化，无需额外的价值模型（Value Model）。

### 3. 实验设计
*   **实验对象**：以 **Qwen3-8B** 作为智能体骨干模型，同时也作为所有模拟组件（用户、工具、验证器）的骨干模型。
*   **数据集/工具库**：使用 **ToolBench** 中的 REST API 作为工具池（仅使用其文本描述）。
*   **评估基准（Benchmarks）**：
    *   **BFCL V4**（Berkeley Function Calling Leaderboard）：评估单轮和多轮工具调用的准确性。
    *   **$\tau^2$-bench**：评估在航空、零售、电信三个真实场景下的多轮交互能力。
*   **对比方法**：包括原始模型、ToolRL、Tool-N1（依赖标注数据）、Simia-RL（依赖商业模型模拟）、EnvScaler 和 AWM（依赖程序合成）。

### 4. 资源与算力
*   **硬件环境**：使用了 1 个节点，配备 **8 张 NVIDIA H20 GPU**。
*   **算力分配**：4 张 GPU 用于智能体训练，另外 4 张 GPU 用于部署模拟环境的 vLLM 推理服务器。
*   **训练细节**：总计训练 200 个步骤（Steps），每个步骤的 Batch Size 为 16。

### 5. 实验数量与充分性
*   **实验规模**：在 BFCL 的多个子集（简单、多样、并行、多轮等）和 $\tau^2$-bench 的三个领域进行了全面测试。
*   **充分性**：实验对比了多种不同技术路线的基线模型（SFT、RLVR、合成环境等），证明了 TRUSTEE 在无外部资源情况下的优越性。
*   **客观性**：论文采用了标准化的第三方榜单进行评估，避免了自建评估标准的偏差。不过，论文提到这仍是“进行中的工作”，消融实验等细节在当前版本中尚待进一步补充。

### 6. 主要结论与发现
*   **8B 模型足够强大**：仅靠 8B 规模的开源模型进行自我模拟和训练，其效果可以超越依赖 GPT-4 或昂贵标注数据的现有方法。
*   **动态环境优于静态环境**：通过 ACL 机制动态调整环境难度，比预先合成的固定环境更能提升模型的泛化能力。
*   **全模拟可行性**：无需真实执行工具（API），仅靠语言模型的逻辑模拟即可训练出具备真实工具调用能力的智能体。

### 7. 优点
*   **极低成本**：完全摆脱了对商业 API 和人工标注的依赖，实现了“数据自由”。
*   **高度灵活**：ACL 机制允许训练过程根据模型表现自动进化，解决了训练初期难度过大或后期难度不足的问题。
*   **泛化性强**：在 BFCL 和 $\tau^2$-bench 上的表现均衡，没有出现某些基线模型在特定任务上过拟合的现象。

### 8. 不足与局限
*   **模拟器能力上限**：模拟环境的质量受限于 8B 模型本身的认知水平，可能无法生成极度复杂或专业性极强的任务场景。
*   **超参数敏感**：ACL 机制涉及多个维度的难度调节和阈值设置，调优过程可能较为复杂。
*   **稳定性风险**：由于完全依赖模型生成，若模拟器产生幻觉或格式错误，可能导致训练轨迹无效，影响训练稳定性。
