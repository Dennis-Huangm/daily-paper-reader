## 摘要
检索增强生成 (RAG) 通过外部视觉知识扩展了大视觉语言模型 (LVLMs)。然而，现有的视觉 RAG 系统通常依赖于通用的检索信号，忽略了复杂推理所必需的细粒度视觉语义。为了解决这一局限性，我们提出了 UniDoc-RL，这是一个统一的强化学习框架，其中 LVLM 智能体共同执行检索、重排序、主动视觉感知和推理。UniDoc-RL 将视觉信息获取建模为具有分层动作空间的序列决策问题。具体而言，它将视觉证据从粗粒度的文档检索逐步细化为细粒度的图像选择和主动区域裁剪，使模型能够抑制无关内容并关注信息密集区域。为了实现有效的端到端训练，我们引入了一种密集多奖励方案，为每个动作提供任务感知的监督。基于组相对策略优化 (GRPO)，UniDoc-RL 在不依赖独立价值网络的情况下，使智能体行为与多个目标保持一致。为了支持这种训练范式，我们策划了一个包含高质量推理轨迹和细粒度动作标注的综合数据集。在三个基准测试上的实验表明，UniDoc-RL 始终优于最先进的基准模型，相比之前的基于强化学习的方法提升高达 17.7%。

## Abstract
Retrieval-Augmented Generation (RAG) extends Large Vision-Language Models (LVLMs) with external visual knowledge. However, existing visual RAG systems typically rely on generic retrieval signals that overlook the fine-grained visual semantics essential for complex reasoning. To address this limitation, we propose UniDoc-RL, a unified reinforcement learning framework in which an LVLM agent jointly performs retrieval, reranking, active visual perception, and reasoning. UniDoc-RL formulates visual information acquisition as a sequential decision-making problem with a hierarchical action space. Specifically, it progressively refines visual evidence from coarse-grained document retrieval to fine-grained image selection and active region cropping, allowing the model to suppress irrelevant content and attend to information-dense regions. For effective end-to-end training, we introduce a dense multi-reward scheme that provides task-aware supervision for each action. Based on Group Relative Policy Optimization (GRPO), UniDoc-RL aligns agent behavior with multiple objectives without relying on a separate value network. To support this training paradigm, we curate a comprehensive dataset of high-quality reasoning trajectories with fine-grained action annotations. Experiments on three benchmarks demonstrate that UniDoc-RL consistently surpasses state-of-the-art baselines, yielding up to 17.7% gains over prior RL-based methods.

*   **研究动机**：
    *   **检索不精准**：粗粒度检索常引入无关背景，导致模型产生幻觉。
    *   **视觉利用率低**：模型通常被动接收全图，无法像人类一样聚焦于关键局部区域。
    *   **优化困难**：现有的强化学习（RL）方法多采用稀疏的最终结果奖励，难以对检索、裁剪等中间决策步骤进行有效的信用分配（Credit Assignment）。

### 2. 方法论
UniDoc-RL 将视觉 RAG 建模为一个**序列决策问题**，核心思想是通过分层动作空间和密集奖励机制实现从粗到细的信息获取。
*   **分层动作空间（Hierarchical Action Space）**：
    1.  **图像搜索（Search）**：调用外部工具进行粗粒度候选文档检索。
    2.  **精准选择（Select）**：利用 LVLM 智能体对候选图进行语义重排，过滤无关噪声。
    3.  **视觉感知（Perceive）**：通过 `<bbox>` 动作主动定位关键区域，执行裁剪（Crop）和缩放（Zoom），获取高分辨率局部细节。
*   **密集多奖励系统（Dense Multi-Rewards）**：
    *   **检索奖励 ($r_{ir}$)**：基于 NDCG 指标评估搜索结果的质量。
    *   **选择奖励 ($r_{sel}$)**：评估模型是否从候选集中选出了正确的金标（Golden）图像。
    *   **感知奖励 ($r_{crop}$)**：基于预测框与真实区域的交并比（IoU）给予反馈。
    *   **模式奖励 ($r_{pat}$)**：确保模型遵循预定义的 XML 标签格式。
    *   **结果奖励 ($r_{ans}$)**：由奖励模型（RM）评估最终答案的正确性。
*   **算法流程**：采用 **GRPO（组相对策略优化）** 算法，在不使用独立价值网络的情况下，通过组内样本的相对优势来对齐多目标行为。

### 3. 实验设计
*   **数据集/场景**：
    *   **SlideVQA**：多图幻灯片问答，侧重多跳推理。
    *   **ViDoSeek**：视觉文档搜索，侧重复杂推理。
    *   **MMLongBench**：长文档理解，侧重不同视觉类型（图表、表格等）的细粒度识别。
*   **对比基准（Baselines）**：
    *   **Vanilla RAG**：标准检索+推理。
    *   **ReAct**：传统的“思考-行动”迭代框架。
    *   **Search-R1(-VL)**：基于 R1 思路改进的搜索基准。
    *   **VRAG-RL**：现有的基于 RL 的视觉感知 RAG 方法（最直接的对比对象）。

*   **硬件资源**：实验使用了 **8 台 NVIDIA A100 80G GPU**。
*   **软件框架**：使用 `llama-factory` 进行监督微调（SFT），使用 `verl` 框架执行 GRPO 训练。
*   **训练细节**：SFT 阶段全参数微调，学习率 1e-5；RL 阶段学习率 1e-6，组大小（Group Size）为 5。

### 5. 实验数量与充分性
*   **实验规模**：在 3 个主流基准上进行了全面测试，涵盖了 3B 和 7B 两种参数规模的模型（基于 Qwen2.5-VL）。
*   **消融实验**：
    *   针对**动作空间**进行了消融，验证了“选择”和“感知”动作分别对推理和细节提取的贡献。
    *   针对**奖励设计**进行了增量实验，证明了密集奖励优于单一的结果奖励。
*   **充分性评价**：实验设计较为充分，不仅有定量指标，还提供了检索召回率分析、动作频率统计以及定性的案例研究（Case Study），客观地展示了 RL 如何改变模型的工具调用行为。

### 6. 主要结论与发现
*   **性能突破**：UniDoc-RL 在所有基准上均显著超越 SOTA。在 7B 模型上，相比之前的 RL 方法（VRAG-RL），整体性能提升了约 **17.7%**。
*   **主动感知**：RL 训练显著提高了模型调用裁剪工具的频率（例如在 MMLongBench 上从 SFT 的极低频率提升至与教师模型持平），使模型从“被动阅读”转变为“主动寻证”。
*   **降噪有效性**：引入“选择（Select）”动作后，检索召回率（Hit Rate）在各个数据集上均有 5%-10% 的提升，证明了 LVLM 作为重排器的价值。

### 7. 优点
*   **统一框架**：将检索、重排、感知和推理整合进单一的 RL 决策流，实现了端到端优化。
*   **密集监督**：通过分解任务目标并提供阶段性奖励，有效解决了长链条推理中的信用分配难题。
*   **高质量数据**：贡献并开源了一个包含 1.2 万条 SFT 轨迹和 5.5 千条 RL 轨迹的高质量数据集，具有较高的社区价值。

### 8. 不足与局限
*   **交互开销**：多轮“思考-动作-观察”的循环会增加推理延迟和计算成本。
*   **依赖外部工具**：第一步搜索仍依赖外部检索器，若初始检索完全未命中，后续的“选择”和“感知”动作将失去意义。
*   **泛化风险**：虽然在文档类任务表现优异，但在自然场景图像或非结构化视频 RAG 中的表现尚未验证。
