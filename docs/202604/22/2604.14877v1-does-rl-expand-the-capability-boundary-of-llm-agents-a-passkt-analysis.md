## 摘要
强化学习是真正扩展了 LLM 智能体的能力，还是仅仅提高了它们的可靠性？对于静态推理，最近的研究给出了后者的答案：基础模型与强化学习模型的 pass@k 曲线在大 k 值下趋于收敛。我们探究这一结论是否适用于智能体工具使用场景，因为在该场景下，T 轮交互能够实现重采样无法恢复的组合策略。我们引入了 PASS@(k,T)，这是一种同时改变采样预算 k 和交互深度 T 的二维指标，旨在将能力扩展与效率提升区分开来。我们的主要发现是，与静态推理的结果相反，工具使用中的强化学习真正扩大了能力边界：强化学习智能体的通过率曲线高于基础模型，且差距在大 k 值下不断扩大而非收敛。这种扩展专门针对组合式、序列化的信息收集任务；在更简单的任务上，强化学习的表现与先前研究的预测一致。在训练数据匹配的情况下，有监督微调在同样的组合任务上反而导致了边界退化，从而确定了自主探索是能力提升的因果因素。机制分析表明，强化学习重新分配了基础策略分布的权重，使其向那些下游推理更常产生正确答案的子集倾斜，改进主要集中在智能体如何整合检索到的信息。这些结果调和了对 LLM 强化学习的乐观与悲观解读：针对不同的任务类型，两者都是正确的。

## Abstract
Does reinforcement learning genuinely expand what LLM agents can do, or merely make them more reliable? For static reasoning, recent work answers the second: base and RL pass@k curves converge at large k. We ask whether this holds for agentic tool use, where T rounds of interaction enable compositional strategies that re-sampling cannot recover. We introduce PASS@(k,T), a two-dimensional metric that jointly varies sampling budget k and interaction depth T, separating capability expansion from efficiency improvement. Our main finding is that, contrary to the static-reasoning result, tool-use RL genuinely enlarges the capability boundary: the RL agent's pass-curve pulls above the base model's and the gap widens at large k rather than converging. The expansion is specific to compositional, sequential information gathering; on simpler tasks RL behaves as prior work predicts. Under matched training data, supervised fine-tuning regresses the boundary on the same compositional tasks, isolating self-directed exploration as the causal factor. Mechanism analysis shows RL reweights the base strategy distribution toward the subset whose downstream reasoning more often yields a correct answer, with the improvement concentrated on how the agent integrates retrieved information. These results reconcile optimistic and pessimistic readings of RL for LLMs: both are correct, on different task types.

#### 1. 核心问题与研究动机
*   **核心问题**：强化学习（RL）究竟是真正扩展了 LLM 智能体“能做什么”的边界（能力扩展），还是仅仅让它们在“已经能做”的事情上变得更可靠（效率提升）？
*   **研究背景**：在静态数学推理领域，近期研究（如 Yue et al., 2025）持悲观态度，认为 RL 只是重新分配了基础模型的概率质量，在大采样量 $k$ 下，RL 模型与基础模型的性能会趋于收敛。
*   **研究动机**：作者怀疑这种悲观结论在“智能体工具使用”场景下可能并不成立。因为智能体与环境的 $T$ 轮交互允许产生复杂的组合策略，这种策略是单纯通过重采样无法恢复的。

#### 2. 方法论：PASS@(k, T) 框架
*   **核心思想**：引入一个二维评估指标 **PASS@(k, T)**，将智能体的能力分解为两个维度：
    *   **$k$ 轴（采样预算）**：独立尝试的次数，用于衡量采样效率和可靠性。
    *   **$T$ 轴（交互深度）**：每轮尝试中与环境交互的最大回合数，用于衡量处理复杂组合任务的能力。
*   **关键定义**：
    *   **能力边界 (Capability Boundary)**：在无限采样（大 $k$）和固定深度 $T$ 下，模型能够解决的问题集合。
    *   **能力扩展 (Capability Expansion)**：RL 模型解决了一些基础模型在任何采样预算下都无法解决的问题。
    *   **效率提升 (Efficiency Improvement)**：RL 模型在基础模型本就能解决的问题上提高了成功概率（即 pass@1 提升，但上限未变）。
*   **算法流程**：使用 **GRPO**（群体相对策略优化）算法进行在线强化学习，通过验证性奖励（Exact Match）引导模型在工具使用轨迹中进行自我探索。

#### 3. 实验设计
*   **模型**：以 **Qwen2.5-7B-Instruct** 为基础模型。
*   **对比方法**：
    1.  **Base**：原始指令微调模型。
    2.  **SFT**：在 200 个专家轨迹上进行有监督微调。
    3.  **RL (GRPO)**：在相同的 200 个问题上进行强化学习训练。
    4.  **RL+explore**：加入探索奖励的 RL 变体。
*   **数据集与场景**：
    *   **Category A (MATH-500)**：纯数学推理（无工具），作为负控制组。
    *   **Category B (HotPotQA Comparison)**：独立检索任务（需检索两个独立实体进行比较）。
    *   **Category C (HotPotQA Bridge)**：序列/组合检索任务（第二个查询依赖于第一个查询的结果），这是测试能力扩展的核心场景。
*   **环境**：一个基于 BM25 的确定性检索工具。

#### 4. 资源与算力
*   **硬件**：使用单张 **NVIDIA RTX 6000 Ada (48GB)** GPU。
*   **训练时长**：
    *   **RL 训练**：10 个 Epoch（1000 个步长），耗时约 **55 小时**。
    *   **SFT 训练**：约 12 分钟。
*   **推理成本**：每个模型/类别组合的评估约需 30-60 GPU 分钟。

#### 5. 实验数量与充分性
*   **实验规模**：针对 3 个类别、4 种模型、多个 $k$ 值（1 到 64）和 $T$ 值（0 到 5）进行了全网格评估。
*   **充分性**：
    *   使用了 **匹配数据 (Matched-data)** 设计，确保 SFT 和 RL 看到的是完全相同的 200 个问题，排除了数据暴露的影响。
    *   进行了**机制分析**（困惑度分解、策略多样性测量、跨策略交换实验），深入探讨了性能提升的来源。
    *   通过 **Bootstrap 置信区间**和**采样充足性检查**验证了结果的统计显著性。
*   **公平性**：所有模型使用相同的 Prompt 模板、温度系数（0.7）和评估环境。

#### 6. 主要结论与发现
*   **RL 确实扩展了能力边界**：在复杂的组合检索任务（Category C）中，RL 模型的 pass 曲线在大 $k$ 时显著高于基础模型，且差距随 $k$ 增大而扩大，这与静态推理的“收敛”结论完全相反。
*   **SFT 导致能力退化**：在相同数据下，SFT 在组合任务上反而丢失了基础模型原本具备的部分能力，证明了**自主探索**（而非数据暴露）是 RL 实现能力扩展的关键。
*   **任务依赖性**：在简单任务（Cat A 和 B）上，RL 的表现符合预期（无影响或仅有微小提升），说明 RL 的能力扩展作用特异于需要复杂组合策略的场景。
*   **机制揭示**：RL 并没有发明全新的搜索查询，而是通过**重新分配权重**，优化了智能体“如何整合已检索信息”的推理过程。

#### 7. 优点
*   **评估维度创新**：PASS@(k, T) 指标为智能体评估提供了更科学的工具，能够清晰区分“变稳了”和“变强了”。
*   **因果分析严谨**：通过匹配数据的 SFT vs RL 对比，有力地反驳了“RL 只是在做数据拟合”的观点。
*   **调和矛盾**：成功解释了为什么在数学推理中 RL 显得“平庸”，而在智能体任务中却表现出“进化”特征。

#### 8. 不足与局限
*   **模型规模有限**：主要实验基于 7B 模型，尚未在大规模模型（如 70B+）上验证。
*   **任务多样性不足**：仅使用了检索工具和 HotPotQA 数据集，未涉及更复杂的 API 调用或多工具协作场景。
*   **训练规模较小**：训练集仅包含 200 个问题，虽然足以证明原理，但可能限制了模型发现更深层次策略的潜力。
*   **检索环境简化**：使用了确定性的 BM25 检索而非更具挑战性的开放域网络搜索。
