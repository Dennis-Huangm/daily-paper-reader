## 摘要
多步智能体强化学习受益于细粒度的信用分配，然而现有方法的选择有限：像 GRPO 这样的无评论家（critic-free）方法为轨迹中的每个动作分配统一的优势值，而学习到的价值网络则会引入显著的开销，且在稀疏奖励下表现脆弱。我们观察到，针对同一问题的组展开（group rollouts）通常会经过重叠的中间状态，隐式地形成一棵分支在连续决策点发散的树。基于这一洞察，我们提出了展开树蒙特卡洛（Rollout-Tree Monte Carlo, RTMC）优势估计，它通过聚合共享相同状态的展开轨迹中的回报统计数据，来生成每步的 Q 值和优势值，且无需任何学习到的评论家网络。一种状态-动作签名系统将原始交互历史压缩为紧凑且可比较的表示，使得跨展开的状态匹配变得可行。在 SWE-bench Verified 基准测试中，RTMC 相比 GRPO 将 pass@1 提高了 3.2 个百分点。

## Abstract
Multi-step agentic reinforcement learning benefits from fine-grained credit assignment, yet existing approaches offer limited options: critic-free methods like GRPO assign a uniform advantage to every action in a trajectory, while learned value networks introduce notable overhead and can be fragile under sparse rewards. We observe that group rollouts targeting the same problem often traverse overlapping intermediate states, implicitly forming a tree whose branches diverge at successive decision points. Building on this insight, we introduce Rollout-Tree Monte Carlo (RTMC) advantage estimation, which aggregates return statistics across rollouts sharing a common state to produce per-step Q-values and advantages--without any learned critic. A state-action signature system compresses raw interaction histories into compact, comparable representations, making cross-rollout state matching tractable. On SWE-bench Verified, RTMC improves pass@1 by 3.2 percentage points over GRPO.

### 2. 方法论：RTMC 核心思想与技术细节
RTMC 的核心思想是利用针对同一问题的多个并行展开（Group Rollouts）天然形成的“树状结构”来估计优势。

*   **展开树（Rollout Trees）**：当 $N$ 个 Rollout 尝试解决同一个问题时，它们往往会经过相似的中间状态。这些轨迹在状态重合处汇聚，在决策点处发散，隐式构成了一棵树。
*   **状态-动作签名系统（Signatures）**：
    *   由于原始文本历史极难完全匹配，作者设计了一套抽象签名系统。
    *   **动作签名**：根据工具调用的效果（如查看、修改、测试）而非具体名称进行分类，并对修改内容进行哈希处理。
    *   **状态签名**：记录已操作的文件集合、操作类型（View/Modify）及结果，采用“视图分桶（View Bucketing）”和“内容哈希”来压缩状态。
*   **优势估计公式**：
    *   **Q值估计**：$\hat{Q}(s, a)$ 是所有经过状态 $s$ 并采取动作 $a$ 的轨迹的平均折扣回报。
    *   **V值估计**：$\hat{V}(s)$ 是所有经过状态 $s$ 的轨迹的平均回报。
    *   **优势值**：$\hat{A}(s, a) = \hat{Q}(s, a) - \hat{V}(s)$。
*   **关键修正技术**：
    *   **先验平滑（Prior-based Smoothing）**：针对仅被访问一次的状态，引入全局成功率作为先验进行贝叶斯收缩，避免优势值为零。
    *   **任务特定奖励塑形**：为验证性动作（如运行测试）提供微小奖励，防止折扣因子 $\gamma$ 导致智能体因“多做多错”而变得消极。

*   **学习曲线**：RTMC 的学习速度和最终收敛水平显著优于轨迹级奖励方法。

### 7. 优点与亮点
*   **Critic-Free（无评论家）**：不需要额外训练价值网络，极大地节省了显存和计算资源，且避免了 Critic 训练不稳定的问题。
*   **即插即用**：可以轻松集成到现有的 GRPO 或 PPO 框架中，只需在数据处理阶段增加签名匹配逻辑。
*   **可解释性**：通过展开树的可视化，可以直观地看到智能体在不同决策分支上的优势分布。

### 8. 不足与局限
*   **签名设计的领域依赖性**：目前的状态-动作签名是针对软件工程任务手动设计的（Hand-crafted），迁移到其他领域（如通用对话或数学证明）需要重新设计签名规则。
*   **状态隔离问题**：由于搜索空间巨大，很多状态在 8 个 Rollout 中依然只出现一次，导致真正的“跨轨迹对比”比例受限。
*   **性能天花板**：虽然提升显著，但受限于手动规则的覆盖范围，未来可能需要引入学习到的嵌入（Learned Embeddings）来替代手动签名。
