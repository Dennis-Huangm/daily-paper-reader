## 摘要
GUI 智能体通过视觉界面而非编程 API 驱动应用程序，通过点击、滑动和按键与任意软件交互，触及了基于命令行界面（CLI）的智能体无法覆盖的长尾应用。然而，该领域的进展瓶颈与其说是建模能力，不如说是缺乏连贯的全栈基础设施：在线强化学习（RL）训练受困于环境不稳定和封闭的流水线，评估协议在不同研究之间悄然发生偏移，且训练好的智能体很少能触及真实设备上的真实用户。我们提出了 ClawGUI，这是一个在单一框架内解决上述三个差距的开源框架。ClawGUI-RL 提供了首个开源的 GUI 智能体强化学习基础设施，经验证支持并行虚拟环境和真实物理设备，并将 GiGPO 与过程奖励模型（Process Reward Model）集成，以实现密集步级监督。ClawGUI-Eval 在 6 个基准测试和 11 个以上模型中强制执行完全标准化的评估流水线，在官方基准测试中实现了 95.8% 的复现率。ClawGUI-Agent 通过 12 个以上的聊天平台将训练好的智能体引入 Android、HarmonyOS 和 iOS，具备混合 CLI-GUI 控制和持久化个性化记忆。在该流水线中端到端训练的 ClawGUI-2B 在 MobileWorld GUI-Only 上实现了 17.1% 的成功率，比同规模的 MAI-UI-2B 基准模型高出 6.0%。

## Abstract
GUI agents drive applications through their visual interfaces instead of programmatic APIs, interacting with arbitrary software via taps, swipes, and keystrokes, reaching a long tail of applications that CLI-based agents cannot. Yet progress in this area is bottlenecked less by modeling capacity than by the absence of a coherent full-stack infrastructure: online RL training suffers from environment instability and closed pipelines, evaluation protocols drift silently across works, and trained agents rarely reach real users on real devices. We present \textbf{ClawGUI}, an open-source framework addressing these three gaps within a single harness. \textbf{ClawGUI-RL} provides the first open-source GUI agent RL infrastructure with validated support for both parallel virtual environments and real physical devices, integrating GiGPO with a Process Reward Model for dense step-level supervision. \textbf{ClawGUI-Eval} enforces a fully standardized evaluation pipeline across 6 benchmarks and 11+ models, achieving 95.8\% reproduction against official baselines. \textbf{ClawGUI-Agent} brings trained agents to Android, HarmonyOS, and iOS through 12+ chat platforms with hybrid CLI-GUI control and persistent personalized memory. Trained end to end within this pipeline, \textbf{ClawGUI-2B} achieves 17.1\% Success Rate on MobileWorld GUI-Only, outperforming the same-scale MAI-UI-2B baseline by 6.0\%.

## 1. 核心问题与整体含义（研究动机和背景）
GUI 智能体（GUI Agents）通过视觉界面操作软件，具有比传统 CLI 智能体更强的通用性。然而，该领域目前面临三大工程瓶颈：
*   **训练生态封闭且不稳定**：在线强化学习（RL）依赖的模拟器易崩溃，且缺乏支持物理设备的开源训练基础设施。
*   **评估标准不统一**：不同论文在提示词格式、分辨率、坐标归一化等设置上的差异，导致实验结果难以直接比较，存在“静默偏移”。
*   **部署链路断裂**：研究模型难以转化为真实应用，缺乏跨平台（Android/iOS/HarmonyOS）及个性化记忆的部署方案。

**ClawGUI** 的提出旨在通过一个开源的全栈框架，打通“训练-评估-部署”的全链路，为 GUI 智能体研究提供标准化的基础设施。

## 2. 方法论：核心思想与关键技术
ClawGUI 由三个核心模块组成：

### ClawGUI-RL（在线强化学习基础设施）
*   **环境管理**：支持 Docker 化的 Android 模拟器并行运行，具备健康检查、崩溃恢复和备用服务器轮换机制；同时支持物理设备的接入。
*   **奖励设计**：结合了**二元结果奖励**（任务是否成功）和基于 **PRM（过程奖励模型）** 的密集步级奖励。
*   **GiGPO 算法**：采用分层优势估计。在剧集（Episode）级别保留宏观相对优势，在步骤（Step）级别通过“锚点状态分组”机制，对经历相同中间状态的步骤进行归一化奖励分配，解决了 GUI 任务中奖励稀疏和信用分配难的问题。

### ClawGUI-Eval（标准化评估流水线）
*   **三阶段流程**：将评估解耦为 **推理（Infer）**、**判定（Judge）** 和 **指标计算（Metric）**。
*   **配置锁定**：为每个模型固定提示词、分辨率和采样参数，确保结果的可复现性。

### ClawGUI-Agent（生产级部署系统）
*   **混合控制**：结合 CLI 的高效精确与 GUI 的通用覆盖。
*   **持久化记忆**：利用向量数据库存储用户习惯和事实，通过语义检索实现个性化适配。
*   **多端接入**：支持通过 12+ 聊天平台（如 Telegram、飞书）远程或本地控制设备。

*   **实验场景**：涵盖了从简单的元素定位到跨 App 的复杂多步操作。

*   **训练配置**：

## 5. 实验数量与充分性
*   **实验规模**：

    *   **消融实验**：对比了 GRPO（剧集级奖励）与 GiGPO（步级密集奖励）的效果。
*   **充分性评价**：实验设计较为全面，不仅验证了模型性能的提升，还重点验证了框架的**复现能力**（达到了 95.8% 的复现率），这在同类论文中较为罕见，体现了极高的客观性和工程严谨性。

## 6. 主要结论与发现
*   **基础设施优于模型规模**：仅 2B 规模的 ClawGUI-2B 在经过在线 RL 训练后，成功率（17.1%）超过了未经针对性 RL 训练的 32B 甚至 72B 模型。
*   **密集奖励至关重要**：GiGPO 相比 GRPO 带来了 17.9% 的相对性能提升，证明了步级监督在长程 GUI 任务中的必要性。
*   **评估差异源于工程细节**：通过标准化流水线，可以消除绝大部分因提示词或分辨率导致的评估偏差。

## 7. 优点
*   **全栈性**：首个将训练、评估、部署整合在一起的开源 GUI Agent 框架。
*   **工程鲁棒性**：针对模拟器不稳定问题设计了自动轮换机制，并支持物理设备训练。
*   **复现贡献**：公开了大量模型的推理预测结果，极大方便了后续研究者的对比。
*   **实用性**：ClawGUI-Agent 支持多平台和个性化记忆，具备实际商用潜力。

## 8. 不足与局限
*   **算力门槛**：虽然模型较小，但在线 RL 训练仍需 8 张 A6000 及大量并行环境，对个人开发者仍有挑战。
*   **物理设备奖励获取**：在物理设备上训练时，由于缺乏系统根权限，高度依赖 MLLM 作为裁判，可能存在判定偏差。
*   **任务多样性**：目前的 RL 训练主要集中在 MobileWorld 任务，对于更复杂的跨设备联动或极长程任务（如超过 50 步）的覆盖仍有待加强。
