## 摘要
现有的合成工具使用语料库主要针对离线监督微调（SFT）设计，而强化学习（RL）则需要支持奖励可检查在线采样（online rollouts）的可执行环境。我们提出了 COVERT，这是一个两阶段流水线：首先通过具有多级验证的自进化合成生成可靠的基础工具使用轨迹，然后应用保持真值（oracle-preserving）的增强手段，系统性地增加环境复杂度。这些增强引入了干扰工具、间接或模糊的用户查询，以及噪声、多格式或错误的工具输出，同时严格保留真值工具调用和最终答案作为地面真值（ground truth）。这种设计使得标准案例可以通过参考匹配进行自动奖励计算，而对于错误检测等特殊行为，则可以通过轻量级的评判员辅助验证，从而支持工具调用策略的强化学习优化。在 Qwen2.5-Instruct-14B 上，COVERT-RL 将 BFCL v3 的总准确率从 56.5 提升至 59.9，将 ACEBench 的准确率从 53.0 提升至 59.3，且在通用能力基准测试上的性能退化极小；当与 SFT 叠加时，准确率进一步达到 62.1 和 61.8，证实了增量收益。这些结果表明，保持真值的合成环境提供了一个实用的强化学习精调阶段，与监督微调互补，能够提高在模糊性和不可靠工具反馈下的工具使用鲁棒性。

## Abstract
Existing synthetic tool-use corpora are primarily designed for offline supervised fine-tuning, yet reinforcement learning (RL) requires executable environments that support reward-checkable online rollouts. We propose COVERT, a two-stage pipeline that first generates reliable base tool-use trajectories through self-evolving synthesis with multi-level validation, and then applies oracle-preserving augmentations that systematically increase environmental complexity. These augmentations introduce distractor tools, indirect or ambiguous user queries, and noisy, multi-format, or erroneous tool outputs, while strictly preserving oracle tool calls and final answers as ground truth. This design enables automatic reward computation via reference matching for standard cases and lightweight judge-assisted verification for special behaviors such as error detection, supporting RL optimization of tool-calling policies. On Qwen2.5-Instruct-14B, COVERT-RL improves overall accuracy on BFCL v3 from 56.5 to 59.9 and on ACEBench from 53.0 to 59.3, with minimal regressions on general-ability benchmarks; when stacked on SFT, it further reaches 62.1 and 61.8, confirming additive gains. These results suggest that oracle-preserving synthetic environments offer a practical RL refinement stage, complementary to SFT, for improving tool-use robustness under ambiguity and unreliable tool feedback.

*   **核心问题**：现有的工具使用（Tool-use）合成数据集主要为离线监督微调（SFT）设计，表现为“静态轨迹”。然而，强化学习（RL）需要一个“动态且可执行”的环境，能够支持在线采样并提供可验证的奖励反馈。
*   **研究动机**：
    *   现实世界的工具调用日志稀缺且涉及隐私。
    *   现有模型在面对模糊查询、干扰工具、噪声输出或工具故障时鲁棒性不足。
    *   缺乏能够在大规模合成数据上实现自动、准确奖励计算的机制。

#### 2. 方法论：COVERT 框架
COVERT 采用两阶段流水线构建 RL 就绪的合成环境：
*   **第一阶段：可靠基础轨迹生成**
    *   **自进化合成**：从少量种子数据开始，利用 LLM 迭代生成新轨迹并回填种子池。
    *   **多级验证**：通过格式检查（JSON 结构）、规则检查（参数类型/枚举约束）和语义检查（LLM 评判员评估逻辑一致性）确保基础数据的绝对可靠。
*   **第二阶段：保持真值（Oracle-Preserving）的环境增强**
    *   **核心思想**：在修改环境组件（查询、工具集、输出）的同时，严格保留“真值工具调用”或“最终答案”，从而使奖励计算依然可行。
    *   **增强类型**：
        *   **精确可验证类**：引入干扰工具、重写模糊查询、多格式/噪声输出。通过参考匹配（Reference Matching）计算奖励。
        *   **评判员辅助类**：引入工具执行错误、无关查询。使用轻量级 LLM 评判员根据预设的“真值标签”判断模型是否正确识别了错误或拒绝了请求。
*   **强化学习算法**：采用 **GRPO**（Group Relative Policy Optimization），无需价值模型，直接利用上述可验证奖励进行策略优化。

*   **模型基础**：Qwen2.5-Instruct-7B 和 14B。
*   **对比方法**：

*   **基准测试（Benchmark）**：

*   **评估协议**：使用“极简系统提示词”，不提供额外的行为指导，以严格测试模型的原生能力。

#### 6. 主要结论与发现
*   **性能显著提升**：在 14B 模型上，COVERT-RL 将 BFCL v3 准确率提升了 3.4%，ACEBench 提升了 6.3%。

#### 7. 优点
*   **可验证性强**：通过“保持真值”的设计，解决了工具调用 RL 中奖励函数难设计的痛点。
*   **可控性高**：可以针对性地通过“旋钮”（Knobs）调整干扰项的难度和类型。
*   **实用性**：证明了合成环境可以作为 SFT 之后的一个标准 RL 精炼阶段，且不损害通用性。

*   **长程规划限制**：目前的模拟环境主要针对短到中程交互，对于需要极长步数、复杂状态依赖的 Agent 任务支持有限（如 ACEBench Agent 分类提升不明显）。

*   **环境真实度**：模拟环境中的工具输出（如错误信息）是预设的，可能无法完全覆盖真实 API 返回的所有极端异常情况。
