## 摘要
强化学习（RL）已被广泛用于训练大语言模型（LLM）智能体以处理多轮交互任务，但其样本效率受到稀疏奖励和长时程的严重限制。同策略自蒸馏（OPSD）通过提供来自可访问标准答案的特权教师的密集 Token 级监督来缓解这一问题。然而，这种固定的特权信息无法捕捉智能体任务中多样化的有效策略，且简单地将 OPSD 与 RL 结合往往会导致训练崩溃。为了解决这些局限性，我们提出了 Skill-SD，这是一个将智能体自身的轨迹转化为动态的、仅限训练的监督信息的框架。已完成的轨迹被总结为简洁的自然语言技能，描述了成功的行为、错误和工作流。这些技能作为动态特权信息仅对教师进行条件约束，而学生始终在普通任务提示下行动，并通过蒸馏学习将指导内化。为了稳定训练，我们推导了一种重要性加权的反向 KL 损失，以提供梯度正确的 Token 级蒸馏，并动态地使教师与不断进步的学生同步。在智能体基准测试上的实验结果表明，Skill-SD 显著优于标准的 RL 基准，在 AppWorld 和 Sokoban 任务上分别比原生 GRPO 提升了 14.0% 和 10.9%，比原生 OPD 提升了 42.1% 和 40.6%。

## Abstract
Reinforcement learning (RL) has been widely used to train LLM agents for multi-turn interactive tasks, but its sample efficiency is severely limited by sparse rewards and long horizons. On-policy self-distillation (OPSD) alleviates this by providing dense token-level supervision from a privileged teacher that has access to ground-truth answers. However, such fixed privileged information cannot capture the diverse valid strategies in agent tasks, and naively combining OPSD with RL often leads to training collapse. To address these limitations, we introduce Skill-SD, a framework that turns the agent's own trajectories into dynamic training-only supervision. Completed trajectories are summarized into compact natural language skills that describe successful behaviors, mistakes, and workflows. These skills serve as dynamic privileged information conditioning only the teacher, while the student always acts under the plain task prompt and learns to internalize the guidance through distillation. To stabilize the training, we derive an importance-weighted reverse-KL loss to provide gradient-correct token-level distillation, and dynamically synchronize the teacher with the improving student. Experimental results on agentic benchmarks demonstrate that Skill-SD substantially outperforms the standard RL baseline, improving both vanilla GRPO (+14.0%/+10.9% on AppWorld/Sokoban) and vanilla OPD (+42.1%/+40.6%). #### 1. 核心问题与研究动机
*   **核心问题**：在多轮交互任务中，强化学习（RL）训练大语言模型（LLM）智能体面临**奖励稀疏**和**时程过长**（Long Horizon）的挑战。
*   **研究背景**：
    *   现有的同策略自蒸馏（OPSD）方法通常依赖于固定的“特权信息”（如数学题的标准答案），但在智能体任务中，解决路径往往是多样化的，固定答案会限制探索。
    *   直接将自蒸馏与 RL 结合在智能体场景下极易导致训练崩溃（Training Collapse）。
    *   智能体在推理时无法获取未来的“经验”或“反思”，如何在不增加推理负担的前提下让模型内化这些经验是一个难题。

#### 2. 方法论
*   **核心思想**：将智能体自身的历史轨迹总结为动态的**自然语言技能**（Skills），仅将其作为**教师模型**的特权输入，引导**学生模型**（使用普通提示词）通过 Token 级的蒸馏来内化这些策略。
*   **关键技术细节**：
    *   **技能生成与存储**：将完成的轨迹总结为包含“成功分析”、“错误分析”和“黄金工作流”的 JSON 结构，并存储在基于 UCB（置信区间上界）算法索引的缓冲区中，实现动态筛选。
    *   **技能条件教师（Skill-Conditioned Teacher）**：教师和学生共享参数，但教师额外输入检索到的技能。
    *   **重要性加权反向 KL 损失（Importance-Weighted Reverse-KL Loss）**：为了解决教师和学生因提示词不同导致的分布偏移，推导了带重要性采样权重的损失函数，确保 Token 级梯度的无偏性。
    *   **动态教师同步**：定期将教师模型更新为最新的学生模型参数，防止教师信号过时导致训练停滞。
    *   **联合优化**：总损失函数由 GRPO（群组相对策略优化）损失和自蒸馏（SDL）损失加权组成。

    *   **Vanilla GRPO**：标准强化学习，无技能引导。
    *   **Skill-Augmented GRPO**：直接在学生提示词中加入技能（测试技能注入的效果）。
    *   **Vanilla OPD**：仅使用同策略蒸馏，无环境奖励。

#### 6. 主要结论与发现
*   **性能提升**：Skill-SD 在 AppWorld 上比原生 GRPO 提升了 **14.0%** 的准确率，在 Sokoban 上提升了 **10.9%**。
*   **技能引导的艺术**：技能应该引导“教师”而非“学生”。直接在学生提示词中加入技能虽然能提高训练分数，但在验证集上表现极差（过拟合）。
*   **稳定性关键**：动态同步教师模型和使用学生侧采样（On-policy）是防止训练崩溃的核心。异策略（教师采样）在复杂任务（如推箱子）中会迅速导致模型崩溃。
*   **内化效果**：通过 SDL 损失，学生模型在不看技能的情况下，其 Token 分布逐渐趋近于拥有技能的教师模型。

#### 7. 优点
*   **动态性**：解决了传统自蒸馏依赖固定答案的局限，使特权信息随智能体水平提升而进化。
*   **推理高效**：训练完成后，学生模型无需检索技能或增加提示词长度，保持了原始的推理速度。
*   **数学严谨**：针对跨提示词的蒸馏场景，推导了重要性加权的梯度修正公式，具有较强的理论支撑。

#### 8. 不足与局限
*   **检索机制简单**：目前使用 UCB 算法进行任务内检索，对于跨任务的技能迁移（Semantic Retrieval）支持有限。
*   **样本依赖**：虽然在小样本任务（90-100个任务）上表现良好，但在更大规模、更泛化的任务池中的表现尚待验证。
*   **计算开销**：尽管是异步的，但依赖外部 LLM 进行轨迹总结仍增加了训练过程中的 API 成本和系统复杂性。
