## 摘要
交通信号控制 (TSC) 是智能交通系统 (ITS) 的核心组成部分，旨在减少拥堵、排放和旅行时间。近期基于强化学习 (RL) 和大语言模型 (LLM) 的方法提高了适应性，但仍面临可解释性有限、交互数据不足以及对异构交叉口泛化能力弱的问题。本文提出了 CuraLight，这是一个以 LLM 为中心的框架，其中 RL 智能体辅助基于 LLM 的交通信号控制器的微调。RL 智能体探索交通环境并生成高质量的交互轨迹，这些轨迹被转换为提示-响应对，用于模仿微调。多 LLM 集成审议系统通过结构化辩论进一步评估候选信号配时动作，为训练提供偏好感知的监督信号。在 SUMO 中针对济南、杭州和亦庄的异构真实路网进行的实验表明，CuraLight 始终优于最先进的基准模型，平均旅行时间减少了 5.34%，平均排队长度减少了 5.14%，平均等待时间减少了 7.02%。结果突显了将 RL 辅助探索与基于审议的数据策展相结合，对于实现可扩展且可解释的交通信号控制的有效性。

## Abstract
Traffic signal control (TSC) is a core component of intelligent transportation systems (ITS), aiming to reduce congestion, emissions, and travel time. Recent approaches based on reinforcement learning (RL) and large language models (LLMs) have improved adaptivity, but still suffer from limited interpretability, insufficient interaction data, and weak generalization to heterogeneous intersections.   This paper proposes CuraLight, an LLM-centered framework where an RL agent assists the fine-tuning of an LLM-based traffic signal controller. The RL agent explores traffic environments and generates high-quality interaction trajectories, which are converted into prompt-response pairs for imitation fine-tuning. A multi-LLM ensemble deliberation system further evaluates candidate signal timing actions through structured debate, providing preference-aware supervision signals for training.   Experiments conducted in SUMO across heterogeneous real-world networks from Jinan, Hangzhou, and Yizhuang demonstrate that CuraLight consistently outperforms state-of-the-art baselines, reducing average travel time by 5.34 percent, average queue length by 5.14 percent, and average waiting time by 7.02 percent. The results highlight the effectiveness of combining RL-assisted exploration with deliberation-based data curation for scalable and interpretable traffic signal control.

### 1. 核心问题与研究动机
*   **核心问题**：如何在保证交通信号控制（TSC）决策**可解释性**的同时，解决大语言模型（LLM）在处理异构路口高维配时方案时**任务对齐数据不足**、**泛化性弱**以及**决策脆弱**的问题。
*   **研究背景**：
    *   **传统方法与强化学习（RL）**：虽然具有较强的适应性，但决策过程类似“黑盒”，缺乏可解释性。
    *   **现有 LLM 方法**：通常依赖少量环境交互，在面对复杂的异构路口（车道配置不同、相位需求多样）时，难以生成精确的绿灯时长，且缺乏特定场景的运行知识。

### 2. 方法论
CuraLight 提出了一个以 LLM 为核心、由 RL 智能体辅助的训练与决策框架，核心流程如下：
*   **扩散模型辅助的 RL 探索（Diffusion-based RL Assistant）**：
    *   利用基于扩散策略的 RL 智能体在异构网络中进行探索，生成高质量的相位和时长轨迹。
    *   使用“压力平衡选择器（Pressure-Based Selector）”确定相位，扩散模型生成时长建议，并由 Critic 网络进行评估。
*   **数据策展与模仿微调（Data Curation & Imitation Fine-tuning）**：
    *   将 RL 探索的轨迹转化为“结构化提示词-推理决策”对。
    *   采用 **LoRA（低秩自适应）** 技术对基础模型（Gemma-3-12B-it）进行模仿微调。
*   **多 LLM 集成辩论系统（Multi-LLM Ensemble Deliberation）**：
    *   **RL 预过滤**：剔除明显劣质的动作。
    *   **辩论机制**：多个“辩护 LLM”为不同的候选动作提供理由，一个“共识 LLM”汇总论点并给出偏好评分。
*   **优先级感知微调（Priority-Aware Fine-tuning）**：
    *   结合 RL Critic 的评分和 LLM 共识评分，计算统一优先级。
    *   在第二阶段微调中，利用优先级分数对样本进行加权，强化高质量决策，抑制低质量决策。

### 7. 优点与亮点
*   **RL 与 LLM 的深度融合**：不只是简单的提示词工程，而是利用 RL 的探索能力为 LLM 提供高质量的“教学数据”，解决了 LLM 在特定工业任务中数据匮乏的问题。
*   **辩论引导的决策优化**：引入多智能体辩论机制，模拟了人类专家在复杂决策时的审议过程，增强了决策的逻辑性和一致性。
*   **处理异构性的能力**：通过统一的建模方式，成功解决了现实中路口几何形状和相位配置各异的难题。

*   **实时性挑战**：多 LLM 辩论和集成审议虽然提升了质量，但会带来额外的计算开销和推理延迟，在超大规模实时交通控制中可能存在瓶颈。
*   **目标单一性**：目前主要关注通行效率（时间、排队），尚未深入探讨碳排放、多模态交通（如公交优先）等更广泛的优化目标。
*   **依赖高质量 RL 辅助**：如果基础 RL 智能体探索效率极低，可能会限制后续 LLM 微调的效果上限。
