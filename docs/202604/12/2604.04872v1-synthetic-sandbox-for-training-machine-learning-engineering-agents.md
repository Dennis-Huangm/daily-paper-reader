## 摘要
随着大语言模型智能体从软件工程（SWE）任务向机器学习工程（MLE）任务迈进，验证智能体行为的成本呈数量级增长：虽然 SWE 任务可以通过快速执行的单元测试进行验证，但 MLE 验证需要在每个展开（rollout）步骤中在大型数据集上运行完整的机器学习流水线——包括数据预处理、模型训练和指标评估——这使得基于轨迹的在线强化学习（on-policy RL）变得极其缓慢。现有方法退而求其次，采用监督微调（SFT）或离线代理奖励，牺牲了在线强化学习在探索和泛化方面的优势。我们观察到，沙盒数据规模是这一瓶颈的主要来源。基于这一洞察，我们提出了 SandMLE，这是一个多智能体框架，能够从少量种子任务中生成多样化且可验证的合成 MLE 环境，在保留现实世界问题的结构和技术复杂性的同时，将数据集限制在微观规模（每个任务仅配有 50-200 个训练样本）。通过广泛的实验，我们证明 SandMLE 将执行时间缩短了 13 倍以上，首次在 MLE 领域实现了大规模、基于轨迹的在线强化学习。在 MLE-bench-lite 上，SandMLE 在 Qwen3-8B、14B 和 30B-A3B 模型上均比 SFT 基准取得了显著提升，相对奖牌率（medal rate）提升幅度在 20.3% 到 66.9% 之间。此外，训练后的策略在未见的智能体脚手架（agentic scaffolds）上具有泛化性，在 MLE-Dojo 上的 HumanRank 分数提升了高达 32.4%。

## Abstract
As large language model agents advance beyond software engineering (SWE) tasks toward machine learning engineering (MLE), verifying agent behavior becomes orders of magnitude more expensive: while SWE tasks can be verified via fast-executing unit tests, MLE verification requires running full ML pipelines -- data preprocessing, model training, and metric evaluation -- on large datasets at each rollout step, rendering trajectory-wise on-policy reinforcement learning (RL) prohibitively slow. Existing approaches retreat to supervised fine-tuning (SFT) or offline proxy rewards, sacrificing the exploration and generalization benefits of on-policy RL. We observe that sandbox data size is the primary source of this bottleneck. Based on this insight, we introduce SandMLE, a multi-agent framework that generates diverse, verifiable synthetic MLE environments from a small number of seed tasks, preserving the structural and technical complexity of real-world problems while constraining datasets to micro-scale (each task is paired with only 50-200 training samples). Through extensive experiments, we show that SandMLE reduces execution time by over 13 times, enabling large-scale, on-policy trajectory-wise RL for the first time in the MLE domain. On MLE-bench-lite, SandMLE yields significant gains over SFT baselines across Qwen3-8B, 14B, and 30B-A3B, with relative medal rate improvements ranging from 20.3% to 66.9%. Furthermore, the trained policy generalizes across unseen agentic scaffolds, achieving up to 32.4% better HumanRank score on MLE-Dojo.

### 2. 方法论
SandMLE 包含两个核心部分：**自动化合成环境生成**和**轨迹级强化学习**。

#### A. 多智能体合成流水线（Environment Factory）
通过四个专门的智能体协作生成任务：
1.  **数据策略师 (Data Strategist)**：从原始种子任务中提取“结构 DNA”（如数据模态、标签分布），并将其映射到新的行业场景，注入数学隐藏规则和噪声。
2.  **MLE 开发员 (MLE Developer)**：编写代码生成微型数据集（50-200 样本），确保数据量极小但逻辑严密，并建立性能基准。
3.  **MLOps 工程师 (MLOps Engineer)**：构建自动评分沙盒，设定确定性的评估指标和里程碑阈值。
4.  **技术作家 (Technical Writer)**：生成符合新场景的任务说明文档。

#### B. 轨迹级 GRPO 训练
*   **算法**：采用 **GRPO**（组相对策略优化），这是一种无需评论者（Critic）网络的高效 RL 算法。
*   **密集奖励函数**：为了解决奖励稀疏问题，设计了多维奖励：
    *   **格式奖励**：推理标签（`<think>`）的使用规范。
    *   **执行奖励**：代码是否能成功运行并生成提交文件。
    *   **里程碑奖励**：根据模型表现是否超过预设的“中位数”、“铜牌”、“银牌”、“金牌”阈值给予阶梯式奖励。

### 3. 实验设计
*   **数据集/基准**：
    *   **训练集**：基于 60 个原始种子任务生成的 848 个合成任务。
    *   **评估集**：**MLE-bench-lite**（22 个真实 Kaggle 任务）和 **MLE-Dojo**（62 个交互式任务）。
*   **对比方法**：
    *   **Base**：原始 Qwen3 模型（8B, 14B, 30B）。
    *   **Seed-SFT**：在由 Claude-4.5 生成的高质量专家轨迹上进行监督微调。
    *   **SandMLE**：直接进行 RL 训练。
    *   **SFT-SandMLE**：先 SFT 再进行 RL 训练。
*   **智能体脚手架**：测试了 ReAct, AIDE, AIRA, MLE-Agent 等多种框架以验证泛化性。

### 4. 资源与算力
*   **硬件**：使用了 **NVIDIA H200 GPU**。
*   **训练细节**：
    *   每个任务分配 1 块 H200。
    *   GRPO 训练 100 步，学习率 $1 \times 10^{-6}$，Batch Size 为 16，组大小（Group Size）为 4。
    *   训练过程中单步执行限时 90 秒，评估时限时 4 小时。
*   **效率提升**：平均执行时间从 196.17s 降至 14.31s（提升 13.7 倍）。

### 5. 实验数量与充分性
*   **实验规模**：涵盖了从 8B 到 30B 不同参数规模的模型，并在两个主流 MLE 基准上进行了测试。
*   **消融实验**：专门对比了“密集里程碑奖励”与“稀疏奖励”的效果，证明了密集奖励对 30B 模型性能翻倍的关键作用。
*   **泛化性测试**：在训练时仅使用 ReAct 框架，但在评估时切换到 AIDE 和 AIRA 等完全不同的脚手架，实验设计较为客观、公平，能够证明模型内在能力的提升而非对特定框架的过拟合。

### 6. 主要结论与发现
*   **性能飞跃**：SandMLE 在所有规模上均显著优于 SFT。在 30B 模型上，Any Medal 率从 13.6% 提升至 27.3%。
*   **RL 优于 SFT**：单纯的 SFT 往往只能学会格式，而 RL 能让模型学会真正的迭代试错和推理策略。
*   **跨框架泛化**：训练后的模型在未见的智能体框架上依然有效，在 MLE-Dojo 上的 HumanRank 分数显著提升。
*   **测试时缩放 (Test-time Scaling)**：模型展现出随思考步数增加而性能提升的趋势，但在超过 30 步后受限于上下文窗口长度。

### 7. 优点
*   **创新性解决效率问题**：通过“微缩沙盒”的概念，巧妙绕过了 MLE 训练中昂贵的计算成本。
*   **全自动化流水线**：任务生成过程无需人工干预，具备极强的可扩展性。
*   **实战导向**：不仅关注最终分数，还通过里程碑奖励引导模型学习工程化的解决步骤。

### 8. 不足与局限

*   **种子任务依赖**：合成任务的多样性仍受限于初始 60 个种子任务的质量。
